# 生成式 AI 时代的资深研究者存在主义危机

> 原文：[`towardsdatascience.com/an-existential-crisis-of-a-veteran-researcher-in-the-age-of-generative-ai/`](https://towardsdatascience.com/an-existential-crisis-of-a-veteran-researcher-in-the-age-of-generative-ai/)

<mdspan datatext="el1745361868385" class="mdspan-comment">我曾是研究者</mdspan>十五年前。一个长时间进行研究的博士候选人。我淹没在许多文章、注释、电子邮件、书签等等中。当我发现文献管理工具 Mendeley 时，我感到如此放松。感觉我又控制了整个过程。当我发现书签管理工具 XBookmark 时，我感到如此高效（仍然有书签）。它们在我当时工作得很好，我完成了我的博士课程并获得了学位。

## 第一集 — 面对现实

这些天，我遇到了一个存在主义危机。我看到 AI 研究助手工具的进步。那天我正在和 Scinito 一起工作，我感到震惊。我试图说服自己，这只是一个帮助进行文献综述的工具。那些做过博士的人都知道，进行扎实的文献综述有多难。这不是玩笑。你必须阅读超过 100 篇文章，对它们进行分类，理解它们，并总结它们。如果我说 15 年前进行扎实的文献综述需要 3-6 个月，我并没有说错。你们听明白了，3-6 个月是你宝贵的生命时光。

起初，我试图说服自己，Scinito 或其他类似工具对研究者只提供了边际价值。遗憾的是，或者说是幸运的是，我错了……

他们不仅能在瞬间为你完成文献综述（对不起，我的同行们，你们听对了），还能对你的文章进行同行评审。我永远不会忘记我本应该等待我的导师和顾问们评审我的文章有多久，我们之间有多少来回的邮件直到事情达到可接受的质量。即使在这些努力之后，你也会从期刊的同行评审中获得广泛的反馈，以考虑你的文章是否可以发表。或者，你可能在 3 个月或 6 个月后因为选择了错误的期刊而遭到拒绝。这些 AI 研究助手工具可以增强所有这些步骤：评审你的文章并为你选择最相关的期刊。

太神奇了。对于这个时代的研究者来说，这确实很神奇，但我看到我在某件事上花费了这么多时间，而这件事本可以更容易、更快地完成，我感到很悲伤。有趣的是，这并不是季节的结束，这只是开始。

这个挑战不仅针对研究人员，也适用于软件开发者。像 Cursor IDE 这样的工具已经极大地改变了我们构建软件的方式。在我完成博士学位后，我开始在工程领域工作。因此，我做了大量的编码、测试等工作。如今，我不再需要阅读 Stack Overflow 来调试我的代码。我也不需要花时间为我自己的代码编写测试。我不再需要成为 React、HTML 或 CSS 的专家才能构建一个网站。我过去在构建网站上花了多少时间？我不想再想这个问题了！

## 第二集 — 接受现实

让我分享一下玻璃杯的一半体验吧。这真是太酷了。我可以要求 AI 研究助手工具在庞大的数据库中进行语义搜索。这是我们以前无法做到的。那只是关键词匹配。我可以通过阅读 AI 在几秒钟内生成的文献综述，在几小时内获取任何主题或研究问题的更新。我可以轻松地编写 LaTeX 代码。我可以在几分钟内将我的论文重新格式化到任何指南中。我为我们这个时代的学者感到高兴。他们可以花更多的时间在创造性、解决问题上，当然，还有他们宝贵的生活，而不是做那些不必要的耗时任务。

我也为我自己感到高兴。我可以使用我想要的任何编程语言编写代码。我可以构建网站，而无需被锁定在 Wix 或 WordPress 上。我可以编写我需要的任何 Python 代码。我可以优化它，并为它编写一系列测试。哇！这真是太酷了。编码、设计、研究以及一切都在快速发展。无论人们或组织如何抵制，技术总会找到自己的道路。

这里有一个问题。用单一（且仅有一个）提示来构建网站的承诺是不正确的。我是根据一个非常最新的经验告诉你们的。这些天，我和一位同事一起在构建一个新网站，我们俩都是软件和 AI 的专家。这次我们甚至没有考虑过 Wix 或 WordPress。我们开始使用 Cursor 及其 Claude-3.7-sonnet 的 Agent 体验。Cursor 的代理可以在一秒钟内生成网站结构，但在细节上却失败了。

例如，当你想要将两种不同的文本相互对齐时，尤其是当其中一个是静态的，另一个是动态的，AI 就无法做到正确对齐。基本上，AI 可以在一秒钟内完成网站结构的构建，但无法像 UI 设计专家那样做好所需的细节（你希望作为人类在预构建结构之上应用的一些细节）。这意味着，尽管我们不需要成为 React 或 CSS 的专家，但我们必须在需要时能够干预代码库。此外，我们必须对概念有足够的了解，以便能够详细阐述它们。如果你说不出来，AI 就无法做到！

我并不惊讶于人工智能模型的这种弱点。它们是基于“群体智慧”原则构建的。这意味着它们是基于汇总最常见的内容构建的，而不是模仿个别专家的直觉。这根植于它们的根本。它们在普遍性上令人惊叹，但在特定性上却有所欠缺。在这段简短的播客中，我从不同的角度解释了类似的概念：“[特定性的侵蚀](https://www.youtube.com/watch?v=7KoVOgpJfyA&ab_channel=AI%3AUnorthodoxLessons)。”

## 最后的话

我很幸运能成为人工智能社区的一员。我是一个有稳固计划来拥抱这一技术变革的人工智能架构师。但我担心许多其他人无法应对这种变化。这绝非易事。如果你因为人工智能而在职业生涯中经历了存在主义的时刻，请告诉我。我可能知道一些能帮到你的东西。

> 如果我能在这里分享一个建议，那将是“深入学习基础”。你可以（应该）将重复性、高级和通用任务留给人工智能，并用你的人类创造力和专业知识专注于细节，让你的工作/产品更加出色。
