# 初创公司数据科学家的真实面貌

> [`towardsdatascience.com/what-being-a-data-scientist-at-a-startup-really-looks-like/`](https://towardsdatascience.com/what-being-a-data-scientist-at-a-startup-really-looks-like/)

<mdspan datatext="el1756874384863" class="mdspan-comment">在令人难以置信的</mdspan>五年后，我在八月份离开了 Brex。时间真的飞逝。我是在 Brex 成立三年时加入的，我仍然清晰地记得我的第一天。同时，这五年感觉要长得多。这一切都归功于我在路上获得的大量知识，从加入一个年轻的数据团队并学习完整的数据周期，到与运营、GTM 和产品团队合作，最终[过渡到管理](https://towardsdatascience.com/from-data-scientist-ic-to-manager-one-year-in/)。这种加速增长是作为数据科学家加入初创公司最大的好处之一。

“我应该加入初创公司还是一家更成熟的公司？”我经常从我的网络中听到这个问题。在这篇文章中，我将根据我的个人经验分享初创公司数据科学家的生活。希望这能帮助你在自己的职业道路上找到方向。

*快速笔记：“初创公司”是一个广泛的概念，涵盖了多种类型的公司。为了便于理解，Brex 是一家位于美国的金融科技初创公司，为商业客户提供企业卡、银行服务和费用管理软件。我在拥有三年全职产品数据科学经验后加入了 Brex。当时，Brex 是一家拥有 500 多名员工的三年级 C 轮融资初创公司。因此，我的经验可能与你在早期阶段公司的工作经验不同。*

* * *

## I. 领域特定与目标导向

初创公司发展迅速——他们需要根据市场反馈调整策略，并不断迭代以确保产品与市场匹配。作为一项昂贵的资源，数据科学家需要相应地调整他们的关注领域，以创造最大的商业价值。

以我自己的经验为例：

1.  当我最初加入 Brex 时，优先事项是尽可能多地上线客户，同时控制欺诈风险。因此，我密切与上线团队合作，优化账户申请流程，并改善运营团队的资源分配。

1.  随着我们的客户群不断扩大，下一个优先事项是扩展我们的客户支持（CX）功能，以提供一流的客户支持。我与 CX 团队合作，分析客户痛点并减少产品摩擦。（阅读[我的文章](https://towardsdatascience.com/the-secret-power-of-data-science-in-customer-support/)了解更多关于 CX 中数据科学的细节。）

1.  之后，我与实施团队合作，加快新客户的上线速度，并与产品团队合作，识别流失驱动因素并提高留存率。

正如你所见，初创公司非常以目标为导向。在大公司中，你可能会专注于一个产品特性多年；而在初创公司，公司的需求可能在季度内完全改变，数据科学家也因此更频繁地改变他们的领域。

优点是这允许数据科学家探索并与更多功能协作，对业务有更全面的了解。在过去五年中，我几乎与 Brex 的每个团队都合作过——观察 GTM（市场进入）如何寻找潜在客户并关闭交易，看到产品与客户支持如何共同解决客户痛点，以及学习客户成功团队如何降低客户流失率并推动增值销售。这比阅读书籍或参加商学院的课程更有效地了解业务运营。这也确保了数据科学家有明确且具有高影响力的业务问题要解决。

但挑战是什么？数据科学家需要遵循公司的战略并不断适应变化。这意味着相应地调整优先级，有时不得不以最快的方式构建事物，而不是最可扩展或最可靠的方式。随着大量一次性仪表板和多个版本的同一样指标的累积，技术债务迅速增加，这会让利益相关者和数据团队本身感到困惑。建立最佳实践需要时间，因此有时不得不与混乱共存——尽管能够从零开始建立标准（或看到别人这样做）也是初创公司工作中的一个有回报的部分，尤其是对于有志成为数据科学领导者的人来说。

## II. 数据分析师 vs. 数据工程师 vs. 数据科学家？以上皆是。

今天的数据职位名称很令人困惑——一些数据科学家专注于实验，而另一些则深入机器学习；一些数据分析师只是构建仪表板，但一些主要进行产品分析。但说到初创公司，头衔的重要性较小——你可能会做所有的事情。

当我加入 Brex 时，每个人都拥有相同的头衔，“数据科学家”，但我们所有人都必须戴多顶帽子。我们只是在去年初将数据团队分为 DS、DA 和 DE 功能。如我上面提到的，初创公司以目标为导向。因此，当人员有限且没有明确的团队结构时，你需要做一切事情来实现目标。

+   **数据工程**：在过去五年中，我学习了数据建模、Airflow、ETL 流程、SQL 优化等许多数据工程技能。[数据建模](https://towardsdatascience.com/top-5-principles-for-building-user-friendly-data-tables-0dfe168cecc1/)、Airflow、ETL 流程、[SQL 优化](https://towardsdatascience.com/mastering-sql-optimization-from-functional-to-efficient-queries-74d8692f10be/)等。我还记得我在 Brex 的第一个项目是将我们的数据管道从 dbt 迁移到内部工具。是的，作为初创公司的数据科学家，你很可能也需要构建、拥有和维护自己的数据管道。

+   **分析**：加入一家初创公司意味着有许多业务领域没有或只有很少的数据支持。因此，为了帮助团队更好地理解他们的表现并赢得他们的信任，在你建立数据管道之后的第一步是定义成功指标并构建仪表板。一旦指标得到密切监控，接下来自然会出现诸如为什么它们会移动或如何移动它们等问题。这些都是常见的分析任务。

+   **数据科学**：初创公司也有很多高级数据科学用例。机器学习模型有助于检测欺诈、预测流失、估算 LTV（终身价值）等。因果推断对于评估营销活动或产品发布的影响也至关重要。

同时戴多顶帽子是好事还是坏事？

如果你还没有确定你对哪种类型的数据跟踪最感兴趣，加入一家初创公司将帮助你尝试一切，并决定下一步走哪条路。或者，如果你希望成为数据部门负责人甚至有朝一日建立自己的公司，这种全周期的接触是非常宝贵的。

同时，缺点也很明显——你会在你不太感兴趣或与你的长期职业目标不太相关的事情上浪费时间。你拥有整个数据生命周期的事实也可能让利益相关者感到困惑，因为他们通常只关心某种类型的输出，例如仪表板或模型，但没有意识到你需要花另外 50%的时间在数据管道上。

## III. 更高的可见性

初创公司的数据科学家从不缺乏可见性。自从加入 Brex 的第一天起，我就有机会直接与高管层合作。领导层经常会带着紧急且重要的业务问题来找你，希望你能施展一些数据魔法来揭示洞察力并推动业务增长。这通常不是你在一家成熟公司能体验到的，尤其是作为一名初级数据科学家。这是一个高压但极具回报的环境。

例如，在 2023 年 3 月的[硅谷银行危机](https://en.wikipedia.org/wiki/Collapse_of_Silicon_Valley_Bank)期间，许多初创公司受到了影响，面临着失去运营资金的危险。我与领导团队紧密合作，帮助初创公司度过这段艰难时期。我创建了一个实时跟踪新客户申请的系统，分析了申请审查速度以估算额外的人力需求，并与其他数据科学家合作自动化和加速入职检查。这是一个紧张的周末，我们从早上 8 点工作到午夜，在战室（通过 Zoom 虚拟会议）跨职能工作。然而，这却是我在 Brex 最好的回忆之一，展示了我们领导层的真正客户至上，以及数据科学家如何直接贡献并推动巨大的商业影响。

## IV. 接触新工具

年轻的初创公司也足够勇敢地尝试新的技术堆栈。因此，它们通常是新工具的早期采用者，而大公司可能需要数月（以及多层审批）才能进行试点，更不用说重大的迁移成本了。

例如，在我任职 Brex 期间，

+   我们两年前开始探索数据科学中的 LLM 用例，并内部访问了所有主要的 LLM API（OpenAI、Claude 和 Gemini），对所有员工开放。每年，Brex 都会举办内部黑客马拉松，并鼓励员工创新。两年前，我与其他工程师合作构建了一个[AI 驱动的客户反馈平台](https://medium.com/brexeng/brexs-ai-powered-engine-for-identifying-customer-insights-72aadeb62e8e?source=list---------5-------predefined%3A5462c48cfc57%3AREADING_LIST----------------------------)，以自动分类、总结和分析各种非结构化反馈数据。去年，我构建了一个基于 RAG 的聊天机器人，帮助利益相关者轻松检索与特定产品功能相关的客户反馈。今年，我与其他数据科学家一起探索了 Claude Code 和 Snowflake CLI 的文本到 SQL 功能。我非常享受将这些尖端技术应用于数据科学工作流程的机会。

+   我们经常试点新的数据解决方案。例如，我们是协作和易于共享数据笔记本的[Hex](https://hex.tech/)的早期客户。我们使用[Statsig](https://www.statsig.com/)进行实验和事件跟踪。我们还尝试了各种 AI 驱动的商业智能软件，以实现更好的自助式分析。

在初创公司工作帮助我了解新技术并将其融入我的日常工作中。这不仅使工作更有趣，而且随着行业的演变，也使我保持了竞争力。

另一方面，作为早期采用者，可能意味着颠覆现有的工作流程和重建基础设施。这也意味着一个不太稳定的开发体验。

## 结论

那么，你应该加入初创公司还是一家更成熟的公司呢？

初创公司提供速度、多样性、可见性和前沿的曝光。但它们也带来了混乱、不断变化的重心和可能不喜欢的角色。如果你在不确定性中茁壮成长，并希望加速你的学习曲线，初创公司可以是一个作为数据科学家成长的绝佳地方。

对我来说，在 Brex 的五年来，我学到了关于商业和数据的宝贵知识。我将永远感激这些教训、这些人以及看到数据科学在快速增长的初创公司中可能是什么样子的机会。
