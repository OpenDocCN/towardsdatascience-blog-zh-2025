### 供应链控制塔 AI 代理—使用 n8n 构建

#### 什么是 n8n？

n8n 是一个开源的工作流程自动化工具，它允许你轻松连接应用程序（电子邮件、CRM、消息系统）、API 和 AI 模型框架，如 LangChain。

你通过连接预构建的节点来构建工作流程。

![图片](img/cbca6f1aab92c0da813a31627beab13c.png)

使用 4 个节点的 AI 驱动的电子邮件解析器—（图片由 Samir Saci 提供）

例如，上面的工作流程**处理电子邮件**

+   第一个节点从 Gmail 账户收集电子邮件。

+   电子邮件内容和元数据被发送到 AI 代理节点，该节点提取相关信息。

+   第三个节点使用 JavaScript 处理输出。

+   最终节点将结果加载到 Google 表格中。

构建此工作流程不需要代码—除了第三个节点，它只使用了两行 JavaScript。

由于我与一支 Python 技能有限的供应链顾问团队合作，这对我来说是一个转折点，因为我寻求开发我的服务产品。

他们可以在 n8n 上经过简短的培训后轻松使用、适应和维护此工作流程。

#### AI 供应链控制塔 n8n 工作流程

AI 供应链控制塔工作流程稍微复杂一些—但仍然比其 Python 版本简单得多。

它包括两个子工作流程。

![图片](img/ddb623d84eb25263f6b2643bcdf696ba.png)

包含 AI 代理的主子工作流程—（图片由 Samir Saci 提供）

主子工作流程包括聊天界面和 AI 代理。

对于 AI 代理节点，你需要

+   使用一个节点连接 LLM（聊天模型），在该节点中输入你的 API 凭证

+   添加一个内存节点来管理对话

+   为 SQL 查询添加一个工具节点，链接到第二个子工作流程

AI 代理生成一个 SQL 查询并将其发送到“调用查询工具”节点，该节点执行查询。

![图片](img/c9f237d88b717939495f3eb3ec885c64.png)

通过“调用查询工具”连接的第二个子工作流程—（图片由 Samir Saci 提供）

子工作流程包括一个代码节点，用于清理查询（删除额外空格和阻止像 DELETE 这样的风险命令）。

输出被发送到一个**BigQuery 节点**，该节点运行查询并返回结果。

该过程非常顺畅，配置有限：

+   系统提示（在 AI 代理节点中）

+   用户提示（在 AI 代理节点中）

![图片](img/ed6203dba48216924231ef3418eb6fb5.png)

AI 代理节点系统提示窗口—（图片由 Samir Saci 提供）

此设置不需要 Python 技能，可以直接由我的顾问处理。

### 结论

这个例子展示了使用 n8n 和最小代码构建 AI 代理是多么容易。

> **这意味着 Python 不再需要用于供应链分析了吗？绝对不是！**

就像许多低代码平台一样，功能限制在框架内可用的范围内。

正因如此，我将它作为我分析产品的补充。

![](img/9b81d5f7ffbca96872b02be141892d4e.png)

使用 HTTP 节点将 AI 代理连接到我的分析产品后端（图片由 Samir Saci 提供）

要做到这一点，你可以使用 HTTP 请求节点将你的工作流程连接到你的分析后端。

#### 还有什么？易于连接到许多服务。

我选择 n8n 来丰富我的分析产品的另一个原因是添加额外连接的简便性。

例如，如果你想添加 Slack 界面或将对话记录到 Google 表格中，只需在你的工作流程中添加一个新的节点。
