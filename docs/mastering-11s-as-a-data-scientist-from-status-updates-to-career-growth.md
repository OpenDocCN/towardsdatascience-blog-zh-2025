# 数据科学家掌握 1:1 会议技巧：从状态更新到职业成长

> 原文：[`towardsdatascience.com/mastering-11s-as-a-data-scientist-from-status-updates-to-career-growth/`](https://towardsdatascience.com/mastering-11s-as-a-data-scientist-from-status-updates-to-career-growth/)

我已经担任数据团队经理六个月了，我的团队从三个人增长到了五个人。

我在 11 月[撰写](https://towardsdatascience.com/from-data-scientist-to-data-manager-my-first-3-months-leading-a-team-40c1c7c05e5c/)了我的最初的管理员经历。在这篇文章中，我想谈谈对 DS 或 DA 个人贡献者（IC）与其经理之间关系更为重要的东西——1:1 会议。我记得当我刚开始我的职业生涯时，我在 1:1 会议中感到紧张和不自在，因为我不知道该期待什么，也不知道什么是有用的。现在，在 1:1 会议中经历了双方的角色，我更好地理解了如何进行有效的 1:1 会议。

如果你曾经为如何充分利用你的 1:1 会议而感到困扰，这里是我的关键建议。

## **I. 设定定期的 1:1 会议节奏**

首先，与你的经理的 1:1 会议应该定期进行。这可能是每周一次或每两周一次，取决于你项目的工作节奏。例如，如果你更专注于分析，并且有很多快速移动的报告和分析任务，那么每周的 1:1 会议可能更好，以便及时更新并就项目优先级达成一致。然而，如果你专注于一个跨越多周的长期机器学习项目，你可能更舒服地采用每两周一次的节奏——这让你有时间进行研究，尝试不同的方法，并在 1:1 会议中进行有意义的对话。

我与我的团队中的每个人都设有每周重复的 30 分钟 1:1 时段，只是为了确保我每周都有这个专门的时间。这些会议有时会变成短暂的 15 分钟聊天，甚至关于工作后的生活的闲聊，但我仍然发现它们对于了解每个人心中的首要任务和建立个人联系非常有帮助。

## **II. 准备工作并更新你的 1:1 议程**

准备你的 1:1 会议至关重要。我与我的经理共享一个 1:1 文档，并在每次会议前更新它。我也很欣赏我的直接下属提前准备他们的 1:1 议程。原因如下：

+   在整个周内，我喜欢在我 1:1 文档中迅速记下任何一闪而过的讨论主题。这确保了我在会议中涵盖了所有重要点，并提高了沟通效率。

+   制定议程有助于你和你的经理跟踪讨论的内容，并确保每个人都承担责任。我们每天都会和很多人交谈，所以如果你忘记了你向某人提到的事情，这是完全正常的。因此，拥有这样的文档会提醒你之前的对话。现在，作为一个管理着五人团队的管理者，我也转向 1:1 文档来确保我解决了上次会议的所有开放问题和行动项，并找到过去项目的链接。

+   它还可以协助你的绩效评估过程。当撰写我的自我评估时，我会阅读我的 1:1 文档来列出我的成就。同样，我也使用 1:1 文档与我的团队一起确保我不会错过他们项目中的任何亮点。

那么，一对一会议的合适话题有哪些呢？请参阅下文部分。

## **III. 1:1 会议议程上的话题**

虽然每位经理都有自己的偏好，但有一系列话题通常适合一对一会议。你不必涵盖所有这些话题，但我希望它们能给你一些灵感，让你不再对一对一会议感到迷茫。

+   **自上次 1:1 会议以来的成就**：我建议在 1:1 文档中列出最新的成就。你不必在会议中详细讨论每一个，但向你的经理展示你的能力和提醒他们你的优秀是很好的。同时，突出你的努力和影响也是一个好主意。商业通常是受影响驱动的，数据团队也不例外。如果你的 A/B 测试导致了一个是/否的决定，请在会议中提及。如果你的分析导致了一个产品想法，提出并讨论你计划如何支持开发和衡量影响。

+   **正在进行和即将到来的项目**：在我 7 年的职业生涯中，我观察到的一个常见模式是数据团队通常有很长的待办事项列表，包含许多“紧急”请求。一对一会议是和你的经理一起调整优先级和时间的良好时机。

    +   如果你的项目受阻，让你的经理知道。虽然独立性总是受到赞赏，但意外的阻碍可能会随时出现。与你的经理一起解决阻碍是完全可以接受的，因为他们通常有更多的经验，并且应该授权你完成你的项目。提前让你的经理知道比让他们自己后来发现并询问你为什么错过了时间线要好。同时，理想情况下，你不仅提出阻碍，还提出可能的解决方案或请求具体帮助。例如，“我在访问 X 数据上遇到了阻碍。我应该优先与数据工程师一起构建数据管道，还是推动临时提取？”这表明你是一个真正的解决问题者，具有成长心态。

+   **职业成长**：你还可以利用一对一的时间来讨论职业成长话题。对于数据科学家来说，职业成长不仅仅是晋升。你可能更感兴趣的是在特定领域，如实验，增长技术专长，或者从数据科学（DS）转向不同的职能，如 MLE，或者获得领导经验并过渡到人员管理角色，就像我一样。为了确保你朝着职业目标前进，你应该定期与你的经理进行这样的对话，以便他们可以提供相应的建议，并为你匹配与你的长期目标一致的项目。

    +   我还与我的团队进行每月的职业成长检查会议，专门讨论职业进展。如果你总是发现你的 1:1 时间被项目更新占据，考虑与你的经理安排这样的单独会议。

+   **反馈**：反馈应该是双向的。

    +   你的经理可能没有你那么多的时间来处理数据项目。因此，你可能会注意到他们在项目工作流程、分析过程或跨职能协作中可能没有意识到的低效。不要犹豫，提出这些问题。并且，与处理阻碍因素一样，建议在开会之前先考虑可能的解决方案，以向你的经理展示你是一个团队玩家，你为团队的文化和成功做出了贡献。例如，与其说“我们收到了太多的临时请求”，不如将其表述为“通过 Slack DM 收到的临时请求减少了我们在计划项目上的专注时间。我们能否邀请利益相关者参加我们的冲刺规划会议，以协调优先级，并在冲刺期间有一个更正式的请求接收流程？”

    +   同时，你也可以利用这个机会向你的经理询问任何关于你表现的反馈。这有助于你识别差距，持续改进，并确保在正式的绩效评估期间没有惊喜 :)。

+   **团队和公司目标**：变化是商业中唯一不变的因素。数据团队与利益相关者紧密合作，因此数据科学家需要了解公司的优先事项和目前最重要的事情。例如，如果你的公司正在关注留存率，你可能想分析导致更高留存率的驱动因素，并向你的利益相关者提出相应的营销活动想法。

为了给你一个更具体的一对一议程的想法，让我们假设你在一家消费银行工作，专注于信用卡奖励领域。以下是一个示例议程：

*日期：2025 年 3 月 3 日*

*✅ 上周成就*

+   ***奖励 A/B 测试分析*** *[链接](https://example.org)*：与利益相关者共享，我们将在第一季度向更广泛的用户推出获胜的治疗方案 A。

+   ***奖励兑换分析*** *[链接](https://example.org)*：大多数用户兑换奖励用于账单余额。与营销团队交谈，以运行广告其他兑换选项的电子邮件活动。

***🗒*** *正在进行的项目*

+   ***[P0] 奖励<>流失分析****：了解奖励活动是否与流失相关。预计 3/7 完成。

+   ***[P1] 奖励成本仪表板****：构建一个跟踪所有奖励活动成本的仪表板。预计 3/12 完成。

+   ***[阻塞] 旅行信用额度使用仪表板****：等待 DE 设置旅行预订表。2/27 跟进。需要升级吗？

+   ***[优先级降低] 零售商户奖金奖励活动支持****：由于我们推迟了活动，营销团队将其优先级降低。

*🔍 其他主题*

+   *我想在机器学习方面获得更多经验。有没有任何项目机会？*

+   *任何关于我与利益相关者合作反馈的吗？*

请记住，在会议期间，你应该积极更新你的 1:1 文档。它应该反映讨论的内容，并包括每个要点的重要笔记。你甚至可以在每个会议议程的底部添加一个“行动项目”部分，以使下一步行动清晰。

## **最终想法**

以上是我作为数据科学家有效进行 1:1 会议的基本技巧。通过**建立****定期会议**、**准备深思熟虑的议程**和**涵盖有意义的主题**，你可以将这些会议从尴尬的状态更新转变为有价值的发展机会。记住，你的 1:1 不仅仅是更新你的经理——它关于获得你成长角色所需的支撑、指导和可见性。
