# 从零开始构建 AI 代理：单个代理

> [`towardsdatascience.com/ai-agents-from-zero-to-hero-part-1/`](https://towardsdatascience.com/ai-agents-from-zero-to-hero-part-1/)

## **简介**

**AI 代理**是执行任务、做出决策并与他人通信的自主程序。通常，它们使用一系列工具来帮助完成任务。在 GenAI 应用中，这些代理处理顺序推理，并在 LLM 知识不足时可以使用外部工具（如网络搜索或数据库查询）。与不确定时生成随机文本的基本聊天机器人不同，AI 代理激活工具以提供更准确、更具体的响应。

我们越来越接近**代理 AI**的概念：这些系统表现出更高水平的自主性和决策能力，无需直接的人为干预。虽然今天的 AI 代理对人类输入做出反应，但明天的代理 AI 将主动参与问题解决，并可以根据情况调整其行为。

今天，从头开始构建代理变得和 10 年前训练逻辑回归模型一样简单。当时，*Scikit-Learn*提供了一个简单的库，只需几行代码就可以快速训练机器学习模型，抽象掉了许多底层复杂性。

在这个教程中，我将展示如何**从头开始构建不同类型的 AI 代理**，从简单到更高级的系统。我将提供一些有用的 Python 代码，这些代码可以轻松应用于其他类似情况（只需复制、粘贴、运行），并逐行代码进行注释，以便您可以复制此示例。

## **设置**

正如我所说，任何人都可以免费运行本地自定义代理，无需 GPU 或 API 密钥。唯一必要的库是[***Ollama***](https://ollama.com/)(`pip install ollama==0.4.7`)，因为它允许用户在本地运行 LLMs，无需云服务，从而在数据隐私和性能方面提供更多控制。

首先，您需要从网站下载*Ollama*。

![图片](img/a6d60584479cb450d63a0dfebc540e5a.png)

然后，在您的笔记本电脑的提示符 shell 中，使用命令下载选定的 LLM。我选择阿里巴巴的***Qwen***，因为它既智能又轻量。

![图片](img/5e62ad326652c20f96342b8f33f5046d.png)

下载完成后，您可以继续使用 Python 并开始编写代码。

```py
import ollama
llm = "qwen2.5"
```

让我们测试 LLM：

```py
stream = ollama.generate(model=llm, prompt='''what time is it?''', stream=True)
for chunk in stream:
    print(chunk['response'], end='', flush=True)
```

![图片](img/f7a8bbb17c7f11d6cb47f2c23c233880.png)

显然，LLM 本身非常有限，它除了聊天之外几乎什么也做不了。因此，我们需要提供它采取行动的可能性，换句话说，就是**激活工具**。

最常见的工具之一是**搜索互联网**的能力。在 Python 中，最简单的方法是使用著名的私有浏览器[*DuckDuckGo*](https://pypi.org/project/duckduckgo-search/)(`pip install duckduckgo-search==6.3.5`)。你可以直接使用原始库或导入[*LangChain*](https://www.langchain.com/)包装器(`pip install langchain-community==0.3.17`)。

使用*Ollama*，为了使用工具，必须在字典中描述该函数。

```py
from langchain_community.tools import DuckDuckGoSearchResults
def search_web(query: str) -> str:
  return DuckDuckGoSearchResults(backend="news").run(query)

tool_search_web = {'type':'function', 'function':{
  'name': 'search_web',
  'description': 'Search the web',
  'parameters': {'type': 'object',
                'required': ['query'],
                'properties': {
                    'query': {'type':'str', 'description':'the topic or subject to search on the web'},
}}}}
## test
search_web(query="nvidia")
```

![图片](img/24112f07f627a8f8c3d9148c59ef9ff4.png)

网络搜索可能非常广泛，我希望给代理提供更精确的选项。比如说，我计划使用这个代理来了解金融更新，所以我可以给它一个特定主题的工具，比如只搜索金融网站而不是整个网络。

```py
def search_yf(query: str) -> str:
  engine = DuckDuckGoSearchResults(backend="news")
  return engine.run(f"site:finance.yahoo.com {query}")

tool_search_yf = {'type':'function', 'function':{
  'name': 'search_yf',
  'description': 'Search for specific financial news',
  'parameters': {'type': 'object',
                'required': ['query'],
                'properties': {
                    'query': {'type':'str', 'description':'the financial topic or subject to search'},
}}}}

## test
search_yf(query="nvidia")
```

## **简单代理（网络搜索**）

在我看来，最基本的代理至少应该能够在一到两个工具之间进行选择，并重新阐述动作的输出，以给用户提供恰当且简洁的答案。

![图片](img/c6a233baf0362a73fffb81338354710e.png)

首先，你需要写一个提示来描述代理的目的，越详细越好（我的非常通用），这将作为与 LLM 聊天历史中的第一条消息。

```py
prompt = '''You are an assistant with access to tools, you must decide when to use tools to answer user message.''' 
messages = [{"role":"system", "content":prompt}]
```

为了保持与 AI 的聊天活跃，我将使用一个循环，从用户输入开始，然后调用代理来响应（这可能来自 LLM 的文本或激活一个工具）。

```py
while True:
    ## user input
    try:
        q = input('🙂 >')
    except EOFError:
        break
    if q == "quit":
        break
    if q.strip() == "":
        continue
    messages.append( {"role":"user", "content":q} )

    ## model
    agent_res = ollama.chat(
        model=llm,
        tools=[tool_search_web, tool_search_yf],
        messages=messages)
```

到目前为止，聊天历史可能看起来像这样：

![图片](img/f04df8b8d50074c4f2b84bc8c81ca716.png)

如果模型想要使用工具，需要运行适当的函数，并使用 LLM 在其响应对象中建议的输入参数：

![图片](img/1efd38e338e011c1f4c9a743b0b058f8.png)

因此，我们的代码需要获取这些信息并运行工具函数。

```py
## response
    dic_tools = {'search_web':search_web, 'search_yf':search_yf}

    if "tool_calls" in agent_res["message"].keys():
        for tool in agent_res["message"]["tool_calls"]:
            t_name, t_inputs = tool["function"]["name"], tool["function"]["arguments"]
            if f := dic_tools.get(t_name):
                ### calling tool
                print('🔧 >', f"\x1b[1;31m{t_name} -> Inputs: {t_inputs}\x1b[0m")
                messages.append( {"role":"user", "content":"use tool '"+t_name+"' with inputs: "+str(t_inputs)} )
                ### tool output
                t_output = f(**tool["function"]["arguments"])
                print(t_output)
                ### final res
                p = f'''Summarize this to answer user question, be as concise as possible: {t_output}'''
                res = ollama.generate(model=llm, prompt=q+". "+p)["response"]
            else:
                print('🤬 >', f"\x1b[1;31m{t_name} -> NotFound\x1b[0m")

    if agent_res['message']['content'] != '':
        res = agent_res["message"]["content"]

    print("👽 >", f"\x1b1;30m{res}\x1b[0m")
    messages.append( {"role":"assistant", "content":res} )
```

现在，如果我们运行完整的代码，我们就可以与我们的代理聊天。

## **高级代理（编码**）

LLM 通过接触大量代码和自然语言文本的语料库来学习编码，它们学习编程语言的模式、语法和语义。模型通过预测序列中的下一个标记来学习代码不同部分之间的关系。简而言之，LLM 可以生成 Python 代码但不能执行它，代理可以。

我将准备一个工具，允许代理**执行代码**。在 Python 中，你可以轻松创建一个 shell，以字符串的形式运行代码，使用原生的命令*exec()*。

```py
import io
import contextlib

def code_exec(code: str) -> str:\
    output = io.StringIO()
    with contextlib.redirect_stdout(output):
        try:
            exec(code)
        except Exception as e:
            print(f"Error: {e}")
    return output.getvalue()

tool_code_exec = {'type':'function', 'function':{
  'name': 'code_exec',
  'description': 'execute python code',
  'parameters': {'type': 'object',
                'required': ['code'],
                'properties': {
                    'code': {'type':'str', 'description':'code to execute'},
}}}}

## test
code_exec("a=1+1; print(a)")
```

就像之前一样，我会写一个提示，但这次，在聊天循环的开始，我会要求用户提供文件路径。

```py
prompt = '''You are an expert data scientist, and you have tools to execute python code.
First of all, execute the following code exactly as it is: 'df=pd.read_csv(path); print(df.head())'
If you create a plot, ALWAYS add 'plt.show()' at the end.
'''
messages = [{"role":"system", "content":prompt}]
start = True

while True:
    ## user input
    try:
        if start is True:
            path = input('📁 Provide a CSV path >')
            q = "path = "+path
        else:
            q = input('🙂 >')
    except EOFError:
        break
    if q == "quit":
        break
    if q.strip() == "":
        continue

    messages.append( {"role":"user", "content":q} )
```

由于编码任务对于大型语言模型（LLMs）来说可能稍微复杂一些，因此我将添加**记忆强化**。默认情况下，在一次会话中，并没有真正的长期记忆。LLMs 可以访问聊天历史，因此它们可以暂时记住信息，并跟踪对话中之前给出的上下文和指令。然而，记忆并不总是按预期工作，尤其是如果 LLM 较小的话。因此，一个好的做法是通过在聊天历史中添加周期性提醒来强化模型的记忆。

```py
prompt = '''You are an expert data scientist, and you have tools to execute python code.
First of all, execute the following code exactly as it is: 'df=pd.read_csv(path); print(df.head())'
If you create a plot, ALWAYS add 'plt.show()' at the end.
'''
messages = [{"role":"system", "content":prompt}]
memory = '''Use the dataframe 'df'.'''
start = True

while True:
    ## user input
    try:
        if start is True:
            path = input('📁 Provide a CSV path >')
            q = "path = "+path
        else:
            q = input('🙂 >')
    except EOFError:
        break
    if q == "quit":
        break
    if q.strip() == "":
        continue

    ## memory
    if start is False:
        q = memory+"\n"+q
    messages.append( {"role":"user", "content":q} )
```

请注意，Ollama 中的默认记忆长度为 2048 个字符。如果你的机器可以处理，你可以在调用 LLM 时更改数字来增加它：

```py
    ## model
    agent_res = ollama.chat(
        model=llm,
        tools=[tool_code_exec],
        options={"num_ctx":2048},
        messages=messages)
```

在这个用例中，代理的输出主要是代码和数据，因此我不想让 LLM 重新阐述响应。

```py
    ## response
    dic_tools = {'code_exec':code_exec}

    if "tool_calls" in agent_res["message"].keys():
        for tool in agent_res["message"]["tool_calls"]:
            t_name, t_inputs = tool["function"]["name"], tool["function"]["arguments"]
            if f := dic_tools.get(t_name):
                ### calling tool
                print('🔧 >', f"\x1b[1;31m{t_name} -> Inputs: {t_inputs}\x1b[0m")
                messages.append( {"role":"user", "content":"use tool '"+t_name+"' with inputs: "+str(t_inputs)} )
                ### tool output
                t_output = f(**tool["function"]["arguments"])
                ### final res
                res = t_output
            else:
                print('🤬 >', f"\x1b[1;31m{t_name} -> NotFound\x1b[0m")

    if agent_res['message']['content'] != '':
        res = agent_res["message"]["content"]

    print("👽 >", f"\x1b1;30m{res}\x1b[0m")
    messages.append( {"role":"assistant", "content":res} )
    start = False
```

现在，如果我们运行完整的代码，我们就可以与我们的代理进行聊天。
