# 使用Python进行机器人学：Q-Learning vs Actor-Critic vs 进化算法

> 原文：[https://towardsdatascience.com/reinforcement-learning-for-robotics-complete-tutorial/](https://towardsdatascience.com/reinforcement-learning-for-robotics-complete-tutorial/)

## <mdspan datatext="el1762998038078" class="mdspan-comment">简介</mdspan>

有四种类型的机器学习：

+   **监督学习**—当数据集中的所有观测值都带有目标变量时，你可以执行[回归](https://en.wikipedia.org/wiki/Regression_analysis)/[分类](https://en.wikipedia.org/wiki/Statistical_classification)来学习如何预测它们。

+   **无监督学习**—当没有目标变量时，因此你可以执行[聚类](https://en.wikipedia.org/wiki/Cluster_analysis)来分割和分组数据。

+   **半监督学习**—当目标变量不完整时，因此模型必须学习如何预测未标记的数据。在这种情况下，使用监督和无监督模型的组合。

+   **强化学习**—当存在奖励而不是目标变量，并且你不知道最佳解决方案是什么时，因此它更多的是一个通过试错达到特定目标的过程。

更确切地说，[**强化学习**](https://en.wikipedia.org/wiki/Reinforcement_learning)研究AI如何在交互式环境中采取行动以最大化奖励。在监督训练期间，你已经知道正确答案（目标变量），并且你正在调整模型以复制它。相反，在强化学习问题中，你事先不知道正确答案是什么，唯一的方法是通过采取行动并获得反馈（奖励）来找出答案，因此模型通过探索和犯错误来学习。

强化学习正在被广泛用于**训练机器人**。一个很好的例子是自主吸尘器：当它经过地板的灰尘部分时，它会收到奖励（+1），但碰到墙壁时会受到惩罚（-1）。因此，机器人学习正确的动作是什么以及要避免什么。

在这篇文章中，我将展示如何使用不同的强化学习**算法**为训练机器人构建自定义3D环境。我将提供一些可以轻松应用于其他类似情况的实用Python代码（只需复制、粘贴、运行），并逐行带注释地解释代码，以便你可以复制这个示例。

## 设置

虽然监督用例需要一个目标变量和训练集，但强化学习问题需要：

+   **环境**—智能体的周围环境，它为动作分配奖励，并提供作为决策结果的新状态。基本上，它是AI可以与之交互的空间（在自主吸尘器的例子中将是清洁的房间）。

+   **动作**—AI在环境中可以执行的动作集合。动作空间可以是“**离散的**”（当有固定数量的移动时，如棋类游戏）或“**连续的**”（无限可能的状态，如驾驶汽车和交易）。

+   **奖励** — 行动的后果（+1/-1）。

+   **智能体**— 学习在环境中采取最佳行动以最大化奖励的AI。

关于环境，最常用的3D物理模拟器有：[***PyBullet***](https://pybullet.org/wordpress/)(初学者) ，[***Webots***](https://cyberbotics.com/)(中级)，[***MuJoCo***](https://mujoco.org/)(高级），和[***Gazebo***](https://gazebosim.org/home)（专业人士）。你可以使用它们中的任何一个作为独立软件，或者通过***[*Gym*](https://gymnasium.farama.org/index.html#)***，这是一个由OpenAI制作的库，用于开发基于不同物理引擎的强化学习算法。

我将使用 *Gym* (`pip install gymnasium`) 来加载一个用 *MuJoCo* 制作的[默认环境](https://gymnasium.farama.org/environments/mujoco/)(多关节动力学与接触，`pip install mujoco`)。

## 自定义环境

通常，环境具有[相同的属性](https://gymnasium.farama.org/introduction/create_custom_env/)：

1.  **重置** — 重新启动到初始状态或数据中的随机点。

1.  **渲染** — 可视化正在发生的事情。

1.  **步骤** — 执行智能体选择的动作并改变状态。

1.  **计算奖励** — 在执行动作后给予适当的奖励/惩罚。

1.  **获取信息** — 在执行动作后收集有关游戏的信息。

1.  **终止或截断**  — 在执行一个动作后决定场景是否结束（失败或成功）。

在 *Gym* 中加载默认环境很方便，但并不总是你所需要的。有时你必须构建一个符合你项目要求的自定义环境。这是强化学习用例中最微妙的步骤。模型的质量很大程度上取决于环境设计的好坏。

有几种方法可以创建自己的环境：

+   **从头开始创建**：你设计一切（即物理、身体、环境）。你拥有完全的控制权，但这是最复杂的方式，因为你从一个空的世界开始。

+   **修改现有的XML文件**：每个模拟智能体都是由一个[XML文件](https://en.wikipedia.org/wiki/XML)设计的。你可以编辑物理属性（例如，使机器人更高或更重），但逻辑保持不变。

+   **修改现有的Python类**：保持智能体和物理属性不变，但改变游戏的规则（即新的奖励和终止规则）。甚至可以将连续环境转换为离散动作空间。

## Q学习

[**Q学习**](https://en.wikipedia.org/wiki/Q-learning)是强化学习中最基本的形式，它使用Q值（“Q”代表“质量”）来表示某个动作在获得某些未来奖励时的有用性。简单来说，如果在游戏结束时，智能体在一系列动作后获得一定的奖励，那么初始Q值就是折现的未来奖励。

![图片](../Images/12909e47b0bd5cd9003a746eec9e5155.png)

随着智能体探索并接收反馈，它会更新存储在Q矩阵（[贝尔曼方程](https://en.wikipedia.org/wiki/Bellman_equation)）中的Q值。智能体的目标是学习每个状态/动作的最优Q值，以便它能够做出最佳决策并最大化特定状态下特定动作的预期未来奖励。

在学习过程中，智能体使用**探索-利用**权衡。最初，它通过采取随机动作来探索环境，从而积累经验（关于不同动作和状态相关奖励的信息）。随着学习的进行和探索水平的降低，它开始通过选择每个状态的最高Q值动作来利用其知识。

请注意，Q矩阵可以是多维的，并且要复杂得多。例如，让我们考虑一个交易算法：

![图片](../Images/ea8febb1c0b2daebd5ffc492e4a08ff7.png)

2013年，当谷歌推出了**[深度Q网络（DQN）](https://arxiv.org/abs/1312.5602)**，这是一个突破性的强化学习领域的进展，它旨在从原始像素中学习玩Atari游戏，结合了深度学习和Q学习的两个概念。简单来说，深度学习用于近似Q值，而不是在表中显式存储它们。这是通过一个训练有素的神经网络来完成的，该网络预测每个可能动作的Q值，并将环境的当前状态作为输入。

![图片](../Images/4fd297b8aac83a633fc7ff6ab2c3d554.png)

Q-Learning家族主要设计用于离散环境，因此在机器人Ant上并不真正适用。一个替代方案是对**环境进行离散化**（即使这不是解决连续问题的最有效方法）。我们只需为期望离散动作（例如，“向前移动”）的Python类创建一个包装器，然后根据该命令对关节施加力。

## 行为者-评论家

在实践中，[**演员-评论家**](https://en.wikipedia.org/wiki/Actor-critic_algorithm)算法是最常用的，因为它们非常适合连续环境。基本思想是让两个系统协同工作：一个策略函数（“演员”）用于选择动作，一个价值函数（“评论家”）用于估计期望奖励。模型通过比较它收到的实际奖励与预测来学习如何调整决策。

第一款稳定的深度学习算法是由OpenAI在2016年引入的：**[优势演员-评论家（A2C）](https://arxiv.org/abs/1602.01783)**。它的目标是最小化演员采取行动后实际收到的奖励与评论家估计的奖励之间的损失。神经网络由演员和评论家共享的输入层组成，但它们返回两个不同的输出：动作的Q值（就像DQN一样），以及预测的奖励（这是A2C的加和）。

![](../Images/a3fc06f5f65f526d50a70046339fae9a.png)

几年来，AC算法随着更多稳定和高效的变体（如**[近端策略优化（PPO）](https://en.wikipedia.org/wiki/Proximal_policy_optimization)**）和[**软演员评论家（SAC）**](https://spinningup.openai.com/en/latest/algorithms/sac.html)得到了改进。后者使用两个评论家网络来获得“第二意见”。记住，我们可以在连续环境中直接使用这些模型。

由于Q学习和演员-评论家的流行，最近出现了更多结合两种方法的混合适应。这样，它们也将DQN扩展到连续动作空间。例如，**[深度确定性策略梯度（DDPG）](https://arxiv.org/abs/1509.02971)**和**[双延迟DDPG（TD3）](https://arxiv.org/pdf/1802.09477)**。但是，请注意，模型越复杂，训练越困难。

## 实验模型

除了主要家族（Q和AC）之外，你还可以找到一些在实践中使用较少但同样有趣的模型。特别是，它们可以作为奖励稀疏且难以设计的任务的强大替代品。例如：

+   **[进化算法](https://en.wikipedia.org/wiki/Evolutionary_algorithm)** 通过变异和选择而不是梯度来进化策略。受达尔文进化论的启发，它们很稳健，但计算量很大。

+   **[模仿学习](https://en.wikipedia.org/wiki/Imitation_learning)** 跳过探索，并训练智能体模仿专家演示。它基于“行为克隆”的概念，将监督学习与强化学习思想相结合。

为了实验目的，让我们尝试第一个使用*[EvoTorch](https://evotorch.ai/)*，这是一个用于神经演化的开源工具包。我选择它是因为它与*PyTorch*和*Gym*配合得很好（`pip install evotorch`）。

最佳的强化学习进化算法是**[带有参数探索的政策梯度（PGPE）](https://docs.evotorch.ai/latest/reference/evotorch/algorithms/functional/funcpgpe/#evotorch.algorithms.functional.funcpgpe.pgpe)**。本质上，它不是直接训练一个神经网络，而是构建一个覆盖所有可能权重的概率分布（Gaussian）（μ=平均权重集合，σ=围绕中心的探索）。在每一代中，PGPE从权重种群中采样，从一个随机的策略开始。然后，模型根据奖励（种群进化）调整均值和方差。PGPE被认为是**并行化强化学习**，因为与经典方法如Q和AC不同，这些方法使用样本批次更新一个策略，而PGPE并行采样许多策略变体。
