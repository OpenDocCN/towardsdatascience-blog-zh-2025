## <mdspan datatext="el1755663265451" class="mdspan-comment">介绍</mdspan>

理解如何使用 spaCy 规则来识别文本中的模式非常重要。有一些实体，如时间、日期、IBAN 和电子邮件，遵循严格的格式，因此可以使用确定性规则来识别它们，例如，使用 **正则表达式**（**regexes**）。

SpaCy 通过使它们更易于阅读来简化正则表达式的使用，因此您将使用 Matcher 类的实际描述来代替奇怪的符号。

### 基于标记的匹配

在这个例子中，正则表达式模式捕获了所有前面的字符串。这个模式表示“Marcello”可以可选地后面跟着单词“Danilo”（因为我们使用了符号“？”）。此外，符号“\s”表示在单词之间使用空格、制表符或多个空格都可以。

正则表达式的问题，以及为什么许多程序员不喜欢它们的原因，是它们难以阅读。这就是为什么 spaCy 提供了一个干净且适用于生产的替代方案，即 **Matcher 类**。

spaCy 支持的语法非常庞大。在这里，我报告了一些最常见的模式。

### 文本属性

| 属性 | 描述 | 示例 |
| --- | --- | --- |
| `"ORTH"` | 精确的文本 | `{"ORTH": "Hello"}` |
| `"LOWER"` | 单词的小写形式 | `{"LOWER": "hello"}` |
| `"TEXT"` | 与 `"ORTH"` 相同 | `{"TEXT": "World"}` |
| `"LEMMA"` | 单词的词干（基本形式） | `{"LEMMA": "run"}` |
| `"SHAPE"` | 单词的形状（例如，`Xxxx`，`dd`） | `{"SHAPE": "Xxxx"}` |
| `"PREFIX"` | 标记的第一个字符（例如，“un”） | `{"PREFIX": "un"}` |
| `"SUFFIX"` | 标记的最后字符（例如，“ing”） | `{"SUFFIX": "ing"}` |

### 语言学特征

| 属性 | 描述 | 示例 |
| --- | --- | --- |
| `"POS"` | 通用词性标签 | `{"POS": "NOUN"}` |
| `"TAG"` | 详细词性标签 | `{"TAG": "NN"}` |
| `"DEP"` | 语法依存关系 | `{"DEP": "nsubj"}` |
| `"ENT_TYPE"` | 命名实体类型 | `{"ENT_TYPE": "PERSON"}` |

### 布尔标志

| 属性 | 描述 | 示例 |
| --- | --- | --- |
| `"IS_ALPHA"` | 标记由字母字符组成 | `{"IS_ALPHA": True}` |
| `"IS_ASCII"` | 标记由ASCII字符组成 | `{"IS_ASCII": True}` |
| `"IS_DIGIT"` | 标记是数字 | `{"IS_DIGIT": True}` |
| `"IS_LOWER"` | 标记是小写字母 | `{"IS_LOWER": True}` |
| `"IS_UPPER"` | 标记是大写字母 | `{"IS_UPPER": True}` |
| `"IS_TITLE"` | 标记是大写标题 | `{"IS_TITLE": True}` |
| `"IS_PUNCT"` | 标点符号标记 | `{"IS_PUNCT": True}` |
| `"IS_SPACE"` | 标记是空白字符 | `{"IS_SPACE": True}` |
| `"IS_STOP"` | 标记是停用词 | `{"IS_STOP": True}` |
| `"LIKE_NUM"` | 标记看起来像数字 | `{"LIKE_NUM": True}` |
| `"LIKE_EMAIL"` | 标记看起来像电子邮件地址 | `{"LIKE_EMAIL": True}` |
| `"LIKE_URL"` | 标记看起来像URL | `{"LIKE_URL": True}` |
| `"IS_SENT_START"` | 标记在句子开头 | `{"IS_SENT_START": True}` |

### 操作符

用于重复或使图案可选：

| 操作符 | 描述 | 示例 |
| --- | --- | --- |
| `"OP"` | 图案操作符： |  |
|  | `"?"` – 零个或一个 | `{"LOWER": "is", "OP": "?"}` |
|  | `"*"` – 零个或多个 | `{"IS_DIGIT": True, "OP": "*"}` |
|  | `"+"` – 一个或多个 | `{"IS_ALPHA": True, "OP": "+"}` |

示例：

匹配如下字符串的图案是什么：*“我有2个红苹果”，“我们买了5个绿色的香蕉”，或者“他们找到了3个成熟的橙子”*？

**图案要求：**

+   主语代词（例如，“I”，“we”，“they”）

+   动词（例如，“have”，“bought”，“found”）

+   一个数字（数字或书写，如“2”，“five”）

+   可选的形容词（例如，“red”，“ripe”）

+   复数名词（例如，水果）

## 最后的想法

我希望这篇文章能帮助你看到在NLP中我们可以做多少事情，而不必总是使用巨大的模型。很多时候，我们只需要找到遵循规则的事物——比如日期、IBAN、名字或问候语——而为此，spaCy为我们提供了像`Matcher`和`PhraseMatcher`这样的强大工具。

在我看来，使用这样的模式工作是一种更好地理解文本结构的好方法。此外，当你不想在简单的事情上浪费资源时，它使你的工作更加高效。

我仍然认为正则表达式非常强大，但有时难以阅读和调试。使用spaCy，事情看起来更清晰，更容易在实际项目中维护。
