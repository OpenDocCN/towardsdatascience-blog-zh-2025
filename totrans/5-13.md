指导让我学到了很多。

我最近有机会指导一位工作了两年、希望转向数据科学（DS）角色的软件工程师（SE）。最初只是一次闲聊，最终变成了几个小时的时间，规划成为数据科学家的计划。

她的第一个问题是，“*我应该学习什么新技能？*”

当然，我可以在一分钟内列出十几件事，但这需要比技能列表和流行课程的链接更多。顺便说一句，我从未在这个帖子中回答过这个问题。

她的一些现有技能对她新的努力非常有价值。她可以通过仔细学习其中之一来快速过渡。但更重要的是，要像数据科学家一样思考。

她不必忘记任何东西。然而，她的一些 SE 技能对数据科学来说用处不大。

在这篇文章中，我总结了我们的讨论。这包括哪个数据科学领域适合她的兴趣，她需要获取哪些新技能，以及如何从小处着手快速成长。

## 数据科学家与软件工程师有何不同之处？

这并不是说某人的工作比其他人容易。然而，数据科学家在责任方面的目标与软件工程师有显著不同。

软件工程师（SE）关注设计、开发和维护软件。大多数情况下，SE 的工作更具确定性。换句话说，他们知道他们要构建的结果。通常有一组有限的技巧来实现这些目标。

SE 在工作中将不得不做出许多选择，有时将不得不进行课程调整。但这些不确定性通常有可预测的解决方案。

另一方面，数据科学家的工作充满了更多未知。他们无法保证他们拥有的数据质量良好。DS 将不得不处理这些问题。没有人能告诉哪个机器学习模型最适合当前的问题；这更多是试错。哪个评估指标比其他指标更重要？它们的可接受阈值是多少？这些都是数据科学家在行动中必须解决的问题。

由于这些原因，数据科学家有一个更实验性的思维模式。作为一名专业 SE，你通常会在一个像 SCRUM 这样的框架中工作，它期望你在严格的时间框架内完成工作，这个时间框架被称为冲刺。

## 数据科学家是数据科学专业人士的一种。

虽然我们互换使用这些名称，但并非所有数据科学专业人士都是数据科学家。我们还有其他几个与数据科学家相关的角色。

> 我们很少看到有“全栈数据科学家”这样的职位，尽管全栈软件开发者的职位非常普遍。

大多数被雇佣为数据科学家的人要么是分析师、机器学习工程师，要么是数据工程师。

数据科学中最常见的角色是**分析师**。分析师在商业决策中扮演关键角色。他们从他们拥有的数据中提取关键见解，并教育管理层。分析师也不一定是程序员。我所认识的多数分析师只使用 Excel。

> [**这是我在 Python 中创建令人惊叹仪表板的方法。**](https://towardsdatascience.com/plotly-dashboards-in-python-28a3bb83702c)

**机器学习工程师**的核心职责是训练解决商业问题的模型。他们从事数据预处理、模型选择和训练工作。正如安德鲁·吴（Andrew Ng）所指出的，工业环境中的数据科学家（或与机器学习工程师紧密协作的）并不试图发明新的模型。相反，他们试图找到最佳的数据和预处理技术来解决问题。

> [**在生产中部署机器学习模型的 3 种方法**](https://towardsdatascience.com/3-ways-to-deploy-machine-learning-models-in-production-cdba15b00e)

另一方面，**数据工程师**负责创建和维护数据和基础设施。这包括创建数据管道和仓库以及管理数据库。

数据工程师和机器学习工程师经常携手合作。在某些任务上，如模型部署，他们共同工作。

> [**在 8 个关键 MLOps 角色中，你适合哪一个？**](https://towardsdatascience.com/in-the-8-key-mlops-roles-where-do-you-fit-in-251272ca5488)

想要成为数据科学家的人通常需要选择一个。每个都有其挑战和解决问题的工具。例如，虽然机器学习工程师专注于 Scikit-learn、Tensorflow 和 Pandas 等工具，数据工程师则专注于 Airflow、SQL 和云基础设施管理。

现在已经很清楚，不同数据科学家使用的科技相当不同，需要投入时间来掌握它们。因此，我们很少看到有“全栈数据科学家”这样的职位，尽管全栈软件开发者的职位非常普遍。

## 如何选择适合你的数据科学角色

我已经提到过这一点。我不建议任何人试图掌握所有不同的数据科学角色，尽管我的职业生涯有点像厨房水槽。

最好的办法是选择一个适合你个性的角色，并尽早选择。

这就是我的指南（当然，带有个人观点。）

如果你是一个喜欢较少编程和较少技术工作的人，你将更多地关注协助商业决策。尽管如此，如果你想在数据相关角色中工作，你也会很乐意成为一名**数据分析师**。

了解一些 Python 中的数据处理技术是有帮助的，但不是必需的。我曾有同事不断挑战我，说他们可以用 Excel 做这些事情，而我使用 Python。猜猜看，有时候他们会赢。

然而，你应该与你的 HR 讨论如何衡量你的表现，因为从数据中获得的洞察力没有任何实际价值。它们需要被采取行动，即使如此，看到它们的成功也会有所延迟。因此，传统的评估技术不适用于你。

你是一个不关心数字、洞察力或模型的人吗？你认为自己更像软件工程师而不是数据科学家？但你仍然想成为数据科学家吗？考虑**数据工程师**的角色。

你仍然会进行大量的编码和数据库工作。

最后，如果你喜欢训练模型、评估它们、执行数据预处理、构建管道等，那么**机器学习工程师**的角色比其他角色更适合你。在工业环境中，机器学习工程师不会创建新的模型。相反，他们使用数据和模型选择来找到满足要求的最佳解决方案。

## 作为数据科学家很少会使用的技能（如果你之前是软件工程师）

作为一名前软件工程师和数据科学家，我知道一些现在几乎无用的东西。以下是一些。

### 设计模式和原则

软件设计模式区分了熟练的 SE（软件工程师）和业余爱好者。这些最佳实践使我们能够开发易于维护、扩展和重用组件的软件。由于其他开发者广泛认可模式，这有助于新人快速理解和与你合作。

设计原则指导更好的编码并有助于维护代码库。我在开发软件时严格遵循[SOLID 设计原则](https://www.freecodecamp.org/news/solid-design-principles-in-software-development/)。

同样，设计模式就像你可以借用的模板，用来解决问题而不会浪费时间。

我不能说这些完全无用。如果你使用过像 Pandas、scikit-learn（或任何其他东西），它们都使用了这些模式和原则。

但作为一个主要关注从你刚刚收到的数据集中获取洞察力或开发执行一系列指令的管道的人，你很少需要它们。

### 网络开发框架（如 Django）

我曾是 Django 开发者。我喜欢做这件事。

你可能会问，“如果一个数据科学家想将他们的工作打包成 Web 应用？他们不需要 Django 吗？”

他们确实在过去这样做过。

然而，今天，数据科学家的需求已经被 Streemlit 或 BI 工具（如 Tableau 和 PowerBI）等工具所取代。你不必自己编写所有代码。

> [**6 个用于创建桌面、Web 甚至移动应用的 Python GUI 框架**](https://towardsdatascience.com/5-python-gui-frameworks-to-create-desktop-web-and-even-mobile-apps-c25f1bcfb561)

我最近使用 Django 为 Streamlit 的认证问题创建了一个解决方案。我还记录了在两篇之前的帖子中的方法（[认证](https://towardsdatascience.com/secure-your-streamlit-app-with-django-bb0bee2a6519)和[授权](https://towardsdatascience.com/streamlit-access-control-dae3ab8b7888))。当时 Streamlit 还没有这些功能。但今天，这些建议已经过时了。

### 敏捷项目管理（如 Scrum）

在过渡期间，这是我最大的挑战。作为每两周就要推出新功能的软件工程师，SCRUM 这样的系统非常有帮助。

但与数据一起工作时这并不容易——尤其是当客户拥有数据时。

大多数时候，我们做的工作是实验性的。不能保证你能在冲刺的时间内找到完美的机器学习模型。每次你看到新的数据，你都会看到新的挑战，这使得时间估计变得很棘手。

即使分析师也无法提前说他们在两周内会提取出某些数字或洞察。

我记得在我作为数据分析师的早期，一位经理说过的话。“折磨数据，直到它开始尖叫洞察。”而你不知道这会发生什么时候。

我随后询问了经验丰富的数据科学家和 SCRUM 专家如何在数据科学项目中实施这些原则。他们不知道。

## 作为数据科学家可能需要学习的新技能

最后，我们现在可以讨论这篇帖子的目的了——一个想要成为数据科学家的软件工程师必须学习哪些技能。

这是我们最后达成的非官方共识。

我相信 80/20 法则。这意味着有一小部分技能（20%）对你的职业（80%）影响最大。

> [**这 5 种 SQL 技术涵盖了 80%的实际项目**](https://towardsdatascience.com/5-advanced-sql-techniques-for-real-life-projects-f2db9b6680e2)

你的任务是自我评估你的相关技能，并发现你需要学习的 20%的新技能，以帮助你达到未来的自己。

我的这位朋友，一个 SE 和 Django 开发者，已经非常熟悉**Python**。她也擅长**SQL**。仅 Python 和 SQL 就足以让她成为一名优秀的数据分析师。

然而，她认为机器学习比从数据集中提取洞察更有趣。这引发了许多问题：我们应该使用传统的机器学习还是深度学习模型？我们应该使用模型拟合、计算机视觉还是 NLP？

这里是我建议的。

无论发生什么，所有数据科学家都在某种程度上使用**Pandas**。你可以在几个小时之内开始使用它（就像任何其他库一样）。但作为一个数据科学家，你应该在 Pandas 上有所精通。因此，这是第一个。

我没有建议任何关于学习 Pandas 的课程或资源；她应该能够通过搜索几分钟找到一些好的资源。

接下来是**scikit-learn**。同样，很多数据科学家都在使用它。即使你在做深度学习项目，你仍然会使用 scikit-learn 的一些模块。这是值得学习的。

Scikit Learn 不仅仅是一个用于生成机器学习模型的库。你可以进行数据预处理、模型选择等操作。

然后，一点 NLP。我建议使用**TextBlob**库。它是 NLTK 库的包装器，NLTK 是 Python 中 NLP 库的圣杯。

最后，我建议她掌握**Plotly**或**Streamlit**中的一个。Streamlit 更容易学习，而且许多其他数据科学家也在使用它。然而，鉴于她在 SE 领域的经验，像 Plotly 这样的工具仍然触手可及。

Python、SQL、Pandas、Scikit-learn、Streamlit 和 TextBlog 是值得考虑的顶级技能，它们对她的职业生涯有 80%的影响。

任何人都可以争论这些技能对于当今竞争激烈的市场中的数据科学家来说并不足够。我同意。但有了这些，她可以找出下一步该去哪里。

如果是计算机视觉，她会选择像 OpenCV 和 PyTourch 这样的工具。如果她在 NLP 方面有所进步，她会选择像 SpaCy 这样的库。

## 最后的想法

我们中的许多人想要改变职业。很少有人能将梦想中的工作作为他们的第一份工作。

在这篇文章中，我总结了我和一位 SE 朋友讨论的内容，他是一位想要成为数据科学家的人。

我们发现她更想成为一名机器学习工程师，而不是分析师或数据工程师。既然她已经掌握了 Python 和 SQL，学习她需要掌握的内容就变得容易了。

我们发现，任何想要成为机器学习工程师的人都可以专注于 Python、SQL、Pandas、Scikit-learn、Streamlit 和 TextBlog，因为这些是大多数数据科学家生活中最重要的技能。
