所以，这是今天帖子的水壶旁观点：

> 描述性统计完全匹配，所以数据集基本上是相同的。无需进一步挖掘。

当然，如果你从未上过统计学课程，这可能有些道理。🙃 但在现实中，非常不同的数据集可以具有相同的描述性统计指标，例如均值、平均值或标准差。换句话说，尽管描述性统计 *描述* 数据集，但它们并不 *定义* 它。为了获得完整的情况，人们需要始终 *绘制* 数据。

无论如何，第一个展示这一点的人是 [Frank Anscombe](https://en.wikipedia.org/wiki/Frank_Anscombe)，他用他现在臭名昭著的 [Anscombe 的四重奏](https://en.wikipedia.org/wiki/Anscombe%27s_quartet)。

## 那么，Anscombe 的四重奏是什么？

因此，[安斯康姆的四重奏](https://en.wikipedia.org/wiki/Anscombe%27s_quartet)是一组具有几乎相同描述性统计的四个数据集，但可视化时却看起来非常不同。该数据集属于公共领域，并且方便地可在 seaborn 库中找到，允许我们进行实验和探索正在发生的事情。我们可以轻松地通过以下方式加载数据集：

```py
import seaborn as sns
data = sns.load_dataset("anscombe")
```

然后，我们可以使用 matplotlib 创建四个数据集的可视化：

```py
import matplotlib.pyplot as plt

sns.relplot(
    data=data,
    x="x", y="y",
    col="dataset", hue="dataset",
    kind="scatter",
    palette="deep",
    height=4, aspect=1
)
plt.suptitle("Anscombe's Quartet", y=1.05)
plt.show()
```

特别是，所有四个数据集都由 11 个(x, y)点组成：

+   数据集 I 似乎是一个简单的线性关系

+   数据集 II 显然是一个抛物线曲线

+   数据集 III 显然是一个线性关系，但有一个大的异常值

+   并且数据集 IV 是一条垂直线，但同样，被一个大的异常值扭曲

所有四个数据集彼此之间都非常不同，然而，当我们计算它们的描述性统计时，我们遇到了一个剧情转折——所有四个数据集几乎具有相同的描述性统计。更具体地说，我们可以按照以下方式计算描述性统计：

```py
import pandas as pd
from scipy.stats import linregress

def calculate_statistics(group):
    mean_x = group['x'].mean()
    var_x = group['x'].var()
    mean_y = group['y'].mean()
    var_y = group['y'].var()
    correlation = group[['x', 'y']].corr().iloc[0, 1]
    slope, intercept, r_value, p_value, std_err = linregress(group['x'], group['y'])
    r_squared = r_value ** 2

    return pd.Series({
        "Mean of x": mean_x,
        "Variance of x": var_x,
        "Mean of y": mean_y,
        "Variance of y": var_y,
        "Correlation of x and y": correlation,
        "Linear regression line": f"y = {intercept:.2f} + {slope:.2f}x",
        "R2 of regression line": r_squared
    })

statistics = data.groupby("dataset").apply(calculate_statistics)

for dataset, stats in statistics.iterrows():
    print(f"Dataset: {dataset}")
    for key, value in stats.items():
        print(f"  {key}: {value}")
    print()
```

数字相同，视觉效果却截然不同。🤷‍♀️ 疯狂，对吧？这就是为什么无论数字如何，始终可视化数据都如此重要的原因。用安斯康姆自己的话说，统计学家中的一种普遍但错误的信念是“*数值计算是精确的，但图表是粗糙的”*。

安斯康姆的四重奏是强调数据可视化重要性的一个强大例子，因为可视化不仅仅是不同的，而是**明显不同**——只需一眼，就可以理解数据集彼此之间完全不同。换句话说，可视化立即为我们提供了描述性统计无法包含的有意义信息。

* * *

没有人确切知道安斯康姆最初是如何想出这些数据集的，但[这项](https://www.research.autodesk.com/app/uploads/2023/03/same-stats-different-graphs.pdf_rec2hRjLLGgM7Cn2T.pdf) 2017 年的研究提出了一种从头创建此类数据集的方法。这允许我们产生几乎具有相同描述性统计的非常不同的数据集的无限例子——我最喜欢的是[数据龙十二生肖](https://en.wikipedia.org/wiki/Datasaurus_dozen) 🦖。与安斯康姆的四重奏类似，数据龙十二生肖包括十三个——一个十二生肖加上数据龙——具有几乎相同描述性统计的非常不同的数据集。

数据集可在[datasauRus](https://jumpingrivers.github.io/datasauRus/) R 包下以[MIT 许可证](https://github.com/jumpingrivers/datasauRus/blob/v0.1.9/LICENSE.md)获得，该许可证允许商业用途。

再次强调，所有这些数据集几乎都有几乎相同的描述性统计，然而，它们却截然不同。显然，跳过可视化步骤，会导致信息损失惨重。

## 当一个图表可能有所帮助时

安斯康姆四重奏或 Datasaurus 可能看起来像是有趣的、简单的例子，旨在教导我们关于数据可视化的重要性。但，不要被误导；这个教训——数据可视化的重要性——不是一个理论概念，而是一个非常真实的概念，在现实世界中有着实际的含义。

### 1. 2008 年金融危机

以 2008 年的金融危机为例。当时银行和投资公司使用的模型，高度依赖像 _[风险价值（VaR）](https://en.wikipedia.org/wiki/Value_at_risk)这样的聚合风险指标。*更具体地说*，VaR 提供了一个在指定置信水平（例如 95%或 99%）下，给定时间段内投资组合可能的最大损失估计。例如，99%的*VaR*为 100 美元意味着，在正常市场条件下，损失超过 100 美元的可能性只有 1%。

*VaR*的计算基于一系列假设，例如通常使用的假设是市场回报是正态分布的。然而，正如你可能已经意识到的，‘*正态分布*’通常是对现实世界的一种相当粗略的解释——现实生活大多数情况下要复杂得多，并不那么直接。

无论如何，在 2008 年金融危机的情况下，假设的*“正态分布”*未能代表现实，也无法解释市场回报实际数据中出现的*“厚尾”*现象。*厚尾*在分布中代表的是极端事件（在这里是极端损失）发生的概率高于预期（从正态分布中）的情况。换句话说，极端损失发生的频率远高于所使用的模型所假设的。

![由作者在 Python 中使用 scipy.stats 生成数据创建的图像](img/adaab9cab0ecf9f7a1d8252d130a0b64.png)

由作者在 Python 中使用`scipy.stats`生成数据创建的图像

尽管如此，专注于像*VaR*这样的综合指标并不能识别那些洞察。相反，对历史回报的详细可视化可以更深入地了解真正发生的事情。使用可视化工具和详细分析的个体和机构可以识别风险。

最终，*VaR*是一个很好的例子，说明了人们往往在看到数字——一个计算结果——时，无论这个数字是否凭空想象，都会感到自信和安心。在 2008 年金融危机的核心，过度依赖量化模型导致机构大大低估了风险。显然，许多其他因素也导致了这一结果——比如相互关联的投资组合的系统风险——然而，对综合数字的盲目信任仍然是一个强大的因素。

### 2. 挑战者号航天飞机灾难

即使在[挑战者号航天飞机灾难](https://en.wikipedia.org/wiki/Space_Shuttle_Challenger_disaster)中，一些数据可视化也可能有所帮助。例如，1986 年 1 月 28 日，挑战者号航天飞机在起飞后 73 秒因助推器部分 O 形环故障而解体。O 形环的故障是由于极端低温（2°C / 35°F，前一天晚上降至-8°C/17°F）。在发射前的一段时间里，[莫顿·塞科尔](https://en.wikipedia.org/wiki/Thiokol)——制造 O 形环的 NASA 分包商——的工程师们对 O 形环在低温下的性能表示担忧，甚至明确建议不要发射。根据他们提交的事故报告，他们向公司的管理层展示了以下图片中的相关数据。尽管如此，他们的担忧被公司管理层忽视，最终建议 NASA 可以发射。

![O 形环损坏历史和不同固体火箭发动机（SRM）的温度，来源：https://www.nasa.gov/history/rogersrep/v5p896.htm，公有领域](img/3d3d5a6debda011025566043404300e4.png)

O 形环损坏历史和不同固体火箭发动机（SRM）的温度，来源：[`www.nasa.gov/history/rogersrep/v5p896.htm`](https://www.nasa.gov/history/rogersrep/v5p896.htm)，公有领域

与温度相关的 O 型圈故障的说明包含大量数据，然而，说明的重点并没有清楚地传达。爱德华·图费在他的书*[《视觉解释》](https://www.edwardtufte.com/book/visual-explanations-images-and-quantities-evidence-and-narrative/)*中认为，一个简单、清晰的温度与 O 型圈故障之间相关性的图形表示可以走很长的路，甚至可能导致关于批准发射的不同决策。据说，莫顿·塞科尔也向 NASA 展示了与温度相关的 O 型圈故障事件的散点图——但图中缺少没有故障的航班，所以没有太多意义。一个更好的图表，也包括没有 O 型圈事件航班，可以在‘_[总统委员会关于挑战者号航天飞机事故的报告](https://sma.nasa.gov/SignificantIncidents/assets/rogers_commission_report.pdf)._’中找到。显然，这是一个更有用的图表，使人们可以立即怀疑 O 型圈故障与温度之间的相关性。图费的批评强调了这种遗漏如何导致决策不当。

尽管如此，鉴于今天与那些事件的历史和技术距离，很容易在过去的案例中找到错误。然而，意识到在许多情况下，灾难***可以预见，但无法有效地传达给决策者，以便做出正确的决策***，这让人心碎。

### 3. 1854 年布罗德街霍乱疫情

有效数据可视化的一个显著用例是[1854 年布罗德街霍乱疫情](https://en.wikipedia.org/wiki/1854_Broad_Street_cholera_outbreak#cite_note-23)的案例。特别是，这次疫情发生在伦敦的索霍区，靠近布罗德街，在[1846-1860 年霍乱大流行](https://en.wikipedia.org/wiki/1846%E2%80%931860_cholera_pandemic)期间，导致 616 人死亡。

在这次疫情中，约翰·斯诺博士设法在地图上展示了霍乱病例，并意识到它们围绕着布罗德街的一个水泵聚集。这导致了[确定布罗德街的水泵被污染](https://pmc.ncbi.nlm.nih.gov/articles/PMC7150208/)，实际上是疫情的根本来源。仅仅移除水泵的手柄就导致了疫情的结束。

![约翰·斯诺博士的霍乱地图，来源：https://commons.wikimedia.org/wiki/File:Snow-cholera-map-1.jpg?uselang=en#Licensing, 公共领域](img/a6d6afed5dbbcf44145b39c64ce5c295.png)

约翰·斯诺博士的霍乱地图，来源：[`commons.wikimedia.org/wiki/File:Snow-cholera-map-1.jpg?uselang=en#Licensing`](https://commons.wikimedia.org/wiki/File:Snow-cholera-map-1.jpg?uselang=en#Licensing)，公共领域

这是一个巨大的发现，不仅阻止了疫情，还彻底改变了人们对疾病传播和流行病学的理解。毫无疑问，这是一个很好的例子，说明了简单的可视化可以走得很远，并揭示有影响力的洞察力。

## 在我心目中

最终，从安斯康姆四重奏或数据恐龙十二生肖等数据集中得到的教训是，我们根深蒂固的观念“‘数值计算是精确的，但图表是粗糙的’”是错误的。可视化和数值计算都是提取数据中有意义洞察力的必要手段。最终，以有意义的方式解释数据可能更多的是一种艺术，而不是精确的科学，因为没有一种*单一、通用的*计算方法应该遵循。数据可视化不仅仅是漂亮的图片，它是避免数据分析中误解和不良决策的必要手段… 

…因为可能某个地方隐藏着一只数据恐龙。
