## 企业真的应该倾听客户的声音吗？

在一个快速发展的世界中，每一刻都变得更加以人工智能驱动，企业现在需要不断寻求竞争优势以保持可持续性。公司可以通过定期观察和分析客户对其产品和服务意见来做到这一点。他们通过评估来自许多来源的评论来实现这一点，包括在线和离线。识别客户反馈中的正面和负面趋势，使他们能够微调产品特性，并设计满足客户需求的营销策略。

因此，需要适当地辨别客户意见，以找到有价值的见解，这有助于做出明智的商业决策。

## 熟悉情感分析

情感分析，作为自然语言处理（NLP）的一部分，是一种流行的技术，因为它研究任何给定文本中人们的意见、情感和情绪。通过将情感分析应用于他们收集到的包含有价值信息的反馈，企业可以理解公众舆论，监控品牌声誉，并改善客户体验。然而，其非结构化的性质可能会使其难以分析。通过定期分析客户情感，公司可以识别其优势和劣势，决定如何提升产品开发，并构建更好的营销策略。

Python 和 R 中的强大情感分析包使企业能够揭示有价值模式，跟踪情感趋势，并做出数据驱动的决策。在本文中，我们将探讨如何使用不同的包（**[Quanteda](https://cran.r-project.org/web/packages/quanteda/readme/README.html)**，[Sentimentr](https://cran.r-project.org/web/packages/sentimentr/index.html)** 和 **[Textstem](https://cran.r-project.org/web/packages/textstem/index.html)**）通过处理、分析和可视化文本数据来对客户反馈进行情感分析。

## 2. 预处理文本数据

在进行文本分析之前，我们需要对文本进行预处理，以确保格式干净且一致。预处理将涉及几个关键步骤：

1.  **文本清理**，包括移除标点符号、数字和特殊字符；

1.  **文本归一化**，这包括将字母转换为小写；

1.  **分词文本**，这包括将文本分割成单个单词或标记；

1.  **移除停用词**，这包括有意移除对情感分析不贡献的单词（例如，“the”、“and”）；最后，

1.  **词干提取或词形还原**，其中单词被还原到其基本形式。这些步骤有助于减少噪声并提高分析的准确性。

现在，我们将对数据集实施上述预处理步骤。

```py
# Cleaning the dataset
corpus <- quanteda::corpus(data$text)
tokens_clean <- quanteda::tokens(corpus, remove_punct = TRUE, remove_numbers = TRUE, remove_symbols = TRUE) %>%
tokens_tolower() %>%
tokens_remove(stopwords("en"))
# Convert tokens to character vectors for lemmatization
tokens_char <- sapply(tokens_clean, function(tokens) paste(tokens, collapse = " "))
# Lemmatize tokens using WordNet
lemmatized_texts <- lemmatize_strings(tokens_char)
```

在此代码中，我们将数据集的文本列转换为 quanteda 语料库对象。我们通过分词来清理文本，这包括去除标点符号、数字和符号，将所有单词转换为小写，并过滤掉常见的停用词。最后，应用词干提取以将单词还原到其基本形式，例如将"running"改为"run"，以确保文本分析的一致性。在此需要注意的是，我们没有使用词干提取，因为它可能会由于将单词简化为基本形式而导致部分或不完整的单词提取。从另一方面来说，它应用简单的规则来截断单词的末尾。例如，算法可能会从"amazing"中移除后缀"-ing"，得到"amaz"，或者"terrible"可能变成"terribl"。为了避免这种情况并获得更准确的根形式，我们将使用词元化，这是一个更复杂的过程，它依赖于词典来映射单词，并考虑单词的上下文和词性，以返回它们的基形式或词典形式。

现在我们已经清理并分词了文本数据，我们可以进行下一步。我们的目标是分析反馈条目中的情感。我们将使用 sentimentr 包来评估结构化数据中的情感，从而深入了解反馈条目的情感基调。

## 3. 使用 Sentimentr 包进行情感分析

现在，我们可以使用来自`sentimentr`包的`sentiment`函数对这些句子进行情感分析。此函数为每段文本计算情感分数，识别出积极和消极的词汇。

接下来，我们总结每个文档的情感分数。我们将分数按文档分组，计算积极和消极词汇的总数。我们还计算一个复合分数，并将整体情感分为积极或消极。

```py
# Perform sentiment analysis using sentimentr
sentiment_scores <- sentiment(lemmatized_texts)
# Summarize sentiment scores for each document
sentiment_summary <- sentiment_scores %>%
group_by(element_id) %>%
summarize(
positive_words = sum(sentiment > 0),
negative_words = sum(sentiment < 0),
compound = sum(sentiment)
) %>%
mutate(
sentiment = ifelse(compound > 0, "Positive", "Negative")
)
```

最后，我们将这个情感摘要与原始文本数据合并，并打印结果。这为我们提供了对数据集中情感的一个清晰、简洁的评价。

```py
# Merge with original text for context using row number as a common column
sentiment_summary <- sentiment_summary %>%
mutate(doc_id = as.character(element_id)) %>%
left_join(data %>% mutate(doc_id = as.character(1:nrow(data))), by = "doc_id") %>%
select(text, positive_words, negative_words, compound, sentiment)
# Print the sentiment evaluation table
print(sentiment_summary)
```

输出表格清晰地显示了每行每个评论的积极和消极词汇计数，以及复合分数和预测情感。一眼看去，模型在排序积极和消极评论方面做得相当不错。尽管由于评论内容在表格中的不完全显示，一些评论明显看起来是负面的（例如，“不会推荐……”），但很可能在该特定评论中包含更多积极关键词（如 satisfies、best、good 等），根据模型评估导致积极情感。因此，在将这些评论包含在结果解释和决策制定之前，需要仔细单独审查这些评论。

接下来，我们需要打印一个文档-特征矩阵（DFM），它是对文本的结构化表示，其中行代表文档，列代表特征（单词）。每个单元格包含文档中特定单词的频率。在这里，清洗后的语料库被转换成 DFM，使其准备好进行统计分析与可视化。

```py
# Create a document-feature matrix (DFM)
dfm <- dfm(corpus_clean)
```

本节计算每个文本条目的情绪指标。将正面和负面单词计数相加，并计算一个复合分数，作为这些计数的差值。正的复合分数表示正面情绪，负的分数表示负面情绪。这些信息与原始文本结合，进行全面的情绪评估。

## 情感分析的最佳实践

+   **文本上下文**：由于基于词汇的情感分析常常忽略讽刺和上下文，使用像机器学习这样的高级技术有助于更好地捕捉细微差别。

+   **特定领域的语言**：由于通用词汇表可能不理解行业特定术语和俚语，将词汇表定制以包括与行业相关的技术术语可以提高准确性。

+   **非正式语言和表情符号的使用**：由于非正式语言和表情符号可能难以分析，使用像 `quanteda` 这样的工具进行数据清理和系统分析是有益的。

+   **结合技术**：由于依赖单一方法会限制分析深度，将文本处理与机器学习相结合提供全面的见解。

## 关键要点

+   情感分析帮助业务了解客户意见，以改进产品和服务的质量。

+   R 包 `quanteda`、`sentimentr` 和 `textstem` 在分析客户评论的文本方面协同工作得很好。

+   所概述的情感分析方法可以轻松应用于金融、医疗保健和零售等行业以获得可操作见解。
