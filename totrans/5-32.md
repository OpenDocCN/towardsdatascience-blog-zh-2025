我最近创建了一个用于 RAG 的图形数据存储。换句话说，我们构建了一个 GraphRAG。

> [**如何在几分钟内构建知识图谱（并使其企业级就绪）**](https://towardsdatascience.com/enterprise-ready-knowledge-graphs-96028d863e8c)

图形 RAGs 是其他 RAG 应用（如广泛使用的基于向量存储的 RAGs）的绝佳替代品。它们带来了推理能力。例如，使用语义相似度搜索（在向量存储中检索信息的技术），你可以询问 XYZ 公司去年谁是 CFO。因为 XYZ 公司去年的年度报告会明确提到其 CFO。但考虑这样一个问题：XYZ 公司的哪两位董事曾在同一所学校学习？如果没有提到学校名称，检索过程将无法获取相关信息。但图形 RAG 可以做到。

然而，这里的关键问题是我们的检索图是如何构建的。我最近在一篇单独的文章中解决了这个问题。再退一步思考，我们如何准备年度报告，使其更容易创建图？

这就是本文的重点。

我们所有工作的第一步工程是将数据从 PDF 转换为文本。然而，年度报告是复杂的文档。其中不仅仅有文本，还会有图表、表格等。每一项都提供了关于公司的重要信息。

因此，让我们从这里开始。

### 如何将 PDF 转换为富文本

大多数 Python 程序员在某个时候都使用过 PDF 阅读器——至少是为了跟随教程。最受欢迎的是 PyPDF2。

大多数这些库都能完成这项工作。但信息的实用性并不高。

我已经知道多年的**PyPDF2**库，它可以无格式地提取所有 PDF 内容。提取后，你不知道哪个是标题，哪个是列表。

然后是**[PyMuPDF4LLM](https://pymupdf.readthedocs.io/en/latest/index.html)**。这个包直接将 PDF 转换为 Markdown。Markdown 也有关于文本结构的许多关键信息。像 Langchain 这样的框架支持 Markdown。它们使用文本格式中的额外信息来更好地分块和存储数据在数据存储中。这反过来又使得检索相关数据变得更加容易。

PyMuPDF4LLM 的问题在于它对表格的处理不佳。提取的表格与原始表格相差甚远。（*不要放弃 PyMuPDF4LLM。它仍然在我们的最终解决方案中发挥了不可思议的作用*）

最后，我们尝试了几种其他现代工具。一个是[Docling](https://ds4sd.github.io/docling/)，由[IBM Deep Search](https://github.com/DS4SD)开发和开源的库，另一个是[Marker](https://github.com/VikParuchuri/marker)，同样是一个出色的库。

Docling 版本比其他两个版本更整洁，并且包含更多的结构化信息。它提取了包含所有相关信息的数据表。它为图像留下了占位符，并保留了层次标题结构。所有这些信息都有助于创建块、图数据库、更好的检索，最终是一个更好的 RAG 系统。

## 什么没有做好？

Marker 和对接都是很有希望的。然而，如果我们花时间将 PDF 转换为 Markdown，我并不太印象深刻。

为了正确测试，我将年度报告中包含文本、表格和图像的片段分割，然后创建了不同页数的几个副本。

我随后使用每个工具对这些不同大小的 PDF 文件进行了转换，并测量了它们所需的时间。这是最终结果。

![](img/8a4c125ebf4c4d2111dca4bfb4871e7b.png)

结果清楚地表明，尽管 Docling 和 PyPDF2 在转换表格方面表现出色，但它们与其他两种选项相比速度太慢。

Docling 处理单个 PDF 页面大约需要 4 秒，而 Marker 则需要大约双倍的时间。

我做了什么？

## 如何选择最佳的 PDF 转 Markdown 工具

我们的项目有数十份年度报告。幸运的是，它们不是数百份。

如果每份年度报告大约有 300 页，并且有 50 份报告使用对接，我们需要 17 小时（每份报告 300 页 x 50 份报告 x 每页 4 秒 / 每小时 3600 秒）。

因此，我们可以承担时间。

但我们仍在原型设计阶段。我们需要考虑项目的扩展。如果这个项目扩展到所有 S&P500 公司和它们的 30 年历史报告，会怎样呢？

除非我们**分配负载并并行处理**，否则将需要 208 天，这正是我们所做的。

我们创建了一个在云中处理转换并更新 Graph DB 的服务。这可以并行运行并很好地扩展。

> 如果速度是你的关注点，PyPDF2 是最好的选择——前提是你不能并行运行 Docling。

## 最后的想法

PDF 转 Markdown 工具已经变得更好。但将 PDF 中的表格转换为 Markdown 表格仍然不是很好。

我们比较了四个免费和开源的 PDF 转 Markdown 工具。然后，我们将 Markdown 文件分块，并在 RAG 应用中转换为图表。

在我们比较的工具中，Docling 产生了出色的结果。然而，缺点是它的性能。它比广泛使用的 PyPDF2 慢得多。

因此，我们转向了一个扩展性良好且并行进行转换的云服务。
