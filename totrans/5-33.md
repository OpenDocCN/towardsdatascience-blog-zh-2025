## 基于深度学习的预测：来自在线时尚行业的案例研究

这篇论文是一颗宝石。

它最初成为头条新闻，因为领先的在线时尚零售商 Zalando 使用定制的 Transformer 模型超越了 SOTA 预测模型——包括 LGBM。

但不仅如此——这篇论文内容丰富，由时间序列/机器学习生态系统中杰出的研究人员撰写。

### 论文见解概述

本文详细介绍了 Zalando 的深度零售预测流程，并提供了有价值的见解：

+   网上零售时尚预测的独特挑战。

+   如何处理稀疏性、冷启动和短历史（图 1）。

+   使用了哪些外部协变量，以及 Transformer 是如何构建以利用它们的？

+   优雅的技巧，有趣的解释如何改进了普通的 Transformer，并将其定制为时间序列预测。

+   详尽的细节，包括定制的损失函数、评估指标、超参数和训练配置——这在许多论文中很少公开。

本文的主要贡献/关键点包括：

+   **协变量处理**：使用折扣作为动态协变量。

+   **因果关系**：通过分段线性函数（由前馈网络参数化）强制执行折扣和需求之间的单调关系。

+   **两种预测模式**：使用单个全局模型训练和预测短期和长期需求。

+   **规模定律**：在 Transformer 预测模型中展示了规模定律的初步火花。

首先，作者解释了他们如何通过将销售观察结果转换为需求预测来配置预测问题。需求计划是零售预测中最常见的案例。

此外，作者巧妙地估计需求（例如，当出现库存短缺时可能会发生这种情况）而不是将其标记为缺失。

> **注意**：**需求预测是供应链预测管道中的第一步且最为重要**
> 
> **需求**代表客户想要购买的内容，而**销售**受库存可用性的限制。**预测需求至关重要，因为它反映了真正的客户需求，而销售则依赖于供应。**
> 
> 如果由于库存短缺导致销售额降至零，基于销售额的未来预测也将反映为零。这可能会误导自动供应链规划系统，导致没有新的订单。
> 
> 为了确保高效的供应链，始终预测需求而不是销售。准确的需求预测可以防止中断并保持库存以满足客户需求。

虽然所有协变量都提高了性能，但折扣影响最大。此外，数据具有每周频率。

## 预测协调：综述

层次预测和协调是时间序列研究中最活跃的领域之一。

一切始于莫纳什大学（其中包括杰出的教授 Rob Hyndman）的研究团队发表了标志性的论文[Hyndman et al., 2011]。当然，这个领域的工作在此之前就已经开始了。

层次预测通过 M5 预测竞赛进入更广泛的机器学习社区。

由同一思想的研究者撰写的论文[2]是跟踪最新预测协调研究的**最佳资源**。

### 预测协调的预备知识

考虑一个具有多个层级的层次时间序列，例如产品和商店。你的任务是预测所有层级。

![图 6：M5 时间序列层级组织概述（来源）](img/41f9734fa6578b554c351eebb5139d15.png)

**图 6**：*M5 时间序列层级组织概述（来源）* ([Source](https://github.com/Mcompetitions/M5-methods/blob/master/M5-Competitors-Guide.pdf))

+   **朴素方法**：我们分别对每个产品、每个店铺和总量进行销售预测。然而，顶级预测与较低级别的总和不会一致，这使得它们**不连贯**。

+   **自下而上的方法**：在这里，预测从底层开始，向上聚合。这避免了信息丢失，但不是最优的。较低级别的序列通常噪声或稀疏，导致在聚合过程中错误累积。

其他方法，如自上而下和中间向外，也有类似的挑战。

最好的方法是建立一个数学协调方法：

> 预测协调不是时间序列模型，而是一个确保不同级别预测之间一致性的**过程**。在所有级别上给出基线预测后，此过程最优地使基线预测保持连贯。
> 
> **✅ 注意**：一些模型可以直接生成连贯的预测（见 [2]）。

### 为什么使用预测协调？

指南通常将预测协调作为一种确保预测有意义的手段，避免经理提出的问题，比如为什么产品 A 和 B 的销售总和与总估计不符。

好的，这也是真的。但不止这些。

**协调通常可以提高分层数据集中的预测准确性**。以下是原因：

+   你可以使用每个级别的最佳模型。

+   例如，不同级别的模型可以利用不同的损失函数（例如，对于非负、右偏斜的较低级别数据使用 Tweedie 损失；对于异常值倾向的顶级使用 Huber 损失）。

一旦在每个级别生成了基线最优预测，你就可以从 [2] 中选择一个适当的协调方法。就这样——砰！你的预测不仅将保持连贯，而且整体上可能更准确！

### 高级调和方法

自下而上的方法很简单——它通过聚合预测来确保一致性。

然而，它不是一个原生的调和方法，因为预测仅仅通过聚合变得一致。

这就是为什么自下而上的方法有一个简单的投影矩阵 **G**，但在实践中，我们可以通过求解矩阵 **G** 来设计一个调和方法。

> 目标是根据方程 6 找到最优的 G 矩阵，以给出最准确的调和预测。

论文从简单的 OLS 方程开始，用于估计 G：

```py
G = (S'S)^-1 S'
```

接着是里程碑论文 [Wickramasuriya et al. (2019)](https://robjhyndman.com/papers/MinT.pdf)，该论文介绍了最小化调和预测误差方差的 *MinTrace (MinT)* 方法。

矩阵 G 然后计算如下：

```py
G = (S'W^(-1) S)^-1 S' W^(-1)
```

其中 **Wh** 是基础预测误差的方差-协方差矩阵。

我们在这里不会进一步深入细节，请自由阅读原始论文[2]。

## TSMixer：用于时间序列预测的全 MLP 架构

Boosted Trees 在预测具有多个协变量的表格型时间序列数据方面表现出色。

然而，由于过拟合问题，深度学习模型在复杂数据集上有效利用其架构方面遇到了困难。

Google 研究人员通过开发轻量级的基于 MLP 的模型 **TSMixer** 及其增强版本 **TSMixer-Ext** 来解决这个问题，**TSMixer-Ext** 可以容纳外生和已知未来的变量。TSMixer-Ext 在 Walmart 的 M5 竞赛中表现出色，超越了现有的 SOTA 模型。

TSMixer 因其双重混合操作而突出，它利用了时间域（时间混合层）和特征域（特征混合层）的交叉变量信息。

## CARD：用于时间序列预测的通道对齐鲁棒混合 Transformer（ICLR 2024）

DLinear 和 TSMixer 论文的发布揭示了基于 Transformer 的预测模型中的弱点。

在多元设置（其中模型学习所有时间序列之间的交叉变量依赖关系并联合预测它们）中，Transformers 往往会过拟合。

二次自注意力机制倾向于捕获噪声，限制了性能——尤其是在较小或玩具数据集上。因此，新的基于注意力的方法转向了单变量训练，产生了令人印象深刻的结果。

CARD 回顾了多元场景，通过重新设计时间序列的注意力机制来追求 SOTA 结果。

### 标记混合模块

在 Transformer 预测中，每个注意力头专注于输入序列中的数据或特征模式的具体方面。例如：

+   一个头可能专注于**短期依赖性**（例如，相邻时间步之间的关系）。

+   另一个可能捕捉**周期性行为**（例如，每周或季节性循环）。

因此，在单个注意力头内：

+   标记编码关于输入序列的相关信息，通过头部的特定“透镜”或学习到的特征模式进行过滤。

+   相邻的标记对应于序列中的**相邻时间步**，并共享时间邻近性。

在传统的注意力中，标记在头之间合并。所提出的**标记混合模块**（**图 12**）通过在多头注意力之后合并同一头部的相邻标记来修改这一点，为下一阶段创建标记：

其中：

+   **a_t(A)**是在时间 t 的观察值，给定历史信息**A**

+   **ahat_t**是在时间 t 的预测值，给定历史信息**A**

+   **L**是序列长度

因此，这个损失函数更强调近未来的准确性。

通常，CARD 是一个令人印象深刻的模型。不幸的是，它还没有被集成到任何已知的预测库中，但作者已经开源了它。我肯定会在这未来的文章中创建一个关于这个模型的教程。

## Spacetimeformer

论文**“用于动态时空预测的长距离变换器[5]****介绍了名为**Spacetimeformer**的新模型。

### 什么使 Spacetimeformer 具有优势？

一般有 3 个关键级别：

+   **时间模型**：传统的基于注意力的时间序列模型在每个时间步用一个标记表示多个变量，忽略了特征之间的空间关系（**图 13b**）。

+   **图模型**：图注意力模型手动使用静态、硬编码的图来编码特征关系，无法随时间适应（**图 13c**）。

+   **时空模型**：Spacetimeformer 整合了时间和空间注意力，将每个特征在特定时间步的值表示为单个标记。这种方法捕捉了空间、时间和价值之间的复杂交互（**图 13d**）。

![图 13：多元预测中的注意力。 (a) 包含 3 个变量、2 个上下文点和 2 个预测点的序列。 (b) 时间注意力，其中每个标记包含所有三个变量，较深的蓝色线表示标记之间的较强注意力。 © 带有空间交互的时间注意力，其中每个时间步内的空间关系使用预定义的空间图（黑色线条）进行建模。 (d) 空间时间注意力，其中每个时间步的每个变量被视为一个单独的标记。](img/b6295faaacbe37b5a4e1af10a76f276d.png)

**图 13：**多元预测中的注意力。 (a) 包含 3 个变量、2 个上下文点和 2 个预测点的序列。 (b) 时间注意力，其中每个标记包含所有三个变量，较深的蓝色线表示标记之间的较强注意力。 © 带有空间交互的时间注意力，其中每个时间步内的空间关系使用预定义的空间图（黑色线条）进行建模。 (d) 空间时间注意力，其中每个时间步的每个变量被视为一个单独的标记。

基于 Transformer 的先前模型在每个时间步使用一个输入标记，以便时间 t 的标记嵌入代表该时刻的 N 个不同变量。

这是有问题的，因为：

+   **滞后信息：**在实际应用中，依赖关系可以是滞后的，例如`y`依赖于`xi-1`而不是`xi`。基于时间的嵌入可能会忽略这些滞后。

+   **有限的感受野：**与 NLP 不同，NLP 中的词汇是离散的，时间序列数据是连续的。按时间点进行标记限制了感受野，限制了长期、跨时间相关性的捕获。

+   **给定嵌入：**多元输入包括时间信息，缺失值（"?"）在预测时设置为零。这些嵌入表示值是作为上下文提供还是需要预测

+   **时间嵌入**：时间序列通过 Time2Vec 层传递以创建 **时间嵌入**，这些嵌入可以捕获周期性模式。

+   **位置嵌入**：在 BERT 等其他模型中使用的典型学习位置嵌入（**图 15** 中未展示）

所有嵌入相加以模拟时间和变量空间之间的关系，但这会增加输入序列长度。

由于自注意力具有二次复杂度，对于大的 N 值，扩展变得具有挑战性。为了解决这个问题，应用以下优化：

+   **Performer 的 FAVOR+ 注意力**：使用随机核方法对注意力进行线性近似。

+   **全局和局部注意力**：局部注意力允许每个标记关注其空间变量的时间步长，而全局注意力则允许标记在整个时空序列中关注。这减少了计算量，同时不会损害时间动态。
