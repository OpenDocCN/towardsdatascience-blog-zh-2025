## 流程

这里是我们将使用的高级流程：

1.  **加载 shapefile**：使用 GeoPandas 加载包含国家边界的 shapefile。*shapefile*是一种地理空间矢量数据格式，用于地理信息系统（GIS）软件。

1.  **过滤到相关国家**：使用 GeoPandas 将感兴趣的国家的数据过滤到单独的 GeoPandas GeoDataFrames 中。

1.  **提取国家边界**：在 GeoDataFrames 的“Geometry”列中定位国家边界。该列包含用于绘制边界的坐标。

1.  **寻找共享边界：** 通过计算主要国家边界与相邻国家边界的交集来识别共享边界。

1.  **创建边界坐标列表**：将边界数据从几何对象（例如，`LineString`或`MultiLineString`）转换为列表。

1.  **绘制边界**：将边界作为 GeoPandas 的`GeoSeries`绘制。

我们还将探讨如何计算边界的长度。

有关 shapefiles 和 GeoPandas 的概述，包括安装说明，请参阅此文章：

## 完整代码

完整的注释代码如下。我将在接下来的章节中更详细地描述它。

```py
import matplotlib.pyplot as plt
import geopandas as gpd

# Define the primary country name:
# (Must match names in shapefile).
COUNTRY1_NAME = 'Venezuela'

# Define the name(s) of adjacent countries:
# (Must match names in shapefile).
COUNTRY2_NAME = ['Colombia', 'Brazil', 'Guyana']

# Load the shapefile as a GeoDataFrame:
gdf = gpd.read_file('ne_10m_admin_0_countries.zip')
gdf = gdf.rename(columns={'ADMIN': 'name'})

# Make separate GDF for country1 and adjacent countries:
country1 = gdf[gdf['name'] == COUNTRY1_NAME]
others = gdf[gdf['name'].isin(COUNTRY2_NAME)]

# Extract the boundaries:
country1_border = country1.boundary.unary_union
others_border = others.boundary.unary_union

# Find intersections
shared_borders = country1_border.intersection(others_border)

# Ensure geometry is a MultiLineString or LineString:
if shared_borders.geom_type == 'GeometryCollection':
    borders = [geom for geom in shared_borders.geoms if 
               geom.geom_type in ['LineString', 'MultiLineString']]
elif shared_borders.geom_type in ['MultiLineString', 'LineString']:
    borders = [shared_borders]
else:
    borders = []

# Create a dynamic title:
countries_in_title = f"{COUNTRY1_NAME} and {', '.join(COUNTRY2_NAME)}"
title = f"Shared Borders Between {countries_in_title}"

# Set up the figure:
fig, ax = plt.subplots(figsize=(10, 10))
ax.set_title(title, fontsize=14)
# ax.set_xticks([])
# ax.set_yticks([])
# ax.set_xticklabels([])
# ax.set_yticklabels([])

# Plot the borders:
gpd.GeoSeries(borders).plot(ax=ax, 
                            color='firebrick', 
                            linewidth=2)

# plt.savefig(f"{COUNTRY1_NAME}_border_segment.png", dpi=600)

plt.show()
```

### 导入库和分配常量

首先，导入 Matplotlib（用于绘图）和 GeoPandas（用于处理 shapefiles）。接下来，我们为主要的国名（COUNTRY1_NAME）和共享边界的相邻国家（COUNTRY2_NAME）分配常量。后者变量使用一个包含一个或多个国家名称的 Python *列表*。

```py
import matplotlib.pyplot as plt
import geopandas as gpd

# Define the primary country name:
# (Must match names in shapefile).
COUNTRY1_NAME = 'Venezuela'

# Define the name(s) of adjacent countries:
# (Must match names in shapefile).
COUNTRY2_NAME = ['Colombia', 'Brazil', 'Guyana']
```

> **注意：**用作常量的国家名称必须与 shapefile 中的名称匹配。我们将在下一节中更详细地讨论这一点。

### 加载和准备 Shapefiles

现在我们将下载的 shapefile 加载为名为 "gdf" 的 GeoDataFrame。注意我们不需要解压 shapefile。

```py
# Load the shapefile as a GeoDataFrame:
gdf = gpd.read_file('ne_10m_admin_0_countries.zip')
gdf = gdf.rename(columns={'ADMIN': 'name'})

# Make separate GDF for country1 and adjacent countries:
country1 = gdf[gdf['name'] == COUNTRY1_NAME]
others = gdf[gdf['name'].isin(COUNTRY2_NAME)]
```

自然地球 shapefile 使用无意义的名称 "ADMIN" 作为国家名称列。我们可以使用 GeoPandas 的 `rename()` 方法来修复这个问题。

接下来，我们使用新创建的 "name" 列创建两个新的 GeoDataFrames。对于第一个（`country1`），我们使用 COUNTRY1_NAME 进行过滤。由于 COUNTRY2_NAME 变量是一个列表，我们添加了一个转折，并使用了 `isin()` 方法。这样，列表中的任何匹配名称都将用于过滤。

说到匹配名称，如果您不确定国家名称的拼写或存储在 GeoDataFrame 中（例如，“United States of America”与“USA”），您可以使用此命令打印所有名称：

`print(gdf['name'].unique())`

### 寻找共享边界

在 GeoDataFrames 中加载数据后，我们现在提取每个国家的边界并找出它们的交点。

```py
# Extract the boundaries:
country1_border = country1.boundary.unary_union
others_border = others.boundary.unary_union

# Find intersections
shared_borders = country1_border.intersection(others_border)

# Create list from geometry object:
if shared_borders.geom_type == 'GeometryCollection':
    borders = [geom for geom in shared_borders.geoms if 
               geom.geom_type in ['LineString', 'MultiLineString']]
elif shared_borders.geom_type in ['MultiLineString', 'LineString']:
    borders = [shared_borders]
else:
    borders = []
```

要提取边界，我们将 GeoPandas 的`boundary`操作与 Shapely 的`unary_union`属性（Shapely 是 GeoPandas 在底层用于几何操作的库）链式使用。第一部分产生一个包含 Shapely `LineString`或`MultiLineString`对象的`GeoSeries`。第二部分通过在`GeoSeries`中的所有几何形状上执行并集操作来聚合多个几何形状（例如，从`country1.boundary`）。它返回一个单一的 Shapely `Geometry`对象（例如，`LineString`、`MultiLineString`或`GeometryCollection`）。

下一步是使用 Shapely 的`intersection()`方法识别共享边界。此方法计算两个几何形状的重叠部分，例如线或多边形。它根据输入几何形状及其空间关系返回多个 Shapely 几何对象之一。例如，包括线字符串、点、多边形等。

由于`intersection()`方法可能产生的复杂输出，我们必须检查数据类型是否适用于处理线。我们通过一个条件块来完成这项工作，该块将最终边界坐标作为 Python 列表中的 Shapely 几何形状返回。

### 绘制结果

代码的其余部分绘制了边界。在设置完图形后，我们将`borders`从列表转换为 GeoSeries（`gpd.GeoSeries(borders)`），然后在这个对象上调用`plot()`方法。

```py
# Create a dynamic title:
countries_in_title = f"{COUNTRY1_NAME} and {', '.join(COUNTRY2_NAME)}"
title = f"Shared Borders Between {countries_in_title}"

# Set up the figure:
fig, ax = plt.subplots(figsize=(10, 10))
ax.set_title(title, fontsize=14)

# Uncomment to hide axis labels and ticks:
# ax.set_xticks([])
# ax.set_yticks([])
# ax.set_xticklabels([])
# ax.set_yticklabels([])

# Plot the borders:
gpd.GeoSeries(borders).plot(ax=ax, 
                            color='firebrick', 
                            linewidth=2)

# plt.savefig(f"{COUNTRY1_NAME}_border_segment.png", dpi=600)

plt.show()
```

使用 GeoSeries 进行绘图是首选的，因为 GeoPandas 的绘图功能与地理投影集成，并允许对空间数据进行更高级的处理。

### 计算边界的长度

*理论上*，在提取边界后，我们可以计算出边界的长度。第一步是在加载数据后，将 GeoDataFrame 的坐标参考系统（CRS）转换为等面积投影（例如，EPSG:2163）：

`gdf = gdf.to_crs('EPSG:2163')`

然后，在将边界作为 Shapely 几何形状的列表提取出来后，我们可以运行以下代码：

```py
# Calculate the total length of the shared border:
total_length = sum(line.length for line in borders)
print(f"Total shared border length (in projected CRS units): ")
print(f"{total_length / 1000} km")
```

`borders`中的每个几何形状（例如，`LineString`或`MultiLineString`）都有一个由 Shapely 提供的`.length`属性。此属性是几何形状在当前 CRS 数据单位中的计算长度。

不幸的是，我们使用 Natural Earth 数据集将无法获得准确的结果。根据他们的网站，它被设计为“[展示世界在大型的海报上](https://www.naturalearthdata.com/downloads/)”。这意味着它为了简化而平滑掉了细节。

计算一个国家边界的真实长度是一项棘手的工作。你需要一个非常高分辨率的 shapefile 来处理*崎岖*的边界。例如，考虑俄罗斯和哈萨克斯坦的边界，它在某些地方沿着蜿蜒的河流：

![哈萨克斯坦-俄罗斯边界的部分（来源：Google Maps）](img/ff182a04ec460a97c4bc06ba8e0e13b0.png)

哈萨克斯坦-俄罗斯边界的部分（来源：Google Maps）

大多数形状文件无法捕捉到这条边界的每一个曲折，因此我们会低估其长度。当然，我们永远无法知道任何边界的*真实*长度，因为我们永远不会有一个具有*无限*分辨率的形状文件。

除了高分辨率形状文件，您可能还需要使用多个坐标参考系统（CRS）。地理坐标参考系统中的纬度相关畸变使得长度在接近极点时看起来更长或更短。跨越超过六度纬度或经度的边界应分成几个部分，并使用适当的*局部*坐标参考系统重新投影，以最小化每个部分的畸变。选项包括[UTM 区域](https://en.wikipedia.org/wiki/Universal_Transverse_Mercator_coordinate_system)或其他本地优化的投影。

测量方法也同样关键。对于长边界的长度计算应使用*[大地测量方法](https://www.ncesc.com/geographic-faq/what-is-planar-and-geodesic-measurement/)*，这些方法考虑到了地球的曲率。当使用地理坐标参考系统时，GIS 工具如 GeoPandas 支持大地测量长度计算。

GeoPandas 提供了一些工具，可以帮助您找到并提取地图区域之间共享的边界线。一旦提取出来，这些边界线就可以进行绘制和测量。这些测量的准确性取决于形状文件的分辨率以及所使用的投影和测量方法。
