如果你曾经为一个新的项目训练过分割模型，你可能知道这不仅仅关于模型。这是关于数据。

收集图像通常很简单；你通常可以在像 [Unsplash](https://unsplash.com/) 这样的平台上找到很多，甚至可以使用像 Stable Diffusion 这样的生成式 AI 工具来生成更多：

> [**如何在没有训练数据的情况下训练实例分割模型**](https://towardsdatascience.com/how-to-train-an-instance-segmentation-model-with-no-training-data-190dc020bf73)

主要挑战通常在于标注。为分割标注图像是非常耗时的。即使有像 [Meta 的 SAM2](https://ai.meta.com/sam2/) 这样的高级工具，创建一个完全标注、稳健且多样化的数据集仍然需要相当多的时间。

在本文中，我们将探讨另一个较少被探索的选项：使用 3D 工具，例如 [Blender](https://www.blender.org/)。确实，3D 引擎越来越强大和逼真。此外，它们提供了一种引人注目的优势：在创建数据集的同时自动生成标签，消除了手动标注的需求。

在本文中，我们将概述创建手部分割模型的一个完整解决方案，分为以下关键部分：

+   使用 Blender 生成手部，以及如何获得手部姿势、位置和肤色的多样性

+   使用 [OpenCV](https://opencv.org/) 生成数据集，该数据集由生成的 Blender 图像和选定的背景图像组成

+   使用 PyTorch 训练和评估模型

当然，本文中使用的所有代码都是完全可用和可重用的，在以下 [GitHub](https://github.com/vincent-vdb/medium_posts) 仓库中。

## 生成手部

要生成手部图像，让我们使用 Blender。我不是这类工具的专家，但它为我们提供了许多非常实用的功能：

+   它是免费的——没有商业许可，任何人都可以立即下载和使用它

+   有一个庞大的社区，可以在网上找到许多模型，有些是免费的，有些则不是

+   最后，它包括一个 Python API，使图像生成的自动化具有各种功能

正如我们将看到的，这些功能非常有用，并将使我们能够轻松地生成合成数据。为了确保足够的多样性，我们将探讨如何自动随机化以下参数在我们的生成手中：

+   手指的位置：我们希望有各种不同位置的手部图像

+   摄像机位置：我们希望从各种角度拍摄手部图像

+   肤色：我们希望肤色多样化，使模型足够鲁棒

*注意：这里提出的方法并非没有基于肤色的潜在偏见，也不声称是无偏见的。基于此方法产生的任何产品都必须仔细评估其伦理偏见。*

在深入这些步骤之前，我们需要一个手的 3D 模型。在诸如[Turbosquid](https://www.turbosquid.com/)等网站上有很多模型，但我使用了一个可以在此找到的免费手部模型。如果你用 Blender 打开这个文件，你会得到以下截图类似的内容。

### 评估模型

为了评估模型，以及混合图像的改进，让我们进行以下比较：

+   在[Ego Hands 数据集](http://vision.soic.indiana.edu/projects/egohands/)上训练和评估一个模型

+   在 Ego Hands 数据集上训练和评估相同的模型，并将我们的混合生成的数据添加到训练集中

在这两种情况下，我将在 Ego Hands 数据集的相同子集上评估模型。作为一个评估指标，我将使用[交并比 (IoU](https://en.wikipedia.org/wiki/Jaccard_index))（也称为 Jaccard 指数）]。以下是结果：

+   仅在 Ego Hands 数据集上，经过 20 个 epoch 后：**IoU = 0.72**

+   在 Ego Hands 数据集和 Blender 生成的图像上，经过 20 个 epoch 后：**IoU = 0.76**

如我们所见，我们可以通过使用由 Blender 生成的图像组成的数据集，在 IoU 上获得显著的提升，从 0.72 提升到 0.76。

## 结论

让我们以几个关键要点来总结这篇文章：

+   Blender 是一个强大的工具，它允许你在各种条件下生成逼真的图像：光线、相机位置、变形等…

+   利用 Blender 生成合成数据可能最初需要一些时间，但可以使用 Python API 实现完全自动化

+   使用生成的数据提高了语义分割任务的模型性能：将 IoU 从 0.72 提高到 0.76

+   为了获得更多样化的数据集，可以使用更多的 Blender 手部模型：更多的手部形状、更多的纹理可以帮助分割模型更好地泛化

最后，如果你成功构建了一个可工作的模型，并想找到部署它的最佳策略，可以查看这篇指南：

作为旁注，虽然这篇文章侧重于语义分割，但这种方法适用于其他计算机视觉任务，包括实例分割、分类和地标预测。我很乐意听到我可能遗漏的其他 Blender 的潜在用途。
