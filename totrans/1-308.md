# 匈牙利算法及其在计算机视觉中的应用

> 原文：[https://towardsdatascience.com/hungarian-algorithm-and-its-applications-in-computer-vision/](https://towardsdatascience.com/hungarian-algorithm-and-its-applications-in-computer-vision/)

## <mdspan datatext="el1757378960003" class="mdspan-comment">介绍</mdspan>

**多目标跟踪（MOT）**是一个任务，其中算法必须在视频中检测和跟踪多个对象。大多数已知算法都是基于使用为处理单个图像设计的简单检测器（例如YOLO）。整体方法涉及在连续的视频帧上分别使用检测器，然后匹配属于同一对象的对应边界框。

使MOT算法与众不同的核心部分是它们在视频帧之间进行匹配的方式。它们可以考虑到几个因素来执行匹配：

+   *边界框位置；*

+   *遮挡*（当多个对象的边界框相互交叉时）；

+   *对象的运动；*

+   *物理对象相似性；*

在某些情况下，对于连续帧A和B上的每对边界框，这些特征被合并成一个数字，该数字描述了在帧A和B中检测到的边界框对属于同一对象的可能性。

![](../Images/a90dd5bc184602528bb84e2205d79b0c.png)

多目标跟踪（MOT）的例子：该算法跟踪两个球。

这些值是在帧A和B中所有边界框对之间计算的。然后，MOT算法试图识别所有边界框之间可能的最优匹配。更具体地说，给定帧A和B中检测到的n个边界框，目标是创建一个映射，使得A中的每个边界框只映射到B中的一个边界框，并且每个边界框只使用一次。

## 匈牙利算法

匈牙利算法通常在算法和数据结构课程中学习。尽管如此，它也应用于匹配系统，特别是经常用于解决上述提到的轨迹匹配问题。

我们将研究在更简单设置中匈牙利算法的工作流程。一旦我们了解了它的使用方法，我们就可以轻松地将它应用于MOT问题。

> *该算法被称为匈牙利算法，因为它“主要基于两位匈牙利数学家[Dénes Kőnig](https://en.wikipedia.org/wiki/D%C3%A9nes_K%C5%91nig)和[Jenő Egerváry](https://en.wikipedia.org/wiki/Jen%C5%91_Egerv%C3%A1ry)的早期工作” — [匈牙利算法 | 维基百科](https://en.wikipedia.org/wiki/Hungarian_algorithm)*

## 公式化

存在许多示例来演示匈牙利算法。我喜欢工人和任务的例子。以下是公式化：

有n名工人可用，他们需要完成n项任务。关于每个工人每项任务的薪资信息都有所了解。至于公司总监，问题在于在以下条件下最优地分配任务给工人：

+   每个工人只分配一个任务；

+   所有任务都得到完成；

+   在薪水中花费的金额应尽量减少。

我们将使用以下 *4 x 4* 成本矩阵作为例子来解决这个问题：

![图片](../Images/f595a3dc6fe32fe605d9d5a4e49ef16d.png)

包含每个工人完成每项任务的成本信息的成本矩阵。目标是分配每项任务给每个工人，以便所有任务都以最小的总成本完成。

> *从几何学角度来说，给定上述矩阵，目标是在不重复任何行或列中的元素的情况下选择n个矩阵元素，并且所选元素的总和是最小的。*

### 思路

匈牙利方法涉及将初始成本矩阵转换成一种新的形式，这有助于搜索解决方案。为此，我们将使用几个矩阵变换。尽管矩阵将被修改，但问题始终保持等价，这意味着解决方案仍然是相同的。

> *为了简化问题，我们在此不进行数学证明来解释为什么这个或那个矩阵变换保持了问题的不变性。相反，我们将提供一些逻辑思考来解释为什么解决方案保持不变。*

## 示例

### 1. 行与列的缩减

第一步是在矩阵的每一行中识别最小元素，并从每一行中减去它。这里的想法是在每一行中至少得到一个零。在实践中，有更多的零可以简化问题。

> *假设从给定行中减去某个数m。在变换过程中，目标值（总最小薪资）会发生变化（减少m），但相同工人的分配之间的相对成本保持不变。因此，解决方案的排名保持不变。*

然后对列执行类似的程序：从每一列中减去该列的最小元素。

![图片](../Images/1bc7064fabc59cafa3c407843edf884f.png)

前两步：1. 从每一行中减去最小元素；2. 从每一列中减去最小元素。

经过前两次变换后，我们得到一个包含一些零的矩阵，这些零代表潜在的分配。

下一步是在矩阵中绘制最少的水平和垂直线，使它们通过所有零。在下面的图像中，我们可以画出k = 3条线来覆盖所有零。

![图片](../Images/0c2faee05a9151a155c62cb338946c71.png)

所有的零都可以用k = 3条线来覆盖。

如果绘制的线条数等于矩阵的维度 n，那么我们就找到了一个解。在这种情况下，剩下的唯一步骤是选择 n 个零，使得没有零位于另一个零的同一水平或垂直线上。

由于在我们的例子中 n ≠ k（矩阵维度 n = 4；绘制的线条数 k = 3），这意味着我们必须执行一个调整步骤。

### 2. 调整步骤

到目前为止，我们已经绘制了几条线，我们可以将矩阵元素分为三类：

+   未覆盖元素；

+   覆盖元素（只覆盖一次）；

+   角元素（被水平和垂直覆盖两次的元素）。

调整步骤的想法是识别未覆盖元素中的最小元素，并从所有未覆盖元素中减去它。同时，这个值被加到所有角元素上。

在我们的例子中，最小的未覆盖元素是 2。因此，我们从所有未覆盖元素（红色）中减去 2，并从所有角元素（绿色）中减去 2。

![图片](../Images/67ca42a6720dd379043cdf6769eba8e8.png)

调整步骤涉及从每个覆盖元素（红色）中减去最小覆盖元素（2），并将其（2）加到所有角元素（绿色）上。

> *尽管调整步骤后问题不变性为何仍然保持可能并不明显，但可以通过数学证明，从所有未覆盖的成本中减去一个数字，通过将其加到所有覆盖的成本上两次来补偿，从而保持最优解不变。*

这次转换是调整步骤的单次迭代，它使我们到达了另一个等价的矩阵形式。和之前一样，我们执行检查以确定是否可以使用 *n* 条线覆盖所有零。

![图片](../Images/6b1920c90c40ff2d14c477d9e5b63bbc.png)

这次，所有零只能用 k = n = 4 条线来覆盖。这意味着不需要进一步的调整步骤，最终解可以检索到。

如我们所见，这次我们确实需要绘制 k = n = 4 条线来覆盖所有零。这意味着我们最终可以检索到我们的解！

> *否则，如果我们能绘制少于 k < 4 条线，我们应该重复调整步骤，直到线条数变为 k = 4。*

### 3. 解的检索

剩下的唯一步骤是找到不同垂直和水平线上的 n 个零。如果手动操作，最好从零较少的线开始。

在找到矩阵中零的位置后，我们可以切换回原始矩阵，并选择与找到的零具有相同位置的初始元素。那将是最终的分配！

![图片](../Images/0b5200c23ba803e60647ebbaa66eb3dd.png)

在最后一步，必须在水平和垂直线上找到 n = 4 个不重复的零。它们的位置与初始矩阵的成本值的位置完全对应。

> *匈牙利算法的复杂度是 O(n³)，其中 n 是矩阵维度。*
> 
> *我们刚才看到的分配问题也可以通过线性规划方法来解决。*

## 应用

匈牙利算法最明显的应用之一是将其用于分配问题，其中，给定n个任务，目标是将它们与完成它们的其他人或对象（例如机器）最优地关联起来。

在计算机视觉中也有特定的应用。许多视频跟踪算法（MOT）基于标准图像检测算法（例如YOLO）和将来自几个独立帧的检测结果合并到视频流中的逻辑的组合。

让我们以一个简化的视频两个连续图像帧的例子来说明：

![图片](../Images/822bff1287ea8a2d0f33694847c936d9.png)

两个连续视频帧的示例。

我们运行基于YOLO的对象跟踪MOT算法。YOLO的预测如下所示，以灰色框表示。

![图片](../Images/7a1af67e3687347fe87b2a4fccdb8685.png)

YOLO算法在每一帧中检测到树上的两个球。目标是理解两个帧上的检测是如何相互关联的。

目标是将两个帧之间的边界框关联起来以跟踪物体。一种可能的方法是分析两个帧之间边界框距离的变化。如果两个帧之间边界框的位置变化不大，那么可以合乎逻辑地假设这对边界框属于同一物体。

正是在这里，匈牙利算法发挥了作用。我们可以构建一个矩阵，表示两个帧中不同边界框坐标之间的成对距离（这将是成本函数）。我们可以找到这样的映射，以最小化边界框之间的总距离。

![图片](../Images/bb21a0200a805c3eb50c1c3aa99abd14.png)

将匈牙利算法应用于包含边界框中心距离的矩阵的结果。

通过运行匈牙利算法，我们得到以下映射：

*(A₁, C₂), (B₁, A₂), (C₁, B₂)*。

我们可以验证它们的结果是总成本*8 + 1 + 5 = 14*，这是此矩阵可能的最小函数成本。因此，找到的映射是最佳的。

![图片](../Images/974c01c582a9ed634526bbebe63e1cf9.png)

在两个帧上映射边界框。

> *当然，现代MOT算法在匹配边界框时考虑了额外的因素，包括轨迹分析、物体速度和方向以及物理相似性等。为了简单起见，我们只考虑了一个因素：边界框之间的距离。然而，在现实中，考虑的因素更多。*

## 结论

在本文中，我们探讨了用于解决任务分配问题的匈牙利算法。通过在初始数据矩阵上执行简单操作，匈牙利算法将其转换为其他格式，同时保持问题不变。

尽管匈牙利算法的复杂度为立方级，但它广泛应用于匹配和计算机视觉问题中，其中对象的数量不是太多。
