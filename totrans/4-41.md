数据稀缺是许多数据科学家面临的一个大问题。

这可能听起来很荒谬（“这不是大数据的时代吗？”），但在许多领域，简单地没有足够的标记训练数据来使用传统的机器学习方法训练性能良好的模型。

在分类任务中，对这个问题的一种懒惰方法是“把 AI 投进去”：使用现成的预训练 LLM，添加一个巧妙的提示，然后一切就绪。

但大型语言模型（LLMs）并不总是最佳工具。在规模上，LLM 管道可能很慢、昂贵且不可靠。

另一个选择是使用为少样本场景（训练数据很少）设计的微调/训练技术。

在这篇文章中，我将向您介绍我非常喜欢的一项技术：**SetFit**，这是一个微调框架，可以帮助您使用**每个类别最多 8 个标记样本**来构建高性能的 NLP 分类器。

## 我第一次了解到 SetFit 是在一个为金融行业客户交付的项目中

我们试图构建一个能够对具有微妙差异的特定领域文本进行分类的模型。不幸的是，我们每个类别只有大约 10 个样本（以及 ~200+ 个类别），并且在使用我们的传统 NLP 工具（TF-IDF、BERT、DistilBERT、RoBERTa、OpenAI/Llama 3）时并没有取得太多成功。

在寻找解决方案的过程中，我发现了 SetFit，并在看到 SetFit 原创创作者进行的这个令人瞩目的实验结果后决定尝试它：

## 使用 SetFit 训练的模型在小样本量下容易超越 RoBERTa Large

这让我感到震惊，但要理解为什么会这样，我们需要了解 SetFit 究竟是什么。

[SetFit](https://huggingface.co/blog/setfit) 是一个用于 Sentence Transformers（文本嵌入模型）的少样本微调框架。它由 HuggingFace 🤗 的研究人员开发，并且可以通过 `transformers` 和 `setfit` Python 库完全免费使用。

> 什么是句子转换器？它只是专门设计用于编码/嵌入文本的神经网络。它们是现代 NLP 工具包的关键部分，因为它们允许你将文本转换为高维、密集的向量，这些向量捕捉了文本的语义意义。反过来，这些向量表示可以用于训练预测模型（例如，文本分类器）的特征。

因此，SetFit 是一个用于微调句子转换器的框架。但这是什么意思呢？

用简单的话说，这意味着它可以用来创建定制的、微调的嵌入模型，使用非常小的训练数据集。

这通过两个阶段来完成：

1.  微调句子转换器（使用对比学习）

1.  训练分类头（例如，逻辑回归）

## 对比学习是什么，为什么它在小型、特定领域的数据集上表现得如此出色？

大多数嵌入模型/句子转换器都是通过观察**共现模式**来训练嵌入文本的——它们通过观察单词经常一起出现来学习嵌入。

例如，在像维基百科这样的大型训练语料库中，“茶”和“咖啡”这些词可能会频繁地出现在同一类文章、同一类句子、同一类单词附近：

![作者图片。维基百科截图。是的，我们英国人确实把咖啡馆叫做“油腻的勺子”](img/2b81be28bdbee294669655cac9224cb4.png)

作者图片。维基百科截图 [Wikipedia](https://en.wikipedia.org/wiki/Cafe_(British))。是的，我们英国人确实把咖啡馆叫做“油腻的勺子”。

因此，嵌入模型学会了使用相似的嵌入来编码这些共同出现的单词：

![作者图片](img/ea7621af646ed52e08896cfdc6d0f65d.png)

作者图片

这些嵌入对于通用语言模型非常有用（它赋予了它们识别*茶*和*咖啡*属于同一广泛语义空间的能力——即，它们都是热饮），但在我们需要构建能够识别这些类别之间*差异*的细粒度分类器的上下文中，它们并不特别有用。

在对比学习中，**嵌入器被明确训练来生成特定任务的嵌入，这些嵌入在区分不同类别或类别方面非常出色**（在这种情况下，茶与咖啡）。

这分为三个阶段：

1.  **创建对**：SetFit 算法首先创建句子的对。这些对被标记为**正**（如果句子有相同的标签）或**负**（如果它们有不同的标签）：

![作者图片](img/46bb3f9475c1f916a53fe0c28e6485b6.png)

作者图片

**2. 嵌入句子**：每个句子都通过一个预训练的句子转换器来获取嵌入（向量表示）。这些嵌入捕捉了句子的语义意义。

**3. 使用对比损失调整嵌入**：使用类似于余弦相似度损失或三元组边界损失（取决于你设置的配置），模型调整嵌入，以便：

+   **正面对** 中的句子在嵌入空间中被拉近。

+   **负面对** 中的句子在嵌入空间中被推得更远。

例如：

+   "I love drinking tea" 和 "The tea here is terrible" 的嵌入被调整为更接近，因为它们都是关于 **茶** 的（即，它们是一个 "正面" 对）。

+   "I love coffee" 和 "Tea is amazing" 的嵌入被调整为更远，因为它们不是关于相同主题的。

最终结果是模型主动学习在向量空间中区分不同类别的句子，确保来自同一类别的句子更相似，而来自不同类别的句子则不那么相似：

![作者图片](img/067cafa918b72d8c17c6b1e2f6f84c90.png)

作者图片

这最大化了向量中包含的 "信息"（在数学意义上），可以帮助你使用非常少的训练数据构建非常强大的分类器。

## 一个工作示例：分类技术新闻文章

让我们看看一个真实例子，看看 SetFit 与其他方法的比较。我们将使用 [20 个新闻组数据集](https://archive.ics.uci.edu/dataset/113/twenty+newsgroups)（CC BY 4.0 许可证）的一个子集，该数据集包含数千篇新闻文章的标题以及每篇文章对应的类别。

我们将从每个紧密相关（但不同）的 5 个计算类别中各抽取 20 篇文章的样本：图形、Microsoft Windows、IBM、Mac 和 Windows X。

我们的目标是构建一个分类器，可以识别给定技术文章的适当类别。这些类别的语义相似性使得这是一个棘手的挑战：我们的分类器会如何？

首先，让我们 `pip install` 我们所需的包并导入所需的库：

```py
# Fetch data for all the "Science" categories
cats = ['comp.graphics', 'comp.os.ms-windows.misc', 'comp.sys.ibm.pc.hardware', 'comp.sys.mac.hardware','comp.windows.x']
train = fetch_20newsgroups(subset='train', categories=cats, remove=('headers', 'footers', 'quotes'))
test = fetch_20newsgroups(subset='test', categories=cats, remove=('headers', 'footers', 'quotes'))

# Convert to DataFrame
train_df = pd.DataFrame({'text': train.data, 'label': train.target})
train_df['label'] = train_df['label'].apply(lambda x: train.target_names[x])
test_df = pd.DataFrame({'text': test.data, 'label': test.target})
test_df['label'] = test_df['label'].apply(lambda x: test.target_names[x])

# Remove stopwords, lowercase, etc. 
def preprocess_text(text):
    text = text.lower() # Lowercase
    text = re.sub(r'[^a-zA-Zs]', '', text) # Remove special characters and numbers
    tokens = text.split()
    return ' '.join(tokens)

train_df['text'] = train_df['text'].apply(lambda x: preprocess_text(x))
test_df['text'] = test_df['text'].apply(lambda x: preprocess_text(x))

# Stratified sample in the training dataset: 20 samples per class
train_df = train_df.groupby('label', group_keys=False).apply(lambda x: x.sample(20, random_state=42))

train_df # Preview
```

在这些文本被用于训练分类器之前，我们需要将它们嵌入（即，我们需要将它们编码为向量）。

使用 SetFit，我们可以微调现成的句子转换器，使其适应我们文本的具体细微差别，然后拟合逻辑回归分类头：

```py
import os
os.environ["WANDB_DISABLED"] = "true" # If running in Google Colab

# SetFit needs a Dataset class with two cols: `text` and `label`
train_dataset = Dataset.from_pandas(train_df)
val_dataset = Dataset.from_pandas(test_df)

# A popular and performant sentence transformer from HuggingFace
model = SetFitModel.from_pretrained("sentence-transformers/paraphrase-mpnet-base-v2",)

model.labels = train_dataset['label']

trainer = Trainer(
    model=model,
    train_dataset=train_dataset,
)

trainer.train()

accuracy = trainer.evaluate(val_dataset)
y_pred = model.predict(val_dataset['text']) # Generate an array of predictions which we can insert into a `results` DataFrame later

results = pd.DataFrame({
    'text': test_df.text,
    'label': test_df.label,
    'y_pred': y_pred
})

print(f"Accuracy: {accuracy}")
# 0.63
```

因此，使用 SetFit 训练的模型在每类 20 个样本的情况下达到了 63% 的准确率。当每类使用 40 个样本时，准确率上升至 68%。

RoBERTa 模型表现更好，但仅当使用 **大量** 数据时。看看：

```py
from transformers import RobertaTokenizer, RobertaForSequenceClassification, Trainer as HFTrainer, TrainingArguments
from torch.utils.data import Dataset as TorchDataset

# Prepare the data for RoBERTa
class TextClassificationDataset(TorchDataset):
    def __init__(self, texts, labels, tokenizer, max_length=128):
        self.texts = texts
        self.labels = labels
        self.tokenizer = tokenizer
        self.max_length = max_length

    def __len__(self):
        return len(self.texts)

    def __getitem__(self, idx):
        text = self.texts[idx]
        label = self.labels[idx]
        encoding = self.tokenizer(
            text,
            max_length=self.max_length,
            padding="max_length",
            truncation=True,
            return_tensors="pt",
        )
        return {key: val.squeeze(0) for key, val in encoding.items()}, label

# Tokenizer and model initialization
tokenizer = RobertaTokenizer.from_pretrained("roberta-base")
roberta_model = RobertaForSequenceClassification.from_pretrained("roberta-base", num_labels=len(set(train_df['label'])))

train_dataset_roberta = TextClassificationDataset(
    train_df['text'].tolist(),
    train_df['label'].tolist(),
    tokenizer,
)

val_dataset_roberta = TextClassificationDataset(
    test_df['text'].tolist(),
    test_df['label'].tolist(),
    tokenizer,
)

# Training arguments for RoBERTa
training_args = TrainingArguments(
    output_dir="./roberta-results",
    evaluation_strategy="epoch",
    per_device_train_batch_size=8,
    per_device_eval_batch_size=8,
    num_train_epochs=3,
    save_steps=1000,
    logging_dir="./logs",
    logging_steps=10,
)

hf_trainer = HFTrainer(
    model=roberta_model,
    args=training_args,
    train_dataset=train_dataset_roberta,
    eval_dataset=val_dataset_roberta,
)

print("Training RoBERTa model...")
hf_trainer.train()

# Evaluate RoBERTa Model
roberta_y_pred = hf_trainer.predict(val_dataset_roberta).predictions.argmax(axis=-1)
roberta_accuracy = accuracy_score(test_df['label'], roberta_y_pred)

results_summary['RoBERTa'] = {
    'accuracy': roberta_accuracy,
    'predictions': roberta_y_pred,
}

print(f"RoBERTa Model Accuracy: {roberta_accuracy:.4f}")
```

当使用总共 3,000 个样本时，准确率为 74%。但是，当每个类别只使用 20 个样本运行该代码（即与 SetFit 和 TF-IDF 相同）时，准确率仅为 4*7%.* 这与 SetFit 的[原始作者](https://huggingface.co/blog/setfit)的研究结果相符：

> 与标准微调相比，SetFit 在样本效率和抗噪声方面显著更优。
