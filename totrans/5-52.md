## 简介——默认情况下，没有图表可以处理主导类别。

当人们想到条形图时，他们通常会想到清晰且易于区分的条形，这些条形有效地展示了数据。但是，如果有一个条形如此之大，以至于它使其他所有条形都相形见绌，会发生什么呢？在阅读我之前的[精彩的 Plotly 与代码系列（第三部分）：突出显示长尾中的条形](https://medium.com/towards-data-science/awesome-plotly-with-code-series-part-3-highlighting-bars-in-the-long-tails-8e5116e36003)文章后，你可能会说：“约瑟，你已经展示了如何处理小而难以检测的条形图，只需突出显示它们即可”。这完全正确，但我的上一篇文章是突出显示 2 个条形，而不是除了一个以外的所有条形！

在这篇文章中，我将向您展示如何展示一个图表，其中既包括高大的“摩天大楼”条形，也包括较小的“单层房屋”条形，并且它们都被平等地表示。

## 场景 1：摩天大楼条形图和默认绘图问题

想象一下展示一项关于英格兰使用的语言的研究，并尝试创建一个能够有效展示这些见解的条形图。想法是专注于非英语语言，因为很明显英语是主导语言。然而，你仍然想展示英语相对于其他语言的统治程度。以下是你可能正在处理的数据：

![数据来源：ONS 数据](img/43a9db074bb7e580eb3860f20f092ee3.png)

数据来源：[ONS 数据](https://www.ons.gov.uk/datasets/TS024/editions/2021/versions/1)

你可以看到，有 5200 万人说英语……而下一个最接近的语言是波兰语，有 60 万！这相差 86 倍！如果你要在一个条形图中展示这些信息，而不真正考虑非英语语言会发生什么，那么你会发现自己在盯着下面的图表。

![数据图表](img/ff7c43985df7a33fdf53561e4e70ee6b.png)

**我认为这个图表有哪些问题？**

1.  很明显，较小的条形如此之小，以至于几乎无法区分它们的高度差异。

1.  即使你添加了数据标签，你也无法从视觉上看出旁遮普语、乌尔都语、葡萄牙语和西班牙语很接近，而波兰语是这些语言的 3 倍。

1.  事实上，因为你想要用 *"k"* 来表示千，那么对于 5300 万英语使用者来说会发生什么？你也用 *"k"* 来标记数据标签吗？

## 场景 2：断裂的条形图——谁发明了这种异常？

我第一次遇到“断裂条形图”是在学校，这是一种我很快学会在专业数据可视化中避免的方法。我希望我没有这样做。在我年轻的时候，这很有道理，所以它一直留在我脑海中，直到我成年后面对一次演示，我带着“wtf”的表情盯着我的断裂条形图。

什么是断裂条形图？嗯，它是一种条形图，你在主导类别上进行了切割。最常见的断裂条形的方法是使用两条平行线。请查看下面的图表。

![这些断裂的线条究竟意味着什么？？？！——来自 AddTwoDigital 的可视化](img/452ea756ed37d8d1355922ffbf5689ad.png)

这些断裂的线条究竟意味着什么？？？！——来自[AddTwoDigital](https://www.addtwodigital.com/add-two-blog/2021/11/1/rule-26-dont-use-broken-axes-or-bars)的可视化

**我认为这个图表有哪些问题？**

1.  最重要的是，它打破了矩形维度和数据值之间的关系。我们该如何想象最大条形中的差距有多大？

1.  断裂的平行线是一个巨大的对比元素。我们的大脑天生就擅长捕捉差异，而这个差异在喊“看我！”因此，我并没有先阅读标题，然后专注于条形之间的差异，而是我的眼睛直接跳到了摩天大楼条形图中的裂缝。

1.  你不知道读者是否可能会将这个故事理解为关于完全不同的事情。例如，他们为什么不会认为这个故事是在强调数据不完整？

## 场景 3：使用子图平衡主导条形图类别

面对这类故事，最好回到数据中，弄清楚你需要绘制什么图表以及为什么。记住，你想要：

1.  展示非英语语言之间的差异。

1.  展示英语仍然占据绝对主导地位的情况。

在这种情况下，我们似乎想要讲述两个故事。因此，我的建议是构建两个图表。子图是数据可视化中的一个基本工具，尤其是在平衡主导条形图类别时。实际上，子图应该是你数据可视化工具包的一部分！下面你可以看到我讲述这两个故事的尝试。

![](img/a12d89407142b81d6aba2729999f467f.png)

**我为什么认为这个图表更好？**

1.  这个图表清楚地划定了两个故事之间的界限。第一个是关于英语与非英语的比较。第二个是关于非英语语言之间的关系。

1.  两个图表的 y 轴范围不同。为了确保读者不会感到困惑，我使用了颜色来确保故事从第一个图表中的“5m”小条形图流畅地过渡到第二个图表。

1.  我故意给第一个图表留出了较窄的绘图空间，而第二个则更宽。

### 如何创建此类图表的技巧

***如何为两个子图提供不同的宽度？***

+   在`make_subplots()`函数中使用`column_widths`参数

```py
fig = make_subplots(rows=1, cols=2, column_widths=[0.3, 0.7])
```

***如何创建不同的图表并将它们添加到子图中？***

+   使用`add_trace()`，创建`go.Bar()`并指定你想要分配给它的`row`和`col`。

```py
fig.add_trace(
        go.Bar(
            x=first_chart_data['language_group_text'],
            y=first_chart_data['Observation'],
            name='main_languages',
            text=[f'{obs / 1e6:.0f}m' for obs in first_chart_data['Observation']],
            texttemplate='%{text}',
            textposition='outside',
            showlegend=False,
            marker_color=['rgb(52, 152, 219)' if lang == 'English' else 'rgb(115, 198, 182)' for lang in first_chart_data['language_group_text']],
        ),
        row=1, col=1
    )

fig.add_trace(
        go.Bar(
            x=second_chart_data['language'],
            y=second_chart_data['Observation'],
            name='other_languages',
            text=[f'{obs / 1e3:.0f}k' for obs in second_chart_data['Observation']],
            texttemplate='%{text}',
            textposition='outside',
            showlegend=False,
            marker_color='rgb(115, 198, 182)',
        ),
        row=1, col=2
    )
```

***如何调整子图的轴美学？***

+   一个简单的方法是在`update_layout()`函数内部定义参数。在这里，你可以调用`yaxis1`和`yaxis2`，Plotly 会按照绘制痕迹的顺序进行循环。

+   例如，`yaxis1`将指代`(row=1, col=1)`中的子图，而`yaxis2`将指代`(row=1, col=2)`中的子图。

```py
fig.update_layout(
        ...
        yaxis1=dict(
            showline=True,
            linecolor='lightgrey',
            linewidth=1,
            showgrid=False,
        ),
        yaxis2=dict(
            showline=True,
            linecolor='lightgrey',
            linewidth=1,
            showgrid=False,
        ),
        xaxis1=dict(
            showline=True,
            linecolor='lightgrey',
            linewidth=1,
            showgrid=False,
        ),
        xaxis2=dict(
            showline=True,
            linecolor='lightgrey',
            linewidth=1,
            showgrid=False,
        ),
        margin=dict(t=100, pad=0),
        height=450,
        width=800,
    )
```

## 总结

总结来说，掌握具有主导类别的条形图需要深思熟虑的视觉化技术，例如子图，以确保大值和小值都能有效地表示。这篇博客演示了为什么单个条形图在这种情况下往往失败，因为占主导地位的“摩天大楼”条形图可能会掩盖较小的值，使得它们之间的相关性差异难以看清。

使用“断裂条”打破条形图会中断数据连续性，过多地吸引观众对断裂本身的注意，并使他们困惑。相反，使用子图是一种更有效的方法：一个图表可以描绘主导类别，而另一个图表则以更详细的细节展示较小的类别。这样，每个子图都可以使用适当的缩放、颜色和空间来清楚地传达“整体图景”和“详细视图”的故事。
