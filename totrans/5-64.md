## TLDR

一些数学定理可以通过组合探索来解决。在本文中，我们关注某些次群的存在性问题。我们将使用[NuCS](https://github.com/yangeorget/nucs)来证明或否定某些次群的存在性。NuCs 是一个 100%用 Python 编写的快速约束求解器，我现在正在将其作为一个副项目开发。它是在[MIT 许可证](https://github.com/yangeorget/nucs/blob/main/LICENSE.md)下发布的。

## 一些定义

让我们先定义一些有用的词汇。

### 群

引用维基百科：

> 在数学中，**群**是一个集合，该集合有一个操作，将集合中的每个元素与集合中每对元素的组合相关联（就像每个二元操作一样），并满足以下约束：操作是结合的，它有一个单位元素，并且集合中的每个元素都有一个逆元素。

整数集（正数和负数）与加法一起构成一个群。有许多种类的群，例如[魔方](https://en.wikipedia.org/wiki/Rubik%27s_Cube)的操作。

### 次群

> 一个有序*m*次群是一个*m×m*的拉丁方阵。也就是说，一个包含*m*个不同符号的乘法表（我们将乘法符号记为∗），其中每个元素在每个行和列中恰好出现一次。

乘法法则不必是结合的。如果是，则次群是一个群。

在本文的其余部分，我们将关注某些特定次群的存在性问题。我们感兴趣的次群是幂等的，即对于每个元素*a*，有***a*∗*a=a***。

此外，它们还具有额外的属性：

+   QG3.m 问题是有序 m 次群，其中**(a∗b)∗(b∗a)=a**。

+   QG4.m 问题是有序 m 次群，其中**(b∗a)∗(a∗b)=a**。

+   QG5.m 问题是有序 m 次群，其中**((b∗a)∗b)∗b=a**。

+   QG6.m 问题是有序 m 次群，其中**(a∗b)∗b=a∗(a∗b)**。

+   QG7.m 问题是有序 m 次群，其中**(b∗a)∗b=a∗(b∗a)**。

在以下内容中，对于一个有序*m*的次群，我们记*0, …, m-1*为次群的值（我们希望这些值与乘法表中的索引相匹配）。

## 模型次群问题

### 拉丁方模型

我们将通过利用拉丁方问题来模型化拟群问题。前者有两种形式：

+   **LatinSquareProblem**，

+   **LatinSquareRCProblem**。

拉丁方问题简单地声明乘法表中所有行和列的值都必须不同：

```py
self.add_propagators([(self.row(i), ALG_ALLDIFFERENT, []) for i in range(self.n)])
self.add_propagators([(self.column(j), ALG_ALLDIFFERENT, []) for j in range(self.n)])
```

此模型定义了对于每一行 *i* 和列 *j* 的单元格值 *color(i, j)*。我们将称之为**颜色**模型。对称地，我们可以定义：

+   对于每一行 *i* 和颜色 *c*，列 *column(i, c)*：我们称之为**列**模型，

+   对于每个颜色 *c* 和 **列** j，行 *row(c,* j)*：我们称之为**行**模型。

注意，我们有以下属性：

+   *row(c, j) = i <=> color(i, j) = c*

对于给定的 __ colum*n* j*，row(., j)* an*d color(., j)* 是逆排列。

+   *row(c, j) = i <=> column(i, c) = j*

对于给定的 __ colo*r* c*，row(c, .)* an*d column(., c)* 是逆排列。

+   *color(i, j) = c <=> column(i, c) = j*

对于给定的 __ ro*w* i*，color(i, .)* an*d column(i, .)* 是逆排列。

这正是拉丁方 RC 问题在 ALG_PERMUTATION_AUX 传播器的帮助下实现的（注意，在关于旅行商问题的[上一篇文章](https://medium.com/towards-data-science/how-to-tackle-an-optimization-problem-with-constraint-programming-9ae77b4d803d)中，也使用了这种传播器的较不优化版本）：

```py
def __init__(self, n: int):
    super().__init__(list(range(n)))  # the color model
    self.add_variables([(0, n - 1)] * n**2)  # the row model
    self.add_variables([(0, n - 1)] * n**2)  # the column model
    self.add_propagators([(self.row(i, M_ROW), ALG_ALLDIFFERENT, []) for i in range(self.n)])
    self.add_propagators([(self.column(j, M_ROW), ALG_ALLDIFFERENT, []) for j in range(self.n)])
    self.add_propagators([(self.row(i, M_COLUMN), ALG_ALLDIFFERENT, []) for i in range(self.n)])
    self.add_propagators([(self.column(j, M_COLUMN), ALG_ALLDIFFERENT, []) for j in range(self.n)])
    # row[c,j]=i <=> color[i,j]=c
    for j in range(n):
        self.add_propagator(([*self.column(j, M_COLOR), *self.column(j, M_ROW)], ALG_PERMUTATION_AUX, []))
    # row[c,j]=i <=> column[i,c]=j
    for c in range(n):
        self.add_propagator(([*self.row(c, M_ROW), *self.column(c, M_COLUMN)], ALG_PERMUTATION_AUX, []))
    # color[i,j]=c <=> column[i,c]=j
    for i in range(n):
        self.add_propagator(([*self.row(i, M_COLOR), *self.row(i, M_COLUMN)], ALG_PERMUTATION_AUX, []))
```

### 拟群模型

现在我们需要为我们的拟群实现额外的属性。

幂等性可以通过以下方式简单实现：

```py
for model in [M_COLOR, M_ROW, M_COLUMN]:
    for i in range(n):
        self.shr_domains_lst[self.cell(i, i, model)] = [i, i]
```

现在，让我们专注于 QG5.m。我们需要实现 *((b∗a)∗b)∗b=a*：

+   这可以翻译为：*color(color(j, i), j) = i,*

+   或者等价地：*row(i, j) = color(color(j, i), j).*。

最后一个表达式表明，*color(j,i)th* 列的 *jth* 元素是 *row(i, j)*。为了强制执行这一点，我们可以利用 ALG_ELEMENT_LIV 传播器（或一个更专业的 ALG_ELEMENT_LIV_ALLDIFFERENT 优化，以考虑行和列包含所有不同的元素）。

```py
for i in range(n):
    for j in range(n):
        if j != i:
            self.add_propagator(
                (
                    [*self.column(j), self.cell(j, i), self.cell(i, j, M_ROW)],
                    ALG_ELEMENT_LIV_ALLDIFFERENT,
                    [],
                )
            )
```

同样，我们可以通过模型化问题 [QG3.m, QG4.m, QG6.m, QG7.m](https://github.com/yangeorget/nucs/blob/main/nucs/examples/quasigroup/quasigroup_problem.py)。
