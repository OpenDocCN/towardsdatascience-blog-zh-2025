# 表格和图像数据中的深度学习异常检测

> 原文：[`towardsdatascience.com/deep-learning-for-outlier-detection-on-tabular-and-image-data-90ae518a27b3/`](https://towardsdatascience.com/deep-learning-for-outlier-detection-on-tabular-and-image-data-90ae518a27b3/)

在过去几年中，深度学习方法已被证明在许多机器学习问题中极为有效，并且不出所料，这包括几个异常检测领域。实际上，对于包括图像、视频和音频在内的许多数据模态，除了基于深度学习的方法外，几乎没有可行的异常检测选项。

同时，尽管如此，对于表格和时间序列数据，更传统的异常检测方法仍然常常是更可取的。这很有趣，因为深度学习通常是一种非常强大的方法，可以解决许多问题（深度学习已经能够解决使用任何其他方法都无法解决的问题），但表格数据尤其证明了将深度学习方法应用于其中非常困难，至少在与其他更成熟的异常检测方法保持一致竞争力的方式上。

在本文（以及下一篇文章——第二篇文章更多地关注表格数据的自监督学习）中，我将探讨为什么基于深度学习的方法对于某些模态的异常检测往往非常有效（具体来看图像数据，但同样的理念也适用于视频、音频和其他一些类型的数据），以及为什么它在表格数据方面可能存在局限性。

此外，我还会讨论几个理由，尽管如此，仍然应该仔细研究表格数据的深度学习异常检测。一方面，这个领域正在快速发展，取得了巨大的进步，我们很可能会在未来几年看到表格异常检测领域的一些最大进展。

另一个原因是，尽管更传统的方法（包括基于 z 分数、四分位数范围、直方图等统计测试，以及隔离森林、k 近邻、局部异常因子（LOF）和 ECOD 等经典机器学习技术）往往更受欢迎，但也有一些例外，甚至在今天，基于深度学习的方法对于表格异常检测可能是最佳选择。我们也将探讨这些内容。

本文继续介绍异常检测系列，涵盖了[子空间](https://medium.com/towards-data-science/perform-outlier-detection-more-effectively-using-subsets-of-features-d984bde99981)、[PCA](https://towardsdatascience.com/using-pca-for-outlier-detection-afecab4d2b78)、[距离度量学习](https://towardsdatascience.com/distance-metric-learning-for-outlier-detection-5b4840d01246)、[共享最近邻](https://towardsdatascience.com/shared-nearest-neighbors-a-more-robust-distance-metric-064d7f99ffb7)、[频繁模式异常因子](https://towardsdatascience.com/interpretable-outlier-detection-frequent-patterns-outlier-factor-fpof-0d9cbf51b17a)、[计数异常检测器](https://towardsdatascience.com/counts-outlier-detector-interpretable-outlier-detection-ead0d469557a)和[doping](https://medium.com/towards-data-science/doping-a-technique-to-test-outlier-detectors-3f6b847ab8d4)的使用。

本文还包含了我书籍《Python 中的异常检测》[Outlier Detection in Python](https://www.manning.com/books/outlier-detection-in-python)的摘录。该书详细介绍了图像数据和基于深度学习的异常检测，但本文提供了对主要思想的良好介绍。

## 基于图像数据的异常检测

如前所述，对于某些数据模式，包括图像数据，目前除了基于深度学习的方法外，没有可行的异常检测选项，因此我们将首先探讨基于深度学习的图像数据异常检测。

本文假设你对神经网络和嵌入的概念有合理的了解。如果不是，我建议阅读一些在线的许多入门文章，并掌握这些知识。不幸的是，我无法在这里提供这些内容，但一旦你对神经网络和嵌入有了足够的理解，你应该能够理解本文的其余部分。

有许多这样的方法，但它们都涉及以某种方式使用深度神经网络，并且大多数方法通过生成嵌入来表示图像。

最常见的基于深度学习的异常检测技术之一是基于自编码器、变分自编码器（VAEs）和生成对抗网络（GANs）。本文将介绍几种异常检测方法，但自编码器、VAEs 和 GANs 是一个很好的起点。

这些是较老、已确立的想法，是异常检测中一个常见主题的例子：工具或技术最初是为了一个目的而开发的，后来发现对异常检测也有效。许多其他例子包括聚类、[频繁项集](https://towardsdatascience.com/interpretable-outlier-detection-frequent-patterns-outlier-factor-fpof-0d9cbf51b17a)、马尔可夫模型、空间填充曲线和关联规则。

由于篇幅限制，本文将仅介绍自动编码器，但将在未来的文章中尝试涵盖 VAEs、GANs 和其他一些内容。自动编码器实际上是一种最初设计用于压缩数据的神经网络形式。（有时也会使用其他压缩算法进行异常值检测。）

与聚类、频繁项集、关联规则、马尔可夫模型等类似，其理念是：我们可以使用某种类型的模型来模拟数据，从而创建数据中主要模式的简洁摘要。例如，我们可以通过描述数据中的聚类（如果数据聚类良好）、数据中的频繁项集、特征之间的线性关系等来模拟数据。使用自动编码器，我们使用原始数据的压缩向量表示来模拟数据。

这些模型通常能够很好地表示数据中的典型项目（假设模型构建良好），但往往无法很好地模拟异常值，因此可以用来帮助识别异常值。例如，在使用聚类（即使用一组聚类来模拟数据时），异常值是不太适合聚类的记录。在频繁项集中，异常值是包含很少频繁项集的记录。在自动编码器中，异常值是压缩效果不佳的记录。

当模型是深度神经网络的形式时，它们的优势在于能够表示几乎任何类型的数据，包括图像。因此，自动编码器（以及其他深度神经网络，如 VAEs 和 GANs）对于图像数据的异常值检测非常重要。

许多异常值检测器也是使用一种称为*自监督学习*（SSL）的技术构建的。这些技术在异常值检测方面的使用可能不如自动编码器、VAEs 和 GANs 广泛，但它们非常有趣，至少应该快速了解一下。以下将介绍这些技术，但首先将探讨使用图像数据进行异常值检测的一些动机。

## 使用图像数据进行异常值检测的动机

一个应用实例是与自动驾驶汽车相关。汽车将配备多个摄像头，每个摄像头可以检测一个或多个物体。系统随后将对图像中出现的每个物体进行预测。这些系统面临的一个问题是，当摄像头检测到物体时，系统预测该物体的类型，可能会预测错误。而且，它可能会错误地预测，但具有很高的置信度；神经网络特别倾向于在最佳匹配时表现出高置信度，即使预测错误，也难以从分类器本身判断系统是否应该对检测到的物体更加谨慎。这种情况最可能发生在所看到的物体与用于训练系统的任何训练示例都不同的情况下。

为了解决这个问题，异常检测系统可以与图像分类系统并行运行，并且当以这种方式使用时，它们通常专门寻找看起来似乎位于训练数据分布之外的物品，这被称为*分布之外的*数据，OOD。

也就是说，任何视觉分类系统都是在一些，可能非常大但却是有限的物体集合上训练的。在自动驾驶汽车的情况下，这可能包括交通信号灯、停车标志、其他汽车、公交车、摩托车、行人、狗、消防栓等等（模型将被训练以识别这些类别中的每一个，并且是在每个类别的许多实例上训练的）。但是，无论系统被训练以识别多少种类型的物品，在道路上行驶时可能会遇到其他类型的（分布之外的）物体，确定系统何时遇到未识别的物体是很重要的。

这实际上是与图像数据中的异常检测的一个常见主题：我们通常非常感兴趣的是识别不寻常的*物体*，而不是不寻常的*图像*。也就是说，像不寻常的照明、着色、摄像机角度、模糊以及其他图像本身的属性通常不太有趣。通常，背景也可能分散我们对识别不寻常物品的主要目标的注意力。虽然有一些例外，但这种情况相当普遍，我们真正感兴趣的是图片中显示的主要物体（或少数相关物体）的本质。

自动驾驶汽车对物体进行错误分类可能是一个相当严重的问题——车辆可能会得出结论，一个新颖的物体（例如，它在训练期间没有看到的车辆类型）是另一种完全不同的物体，可能是与训练期间看到的任何物体类型在视觉上最接近的匹配。例如，它可能会预测这个新车辆是广告牌、电话杆或其他静止的物体。但是，如果并行运行的异常检测器识别出这个物体是不寻常的（并且可能是 OOD），整个系统可以采取更保守和谨慎的方法来处理这个物体，并且可以激活任何相关的安全机制。

异常检测在图像数据中的另一个常见用途是在医学成像中，图像中出现的任何不寻常的物体都可能值得关注并进一步调查。同样，我们并不对图像本身的异常属性感兴趣——只关心图像中的任何物体是否是 OOD：与训练期间看到的任何物体不同（或仅在训练期间很少看到）并且因此是罕见的，可能存在问题。

其他例子包括在安全摄像头中检测不寻常物体出现在哪里，或在监控工业过程的摄像头中。同样，任何不寻常的物体都可能值得注意。

在自动驾驶汽车中，检测 OOD 对象可能允许团队增强其训练数据。在医学成像或工业过程中，任何异常情况都可能是问题的风险。而且，就像汽车一样，仅仅知道我们检测到了 OOD 对象，就允许系统更加保守，不假设分类预测是正确的。

由于在图像中检测 OOD（Out-of-Distribution，分布外）对象对于视觉中的异常值检测至关重要，因此通常进行的训练和测试都与这一点相关。通常，在图像数据中，异常值检测系统是在一个数据集的图像上训练的，而测试则是使用另一个类似的数据库集进行，假设这些图像足够不同，可以被认为是来自不同的分布（并包含不同类型的对象）。这样，就测试了检测 OOD 数据的能力。

例如，训练可能使用一组包含 100 种鸟类的图像集进行，而测试则使用另一组鸟类的图像集。我们通常假设，如果使用不同的图像来源，第二组中的任何图像至少会有所不同，可以被认为是分布外的，尽管标签也可以用来更好地说明这一点：如果训练集包含欧洲金翅雀，而测试集也包含，那么合理地认为这些不是 OOD。

## 自编码器

要具体探讨如何使用神经网络进行异常值检测，我们首先来看一种最实用且直接的方法，即自编码器。在《Python 中的异常值检测》（[Outlier Detection in Python](https://www.manning.com/books/outlier-detection-in-python)）一书中，有更详细的介绍，包括变分自编码器（VAEs）、生成对抗网络（GANs）以及不同软件包中可用的这些技术的变体，但这里将至少介绍一种执行异常值检测的方法。

如前所述，自编码器是一种传统的神经网络形式，通常用作压缩工具，但后来发现它们在异常值检测方面也非常有用。自编码器接收输入并学习以尽可能小的损失压缩这些输入，以便可以重建并接近原始数据。对于表格数据，自编码器一次处理一行，输入神经元对应于表格的列。对于图像数据，它们一次处理一张图像，输入神经元对应于图片的像素（尽管图像也可以以嵌入格式提供）。

下图提供了一个自编码器的示例。这是一种专门设计的神经网络形式，它旨在不预测单独的目标，而是重现输入给自编码器的输入。我们可以看到，网络具有与输入（网络最左侧的神经元，以橙色显示）一样多的元素，以及与输出（网络最右侧的神经元，以绿色显示）一样多的元素，但在两者之间，层中的神经元较少。中间层有最少的神经元；这一层代表每个对象的嵌入（也称为瓶颈或潜在表示）。

具有八个输入神经元和中间层两个神经元的自编码器

中间层的大小是我们尝试压缩所有数据的大小，以便可以在后续层中重新创建（或几乎重新创建）它。创建的嵌入本质上是一个简洁的浮点数向量，可以表示每个项目。

自编码器有两个主要部分：网络的最初几层被称为编码器。这些层将数据逐渐压缩到更少的神经元，直到达到网络的中间部分。网络的第二部分被称为解码器：一组与编码器层对称的层，它们接受每个输入的压缩形式，并尝试尽可能精确地将其重建为原始形式。

如果我们能够训练出一个自编码器，其重建误差通常较低（网络的输出通常与输入非常接近），那么如果某些记录具有高重建误差，它们就是异常值——它们不遵循允许压缩的数据的一般模式。

压缩是可能的，因为通常在表格数据中的特征之间、文本中的单词之间、图像中的概念之间等存在一些关系。当项目典型时，它们遵循这些模式，压缩可以非常有效（损失最小）。当项目不典型时，它们不遵循这些模式，并且无法在不造成更大损失的情况下进行压缩。

层数的数量和大小是建模决策。数据中包含的模式（特征之间的规律性关联）越多，我们能够压缩的数据就越多，这意味着我们可以在中间层使用更少的神经元。通常需要一些实验，但我们希望设置网络的大小，以便大多数记录可以以非常少的错误构建。

如果大多数记录可以无误差地重建，那么网络可能具有过多的容量——中间层能够完全描述传递的对象。我们希望任何异常的记录具有更大的重建误差，但也能够将其与典型记录的适度误差进行比较；如果几乎所有其他记录的误差都是 0.0，那么很难判断一个记录的重建误差有多异常。如果发生这种情况，我们知道我们需要降低模型的容量（减少神经元数量）直到不再可能。实际上，这可以是一种调整自动编码器的实用方法——例如，从中间层有大量神经元开始，然后逐渐调整参数，直到得到你想要的结果。

以这种方式，自动编码器能够为每个对象创建一个嵌入（项目的压缩形式），但我们通常不会在这个自动编码器之外使用嵌入；异常值分数通常完全基于重建误差。

虽然情况并不总是如此。中间层创建的嵌入是对象的合法表示，可用于异常值检测。下方的图示展示了我们使用两个神经元作为中间层的例子，这允许我们将潜在空间绘制为散点图。x 轴代表一个神经元中出现的值，y 轴代表另一个神经元中的值。每个点代表一个对象的嵌入（可能是图像、声音剪辑、文档或表格行）。

然后，任何标准异常值检测器（例如 KNN、隔离森林、凸包、马氏距离等）都可以用于潜在空间。如果限制在两个或三个维度内，这提供了一种可解释的异常值检测系统，但与主成分分析（PCA）和其他降维方法一样，潜在空间本身是不可解释的。

中间层有两个神经元的自动编码器创建的潜在空间示例。每个点代表一个项目。在这个空间中可视化异常值非常直接，尽管这个二维空间本身并不容易理解。

假设我们使用重建误差来识别异常值，为了计算误差，可以使用任何距离度量来衡量输入向量和输出向量之间的距离。通常使用余弦、欧几里得或曼哈顿距离，以及其他一些相当常见的距离度量。在大多数情况下，在执行异常检测之前最好对数据进行标准化，这样既可以允许神经网络更好地拟合，也可以更公平地衡量重建误差。据此，每条记录的异常分数可以计算为重建误差除以中值重建误差（对于某些参考数据集）。

另一种可能更稳健的方法是不使用单个误差度量进行重建，而是使用多个。这使我们能够有效地使用自动编码器为每条记录生成一组特征（每个特征都与重建误差的测量相关），并将其传递给标准的异常检测工具，该工具将找到由一个或多个重建误差度量给出异常大值的记录。

通常，自动编码器可以是一种在数据中定位异常的有效手段，即使在有众多特征且异常复杂的情况下也是如此——例如，对于表格数据，跨越许多特征。自动编码器的一个挑战是，它们确实需要设置架构（网络的层数和每层的神经元数量），以及许多与网络相关的参数（激活方法、学习率、dropout 率等），这可能是困难的。

任何基于神经网络的模型都将必然比其他模型更难以调整。自动编码器的另一个限制是，它们可能不适用于所有类型的异常检测。例如，对于图像数据，它们将在像素级别进行重建测量（至少如果使用像素作为输入），这可能并不总是相关的。

有趣的是，GANs 在这方面可以表现得更好。将 GAN 应用于异常检测的一般方法在某些方面是相似的，但稍微复杂一些。不过，这里的主要思想是，这样的深度网络可以有效地用于异常检测，并且它们对任何数据类型的工作方式相似，尽管不同的检测器会标记不同类型的异常，这些异常可能比其他异常更有或更少兴趣。

## 自监督学习在图像数据中的应用

如所示，自监督学习（SSL）是另一种用于图像数据（以及所有其他类型的数据）的异常检测技术，也值得一看。

如果你习惯于在其他环境中使用深度学习，你可能已经熟悉 SSL 了。在大多数深度学习领域，这相当标准，包括最终用于分类、回归、生成或其他任务的大型神经网络。如果你对大型语言模型有所了解，你很可能熟悉在文本中掩码单词并训练神经网络猜测掩码单词的想法，这是一种 SSL 的形式。

当处理图像时，我们的想法通常是，我们通常拥有一个非常大的图像集合，或者可以轻松在线获取大量图像。在实践中，我们通常会使用一个自身以自监督方式训练的基础模型，但在原则上我们可以自己这样做，无论如何，我们在这里描述的内容大致是创建基础模型的团队所做的事情。

一旦我们拥有大量图像集合，这些图像几乎肯定是没有标签的，这意味着它们不能立即用于训练模型（训练模型需要定义一些损失函数，这需要每个项目的真实标签）。我们需要以某种方式为每张图像分配标签。一种方法是手动标记数据，但这既昂贵又耗时，且容易出错。也有可能使用自监督学习，而且很多时候这更加实用。

在自监督学习（SSL）中，我们找到了一种方法来安排数据，使其能够以某种方式自动标记。如前所述，掩码是一种方法，在训练大型语言模型时非常常见，并且相同的掩码技术也可以用于图像数据。在图像中，我们不是掩码一个单词，而是可以掩码图像的一个区域（如下面杯子图像所示），并训练一个神经网络来猜测掩码区域的图像内容。

![一张杯子图像，其中 3 个区域被掩码。](img/a2e906d8f3f69515e48d0b0966905624.png)

一张杯子图像，其中 3 个区域被掩码。

使用图像数据，还可以实现几种其他自监督学习的技术。

通常，它们基于创建所谓的*代理任务*或*预训练任务*的原则。也就是说，我们训练一个模型在*预训练任务*的背景下预测某些内容（例如图像的缺失区域），尽管我们的实际目标是训练一个能够理解图像的神经网络。我们也可以说，这个任务是这个目标的*代理*。

这很重要，因为没有具体的方法来专门训练异常检测；代理任务是必要的。使用这些方法，我们可以创建一个具有良好图像理解能力的基础模型（足够理解以执行代理任务）。就像语言的基础模型一样，这些模型随后可以被微调以用于其他任务。这可以包括分类、回归以及其他类似任务，也可以是异常检测。

也就是说，以这种方式训练（使用自监督学习创建标签，并在代理任务上训练以预测此标签），可以创建一个强大的基础模型——为了执行代理任务（例如，估计图像被遮挡区域的内容），它需要对其正在处理的图像类型有很强的理解。这也意味着，它可能非常适合识别图像中的异常。

对于异常检测中的 SSL，技巧在于识别良好的代理任务，这些任务允许我们创建我们正在建模的领域的好表示，并允许我们可靠地识别我们拥有的数据中的任何有意义的异常。

使用图像数据，有许多机会来定义有用的先验任务。我们拥有许多其他模态所不具备的巨大优势：如果我们有一个物体的图片，并且以任何方式扭曲该图像，它仍然是该物体的图像。而且，正如所指示的，我们通常对物体感兴趣，而不是图片。这使我们能够在图像上执行许多操作，即使间接地，也能支持我们的最终目标——异常检测。

其中一些包括：旋转图像、调整颜色、裁剪和拉伸，以及图像的其他此类扰动。执行这些变换后，图像可能看起来相当不同，在像素级别上，它*确实是*不同的，但显示的物体是相同的。

这至少为异常检测开辟了两种方法。一种方法是利用这些变换为图像创建嵌入，并将具有不寻常嵌入的图像识别为异常。另一种方法是更直接地使用这些变换。我将在下一节中描述这两种方法。

### 创建嵌入和使用特征建模

有许多方法可以为图像创建嵌入，这些嵌入可能对异常检测有用。我将在这里描述一种称为对比学习的方法。

这利用了这样一个事实：同一图像的扰动版本将代表相同的物体，因此应该具有相似的嵌入。鉴于这一点，我们可以训练一个神经网络，给定同一图像的两个或多个变体，给出这些相似的嵌入，同时为不同的图像分配不同的嵌入。这鼓励神经网络关注每张图像中的主要物体，而不是图像本身，并且对颜色、方向、大小等方面的变化具有鲁棒性。

但是，对比学习仅仅是创建图像嵌入的一种方法，对于任何给定的异常检测任务，许多其他方法，包括任何自监督方法，可能效果最好。

一旦我们为图像创建了嵌入，我们就可以识别具有不寻常嵌入的物体，这些嵌入将是最远离大多数其他嵌入的。为此，我们可以使用嵌入空间中图像之间的欧几里得、余弦或其他距离度量。

在本系列的下一篇文章中，将介绍一个使用表格数据的例子。

### 直接使用预训练任务

使用扰动更直接地识别异常值可能既有趣又非常有效。例如，考虑旋转一张图像。

给定一个图像，我们可以将其旋转 0 度、90 度、180 度和 270 度，从而得到同一图像的四个版本。然后我们可以训练一个神经网络，根据任何图像预测它是否被旋转了 0 度、90 度、180 度或 270 度。正如上面的一些例子（其中异常值可能是那些不太适合聚类的项目，不包含频繁项模式，压缩效果不佳等），这里的异常值是神经网络无法很好地预测图像每个版本旋转程度的图像。

![一张旋转了 0 度、90 度、180 度和 270 度的杯子图像](img/171551c674ab5f2a8c79530708323272.png)

一张旋转了 0 度、90 度、180 度和 270 度的杯子图像

使用典型图像时，当我们通过网络（假设网络已经经过良好训练）传递图像的四种变化（旋转 0 度、90 度、180 度和 270 度），它倾向于正确预测这些图像中的每一个的旋转，但使用非典型图像时，它将无法准确预测，或者对预测的信心较低。

同样的方法也可以用于其他扰动，包括翻转图像、放大、拉伸等 – 在这些例子中，模型预测图像是如何翻转的，图像的缩放比例，或者它是如何拉伸的。

其中一些也可以用于其他模态。例如，掩码可以与几乎任何模态一起使用。但有些则不太通用；例如，翻转可能对音频数据不太有效。

## 使用图像数据进行异常值检测的技术

我在这里简要回顾一下一些最常见的选项：

+   **自编码器、变分自编码器和生成对抗网络**。这些是已经建立并且很可能是最常见的异常值检测方法。

+   **特征建模** – 为每个对象创建嵌入，然后使用标准异常值检测（例如，隔离森林、局部离群因子（LOF）、k-最近邻（KNN）或类似的算法）对这些嵌入进行检测。正如下一篇文章中讨论的，为支持分类或回归问题而创建的嵌入通常在这种情况下不太有效，但我们稍后将会探讨一些与创建更适合异常值检测的嵌入相关的研究。

+   **直接使用预训练任务**。例如，预测图像的旋转、拉伸、缩放等。这是一个有趣的方法，可能是最有效的异常值检测方法之一。

+   **置信度分数** – 在这里，我们考虑的是分类器被使用的地方，以及与所有类别相关的置信度都很低。如果一个分类器被训练来识别，比如说，100 种鸟类，那么当它面对一张新图像时，将为这 100 种鸟类的每一种生成一个概率。如果所有这些概率都非常低，那么这个物体在某种程度上是不寻常的，并且很可能超出了分布。正如所指示的，分类器即使在错误的情况下也经常提供很高的置信度，因此这种方法并不总是可靠的，但即使在这种情况下，当返回低置信度分数时，系统可以利用这一点并认识到图像在某些方面是不寻常的。

## 基于图像数据的神经网络

对于图像数据，我们很好地利用了深度神经网络，可以创建非常复杂的数据模型：我们能够访问极其庞大的数据集，我们可以使用自动编码器、VAEs 和 GANS 等工具，并且自监督学习是相当可行的。

深度神经网络的一个重要特性是它们可以被扩展到非常大的规模，这使得它们能够利用更多的数据并创建更加复杂的模型。

这与传统异常值检测模型（如[Frequent Patterns Outlier Factor (FPOF)](https://medium.com/towards-data-science/interpretable-outlier-detection-frequent-patterns-outlier-factor-fpof-0d9cbf51b17a)、关联规则、k-最近邻、隔离森林、LOF、半径等）形成对比：随着它们在更多数据上训练，它们可能会开发出稍微更准确的正常数据模型，但它们在一段时间后会趋于平稳，并且从额外数据中训练的回报会大幅减少。另一方面，深度学习模型倾向于继续利用更多数据的访问，即使已经使用了大量数据。

然而，我们应该注意的是，尽管在图像异常值检测方面已经取得了很大的进展，但这还不是一个问题。它比其他模态要主观得多，至少在它被定义为严格处理分布外数据的情况下（尽管当物体实际上与训练期间看到的物体不同类型时，它仍然有些模糊——例如，对于鸟类，如果乌鸦和蓝乌鸦是不同的类别）。图像数据处理起来具有挑战性，异常值检测仍然是一个具有挑战性的领域。

## 基于深度学习的异常值检测工具

有几种工具可以用于基于深度学习的异常值检测。其中三个，我们将在本文和下一篇文章中探讨，是 PyOD、DeepOD 和 Alibi-Detect。

[PyOD](https://github.com/yzhao062/pyod) 是我在一些之前的文章中介绍过的，可能是目前 Python 中用于表格数据异常检测最全面的工具。它包含几个标准的异常检测器（隔离森林、局部异常因子、核密度估计 (KDE)、基于直方图的异常检测 (HBOS)、高斯混合模型 (GMM) 以及其他几个），以及基于自动编码器、变分自动编码器、GANs 及其变体的多个深度学习模型。

[DeepOD](https://github.com/xuhongzuo/DeepOD) 为表格和时间序列数据提供异常检测。我将在下一篇文章中对此进行更详细的探讨。

[Alibi-Detect](https://github.com/SeldonIO/alibi-detect) 覆盖了表格、时间序列和图像数据的异常检测。以下是一个使用图像数据的示例。

目前大多数深度学习工作都是基于 TensorFlow/Keras 或 PyTorch（PyTorch 获得了越来越大的份额）。同样，大多数基于深度学习的异常检测都使用其中之一。

根据我的经验，PyOD 可能是这三个库中最直接的一个，但所有这些都非常易于管理且文档齐全。

### 使用 PyOD 的示例

本节展示了使用 PyOD 的 AutoEncoder 异常检测器对一个表格数据集（特别是 KDD 数据集，可公开获取）的示例。

在本例中，我们只需加载数据，将分类列转换为数值格式（这对于任何神经网络模型都是必要的），创建一个自动编码器检测器，拟合数据，并评估数据中的每个记录。

## 图像数据异常检测示例

对于这个例子，我仅提供了一些通用的代码，这些代码可以用于任何数据集，尽管层的维度将需要调整以匹配使用的图像大小。这个例子只是调用一个未定义的方法 load_data() 来获取相关数据（下一个例子将更详细地查看特定数据集——这里我只是展示 Alibi-Detect 使用的通用系统）。

这个例子首先使用 Keras（如果你更熟悉 PyTorch，使用 Keras 的思路是相似的）来创建自动编码器使用的编码器和解码器，然后将这些作为参数传递给 alibi-detect 提供的 OutlierAE 对象。

如同图像数据常见的那样，神经网络包括卷积层。这些层有时也用于其他类型的数据，包括文本和时间序列，尽管很少用于表格数据。它还使用了一个密集层。

代码假设图像大小为 32×32。如果使用其他尺寸，解码器必须组织得使其也能输出这种大小的图像。OutlierAE 类通过比较输入图像与输出图像（在将输入图像通过编码器和解码器处理后）来工作，因此输出图像的大小必须与输入图像相同。当使用 Conv2D 和 Conv2DTranspose 层时，这比使用密集层要复杂一些，如本例所示。

然后我们调用 fit() 和 predict()。对于 fit()，我们指定五个周期。使用更多周期可能效果更好，但也会需要更多时间。Alibi-detect 的 OutlierAE 使用重建误差（特别是从原始图像重建的图像的均方误差）。

```py
import matplotlib.pyplot as plt
import numpy as np
import tensorflow as tf
tf.keras.backend.clear_session()
from tensorflow.keras.layers import Conv2D, Conv2DTranspose, 
    Dense, Layer, Reshape, InputLayer, Flatten
from alibi_detect.od import OutlierAE

# Loads the data used
train, test = load_data() 

X_train, y_train = train
X_test, y_test = test
X_train = X_train.astype('float32') / 255
X_test = X_test.astype('float32') / 255

encoding_dim = 1024

# Defines the encoder portion of the AE
encoder_net = tf.keras.Sequential([ 
      InputLayer(input_shape=(32, 32, 3)),
      Conv2D(64, 4, strides=2, padding='same', activation=tf.nn.relu),
      Conv2D(128, 4, strides=2, padding='same', activation=tf.nn.relu),
      Conv2D(512, 4, strides=2, padding='same', activation=tf.nn.relu),
      Flatten(),
      Dense(encoding_dim,)])

# Defines the decoder portion of the AE
decoder_net = tf.keras.Sequential([ 
      InputLayer(input_shape=(encoding_dim,)),
      Dense(4*4*128),
      Reshape(target_shape=(4, 4, 128)),
      Conv2DTranspose(256, 4, strides=2, padding='same',
          activation=tf.nn.relu),
      Conv2DTranspose(64, 4, strides=2, padding='same', 
          activation=tf.nn.relu),
      Conv2DTranspose(3, 4, strides=2, padding='same', 
          activation='sigmoid')])

# Specifies the threshold for outlier scores
od = OutlierAE(threshold=.015,  
               encoder_net=encoder_net,  
               decoder_net=decoder_net)
od.fit(X_train, epochs=5, verbose=True)

# Makes predictions on the records
X = X_train
od_preds = od.predict(X,
                      outlier_type='instance',    
                      return_feature_score=True,  
                      return_instance_score=True)
print("Number of outliers with normal data:",  
   od_preds['data']['is_outlier'].tolist().count(1))
```

这是在训练数据中使用的行上进行预测。理想情况下，没有异常值。

## 使用 PyTorch 的示例

由于自动编码器创建起来相对简单，因此通常直接创建，也可以使用 Alibi-Detect 或 PyOD 等工具。在这个例子中，我们使用 [MNIST 数据集](https://yann.lecun.com/exdb/mnist/)（以公共许可证提供，在这种情况下与 PyTorch 的 torchvision 一起分发）并展示了使用 PyTorch 的快速示例。

```py
import numpy as np
import torch
from torchvision import datasets, transforms
from matplotlib import pyplot as plt
import torch.nn as nn
import torch.nn.functional as F
import torch.optim as optim
from torchvision.utils import make_grid

# Collect the data
train_dataset = datasets.MNIST(root='./mnist_data/', train=True, 
  transform=transforms.ToTensor(), download=True)
test_dataset = datasets.MNIST(root='./mnist_data/', train=False, 
  transform=transforms.ToTensor(), download=True)

# Define DataLoaders
batchSize=128
train_loader = torch.utils.data.DataLoader(dataset=train_dataset, batch_size=batchSize, shuffle=True) 
test_loader = torch.utils.data.DataLoader(dataset=test_dataset, batch_size=batchSize, shuffle=False)  

# Display a sample of the data
inputs, _ = next(iter(test_loader))
fig, ax = plt.subplots(nrows=1, ncols=10, figsize=(12, 4))
for i in range(10):
  ax[i].imshow(inputs[i][0])
plt.tight_layout()
plt.show()

# Define the properties of the autoencoder
num_input_pixels = 784  
num_neurons_1 = 256  
num_neurons_2 = 64   

# Define the Autoencoder
class Autoencoder(nn.Module):
    def __init__(self, x_dim, h_dim1, h_dim2):
        super(Autoencoder, self).__init__()

        # Encoder
        self.layer1 = nn.Linear(x_dim, h_dim1)
        self.layer2 = nn.Linear(h_dim1, h_dim2)

        # Decoder
        self.layer3 = nn.Linear(h_dim2, h_dim1)
        self.layer4 = nn.Linear(h_dim1, x_dim)

    def encoder(self, x):
        x = torch.sigmoid(self.layer1(x))
        x = torch.sigmoid(self.layer2(x))
        return x

    def decoder(self, x):
        x = torch.sigmoid(self.layer3(x))
        x = torch.sigmoid(self.layer4(x))
        return x

    def forward(self, x):
        x = self.encoder(x)
        x = self.decoder(x)
        return x

model = Autoencoder(num_input_pixels, num_neurons_1, num_neurons_2)
model.cuda()

optimizer = optim.Adam(model.parameters())
n_epoch = 20
loss_function = nn.MSELoss()

for i in range(n_epoch):
    train_loss = 0
    for batch_idx, (data, _) in enumerate(train_loader):
        data = data.cuda()        
        inputs = torch.reshape(data,(-1, 784)) 
        optimizer.zero_grad()

        # Get the result of passing the input through the network      
        recon_x = model(inputs)        

        # The loss is in terms of the difference between the input and
        # output of the model
        loss = loss_function(recon_x, inputs)        
        loss.backward()
        train_loss += loss.item()
        optimizer.step()

    if i % 5 == 0:    
        print(f'Epoch: {i:>3d} Average loss: {train_loss:.4f}')
print('Training complete...')
```

这个例子使用了 cuda，但如果没有 GPU 可用，可以将其移除。

在这个例子中，我们收集数据，为训练数据和测试数据创建一个数据加载器（这在大多数使用 PyTorch 的项目中是这样做的），并展示数据的一个样本，如下所示：

![测试数据的图像样本](img/f17d8f0b95680a03e76dac8dc3ade6bc.png)

为了保持这篇文章的篇幅适中，我在这里结束，并在下一篇文章中继续讨论表格数据。目前，我只是简要回顾一下，上述方法（或它们的变体）可以应用于表格数据，但与表格数据相比，这些技术有一些显著的不同，使得这些技术更加困难。例如，创建嵌入来表示比原始记录更有效的异常值检测的表格记录是困难的。

此外，可能应用于图像的转换通常不太适合表格记录。当我们对一个给定对象的图像进行扰动时，我们可以确信新的图像仍然是同一对象的图像，但当我们扰动一个表格记录（尤其是在没有强大领域知识的情况下），我们无法确信扰动前后的语义是否相同。不过，我们将在下一篇文章中探讨处理表格数据的技巧，这些技巧通常非常有效，以及一些可用于此的工具。

## 结论

深度学习对于包括图像数据在内的多模态异常检测是必要的，并且在其他尚未得到充分建立的领域也显示出希望，例如表格数据。然而，目前，更多传统的异常检测方法对于表格数据仍然效果最佳。

话虽如此，现在有一些情况，基于深度学习的异常检测可能是识别表格数据中异常的最有效方法，或者至少在考虑的方法（以及可能包含在更大的检测器集成中）中是有用的。

深度学习在异常检测中的应用方法有很多，未来几年我们可能会看到更多的发展。其中一些最成熟的方法包括自编码器、变分自编码器和生成对抗网络（GANs），这些方法在主要的异常检测库中都有很好的支持，包括 PyOD 和 Alibi-Detect。

自监督学习在异常检测中也显示出巨大的潜力。我们已经介绍了它如何应用于图像数据，并在下一篇文章中介绍表格数据。它还可以以某种形式应用于大多数模态。例如，对于大多数模态，通常有一些方法可以实现掩码，其中模型学习预测数据中被掩码的部分。例如，对于时间序列数据，模型可以学习预测时间序列中某个范围或一系列范围内的掩码值。

以及本系列的下一篇文章（将涵盖表格数据的深度学习和基于嵌入的异常检测），在未来的文章中，我将尝试继续介绍传统的异常检测，包括表格和时间序列数据，但也会涵盖更多基于深度学习的方法（包括更多异常检测技术，更多现有工具的描述，以及更多其他模态的覆盖）。
