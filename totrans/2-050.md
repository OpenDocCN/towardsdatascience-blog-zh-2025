# 从无数据中估计：从类别中推导出连续分数

> 原文：[https://towardsdatascience.com/estimating-from-no-data-deriving-a-continuous-score-from-categories/](https://towardsdatascience.com/estimating-from-no-data-deriving-a-continuous-score-from-categories/)

<mdspan datatext="el1754947735246" class="mdspan-comment">想象一个国家</mdspan>收集了感染了导致呼吸道传染病的“病原体A”的患者结果数据。每个患者都有8个特征和结果：（a）在家治疗并康复，（b）住院并康复，或（c）死亡。

已经证明，训练一个神经网络从8个特征中几乎完全准确地预测三种结果之一是非常简单的。然而，卫生当局希望预测一些未被捕捉到的东西：对于那些可以在家中治疗的病人，谁是最有可能需要住院的？对于那些预测需要住院的病人，谁是最有可能无法从感染中幸存下来的？我们能否得到一个数值分数，表示感染将有多严重？

在这篇笔记中，我将介绍一个具有瓶颈和特殊头部的神经网络，从几个类别中学习评分系统，并介绍一些可能会遇到的小神经网络的一些属性。相关的代码可以在[https://codeberg.org/csirmaz/category-scoring](https://codeberg.org/csirmaz/category-scoring)找到。

## 数据集

为了能够说明这项工作，我开发了一个玩具示例，这是一个非线性但[确定性代码块](https://codeberg.org/csirmaz/category-scoring/src/tag/v1.0/generate_data.py#L56)，从8个特征计算结果。这个计算只是为了说明——它并不打算忠实于科学；所使用的特征名称仅是为了与医学示例保持一致。本笔记中使用的8个特征是：

+   前期感染过病原体A（布尔值）

+   前期感染过病原体B（布尔值）

+   急性/当前感染病原体B（布尔值）

+   癌症诊断（布尔值）

+   体重偏离平均值，任意单位（-100 ≤ x ≤ 100）

+   年龄，年数（0 ≤ x ≤ 100）

+   血压偏离平均值，任意单位（0 ≤ x ≤ 100）

+   吸烟年数（0 ≤ x ≤ ~88）

在生成样本数据时，特征是独立且从均匀分布中选择的，除了吸烟年数，它取决于年龄，并且建立了一个非吸烟者队列（50%）。我们检查了使用这种采样，三种结果发生的概率大致相等，并测量了吸烟年数的平均值和方差，以便我们可以将所有输入归一化到零均值单位方差。

## 一个经典的分类器

数据是非线性的但非常整洁，因此一个小型分类器网络可以将其学习到98-99%的验证准确率并不令人惊讶。启动`**train.py --classifier**`来训练一个简单的神经网络，具有6层（每层8个神经元）和ReLU激活，定义在`**[ScoringModel.build_classifier_model()](https://codeberg.org/csirmaz/category-scoring/src/tag/v1.0/scoring_model.py#L71)**`中。

## 但如何训练一个评分系统呢？

我们的目标是训练一个系统，给定8个特征作为输入，可以产生一个与患者感染病原体A时的危险程度相对应的评分。复杂性在于，我们的训练数据中没有评分，只有三个结果（类别）。为了确保评分系统有意义，我们希望某些评分范围对应于三个主要结果。

第一个人可能会尝试为每个类别分配一个数值，例如，将0分配给家庭治疗，1分配给住院，2分配给死亡，并将其用作目标。然后设置一个具有单个输出的神经网络，并用例如MSE损失进行训练。

这种方法的问题在于，模型会学会扭曲（压缩和扩展）三个目标周围的输入投影，因此最终模型总是会返回一个接近0、1或2的值。你可以通过运行`**train.py --predict-score**`来尝试，这将训练一个具有2个具有ReLU激活的密集层和单个输出的密集层，定义在`**[ScoringModel.build_predict_score_model()](https://codeberg.org/csirmaz/category-scoring/src/tag/v1.0/scoring_model.py#L88)**`中。

## 第一步：一个低容量网络

为了避免这种情况并得到一个更连续的评分，我们希望大大减少网络扭曲输入的能力。我们将采取极端措施并使用线性回归——在[之前的TDS文章](https://medium.com/data-science/linear-and-logistic-regressions-as-degenerate-neural-networks-in-keras-c9b309cbd80a)中，我已经描述了如何使用Keras提供的组件“训练”一个。我们将重用这个想法——并使用单个密集层（无激活）构建一个“退化”神经网络。这将允许评分更符合输入，而且还有一个优点，即生成的网络高度可解释，因为它简单地为每个输入提供一个权重，而最终的评分是它们的线性组合。

然而，随着这种简化的实施，模型失去了将结果压缩和扩展以匹配每个类别目标分数的所有能力。它将尝试这样做，但特别是在有更多输出类别的情况下，没有任何保证它们会在输入的任何线性组合中以规律的时间间隔发生。

我们希望使模型能够确定类别之间的最佳阈值，即使阈值成为可训练的参数。这就是“类别近似器头”发挥作用的地方。

## 第2步：类别近似器头

为了能够使用类别作为目标来训练模型，我们添加了一个学习根据分数预测类别的头部。我们的目标是简单地建立两个阈值（对于我们的三个类别），t0 和 t1，使得

+   如果分数 < t0，则我们预测在家接受治疗和康复，

+   如果 t0 < 分数 < t1，则我们预测在医院接受治疗和康复，

+   如果 t1 < 分数，则我们预测患者无法存活。

该模型采用编码器-解码器结构，其中编码器部分生成分数，解码器部分允许比较和训练分数与类别。

![展示具有单个输出的密集层，另一个将此扩展到三个输出的密集层和softmax层的神经网络图](../Images/47a538b1bf2075e39d81788cfe9084ac.png)

第二次尝试：线性回归和解码器。图由作者提供

一种方法是在分数之上添加一个密集层，具有单个输入和与类别数量相同的输出。这可以学习阈值，并通过softmax预测每个类别的概率。训练可以使用分类交叉熵损失像往常一样进行。

显然，密集层不会直接学习阈值；相反，它将学习 N 个权重和 N 个偏差，给定 N 个输出类别。因此，让我们弄清楚如何从这些中获取阈值。

## 第3步：提取阈值

注意，softmax层的输出是每个类别的概率向量；预测的类别是概率最高的那个。此外，[softmax](https://en.wikipedia.org/wiki/Softmax_function) 以一种方式工作，它总是将最大的输入值映射到最大的概率。因此，密集层的最大输出对应于它根据传入的分数预测的类别。

如果密集层已经学习到权重 `[w1, w2, w3]` 和偏差 `[b1, b2, b3]`，那么其输出为

`o1 = w1*score + b1 o2 = w2*score + b2 o3 = w3*score + b3`

这些都是作为输入分数的函数的直线（例如，`y = w1*x + b1`），在给定分数时位于顶部的是获胜类别。以下是一个快速说明：

![显示给定x值时哪条线最大的二维图表](../Images/e29a0ca127c007b7dc90dff91198f651.png)

三个线性函数将单个分数映射到每个类别的原始似然。图由作者提供

阈值是相邻线之间的交点。假设类别的顺序为 o1（家庭）→ o2（医院）→ o3（死亡），我们需要解 o1 = o2 和 o2 = o3 的方程，得到

`t0 = (b2 – b1) / (w1 – w2) t1 = (b3 – b2) / (w2 – w3)`

这在 `**[ScoringModel.extract_thresholds()](https://codeberg.org/csirmaz/category-scoring/src/tag/v1.0/scoring_model.py#L183)**` 中实现（尽管那里还有一些额外的逻辑，下面会解释）。

## 第4步：对类别进行排序

但我们如何知道类别的正确顺序呢？显然我们有一个首选顺序（家庭 → 医院 → 死亡），但模型会说什么呢？

值得注意的是，关于代表每个分数下哪个类别获胜的线，有几个需要注意的地方。由于我们感兴趣的是哪条线是最高的，我们谈论的是所有线之上的区域的边界：

![2D图表显示三条线根据在给定x处的最大值着色](../Images/ef5ff0abc126539f26a76d24028904ab.png)

获胜（最大）的线段是高亮凸区域的边界。图由作者提供

由于这个区域是所有在每条线之上的半平面的交集，它必然是凸的。（注意，没有线可以是垂直的。）这意味着每个类别恰好在一个分数范围内获胜；它不能在之后再次回到顶部。

这也意味着这些范围必然按照线的斜率顺序排列，即权重。偏差影响阈值的值，但不影响顺序。我们首先有负斜率，然后是小正斜率，最后是大正斜率。

这是因为对于任意两条线，当向负无穷大方向延伸时，斜率（权重）较小的一条线将获胜，而当向正无穷大方向延伸时，另一条线将获胜。从代数上来说，给定两条线

`f1(x) = w1*x + b1` 和 `f2(x) = w2*x + b2` 其中 `w2 > w1`，

我们已经知道它们在 `(b2 – b1) / (w1 – w2)` 处相交，并且在此以下，如果 `x < (b2 – b1) / (w1 – w2)`，那么

`(w1 – w2)x > b2 – b1` （w1 – w2 是负数！）

`w1*x + b1 > w2*x – b2`

`f1(x) > f2(x)`,

因此 f1 获胜。同样的论点也适用于另一个方向。

## 第4.5步：我们搞砸了（传播求和）

这里就存在一个问题：评分模型可以自由地决定将类别放在什么顺序。这并不好：一个预测死亡为0分，家庭治疗为10分，住院为20分的评分显然是没有意义的。然而，在某些输入下（特别是如果某个特征主导了一个类别），即使是非常简单的评分模型，如线性回归，也可能发生这种情况。

尽管如此，有一种方法可以防止这种情况。Keras允许向密集层添加核约束，以强制所有权重都为非负。我们可以使用这段代码并实现一个核约束，强制权重按递增顺序排列（`w1 ≤ w2 ≤ w3`），但如果我们坚持使用现有工具会更简单。幸运的是，Keras张量支持切片和连接，因此我们可以将密集层的输出拆分为组件（例如，`d1, d2, d3`），并将其用作softmax的输入：

+   `o1 = d1`

+   `o2 = d1 + d2`

+   `o3 = d1 + d2 + d3`

在代码中，这被称为“传播求和”。

![展示两个密集层在编码器-解码器关系后跟随porpagate-sum和softmax操作的神经网络图](../Images/c6f8b3d5c525f0175699e73f0e4d6c6c.png)

最终模型：线性回归和一个强制权重递增顺序的类别近似头（见`build_linear_bottleneck_model`）。图由作者提供

将权重和偏差代入上述公式中，我们得到

+   `o1 = w1*score + b1`

+   `o2 = (w1+w2)*score + b1+b2`

+   `o3 = (w1+w2+w3)*score + b1+b2+b3`

由于`w1, w2, w3`都是非负的，我们现在确保了用于决定获胜类别的有效权重是递增的。

## 第5步：训练和评估

所有组件现在一起用于训练线性回归。该模型在`**[ScoringModel.build_linear_bottleneck_model()](https://codeberg.org/csirmaz/category-scoring/src/tag/v1.0/scoring_model.py#L38)**`中实现，可以通过运行`**train.py --linear-bottleneck**`进行训练。代码还自动在每一轮后提取阈值和线性组合的权重。请注意，作为最终计算，我们需要将每个阈值向编码器层的偏差进行偏移。

[PRE2]

线性回归可以以80%的准确度近似玩具示例，这相当不错。自然地，可达到的最大准确度取决于要建模的系统是否接近线性。如果不是，可以考虑使用更强大的网络作为编码器；例如，几个具有非线性激活的密集层。网络仍然不应该有足够的容量来过度压缩投影得分。

还值得注意的是，与线性组合相比，训练发生的权重空间维度与常规神经网络相比非常小（只是N，其中N是输入特征的数量，与数百万、数十亿或更多相比）。有一个经常描述的直觉是，在高维误差表面上，真正的局部最小值和最大值非常罕见——几乎总是有一个方向，训练可以继续减少损失。也就是说，大多数零梯度的区域是鞍点。在我们的8维权重空间中，我们没有这种奢侈，实际上，即使使用Adam这样的优化器，训练也可能陷入局部极值。尽管如此，训练速度非常快，运行多个训练会话可以解决这个问题。

## 结论

线性回归和其他低容量网络等简单模型在许多应用中具有可取的特性。它们高度可解释且可由人类验证——例如，从上面的玩具示例中我们可以清楚地看到，以前的感染可以保护患者免受更坏的结果，而年龄是确定持续感染严重程度的最重要因素。

线性回归的另一个特性是它们的输出大致与它们的输入保持一致。正是这一特性使我们能够从训练数据中有限信息提供的少数几个锚点中获取一个相对平滑、连续的分数。此外，我们这样做是基于在主要框架（包括Keras）中可用的知名网络组件。最后，我们使用一点数学来从模型的可训练参数中提取所需的信息，并确保学到的分数是有意义的，也就是说，它按照所需的顺序覆盖了结果（类别）。

小型、低容量的模型仍然是解决正确问题的强大工具。通过快速且低成本的训练，它们也可以被实施、测试和快速迭代，非常适合敏捷的开发和工程方法。
