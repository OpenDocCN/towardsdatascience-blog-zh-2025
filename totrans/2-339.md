首先，一些历史。*让我们做交易* 是一个始于 1963 年的美国电视游戏节目。其前提是观众参与者被视为与主持人蒙提霍尔 🎩 做交易的交易者。

问题的核心是一个看似简单的场景：

一个交易者面临着一个问题：从三个门中选择一个，以赢得豪华奖品的机会，例如一辆汽车 🚗。其他两个门后面是山羊 🐐。

保持所选的门 ☝️ 关闭，主持人揭示了一个剩余的门，显示了一只山羊 🐐（我们可以称之为门 C）。

大多数人可能有一种直觉，认为“这无关紧要”，认为在第一种情况下，每个门有 1/3 的机会隐藏奖品，而在主持人干预 🎩之后，当只剩下两个门关闭时，赢得奖品的机会是 50:50。

*一个原则，即简单的解释比复杂的解释更可取* —— 奥卡姆的威廉（1287–1347）

## 肯尼思，频率是多少？

到目前为止，我们从*N*=100 的场景中了解到，换门显然是有益的。对于大多数人来说，推断*N*=3 可能是一种信仰的飞跃。在这里使用一些基本的概率论来量化为什么在任何数量的门场景*N*中换门都是有利的。

我们从标准的蒙提霍尔问题(*N*=3)开始。当它开始时，每个门 A、B 和 C 后面有奖品的概率是*p*=⅓。为了明确起见，让我们定义*Y*参数为有奖品的门 🚗，即*p*(*Y*=A)= *p(Y=*B*)*=*p(Y=*C*)*=⅓。

解决这个问题的技巧是，一旦选择了交易者的门 A ☝️，我们就应该密切关注其他门集合{B,C}，其概率为*p(Y∈{B,C})*=p(Y=B)+p(Y=C)=⅔。这个视觉图可以帮助理解这一点：

![](img/bc30c42601c277071bdd823409b096ea.png)

通过关注{B,C}，其他部分应该随之而来。当揭示山羊 🐐 时

![](img/419c81e8ce205add04bc4c00e5f915f9.png)

显然，干预后的概率发生了变化。请注意，为了便于阅读，我将省略*Y*符号，其中 p(Y=A)将读作 p(A)，*p(Y∈{B,C})*将读作*p(*{B,C}). 此外，由于它是条件性的，干预后的完整术语应该更长，例如，p(*Y*=A|*Z*=C)，*p(Y*∈{B,C}|*Z*=C)，其中*Z*是一个表示主持人选择的参数 🎩。 (在下面的贝叶斯补充部分中，我使用适当的符号而没有这种缩写。)

+   *p*(A)仍然是⅓

+   *p(*{B,C})*=p*(B)+p*(C)仍然是⅔，

+   *p*(C)=0；我们刚刚了解到山羊🐐在门 C 后面，而不是奖品。

+   *p*(B)= *p(*{B,C})-*p*(C) = ⅔

对于任何拥有主持人提供的信息的人来说（意味着交易者和观众），这意味着这不是一个公平硬币的投掷！对他们来说，*p*(C)变为零并不意味着“所有其他船只”都上升（门 A 和 B 的概率），而是*p*(A)保持不变，而*p*(B)翻倍。

重要的是，交易者应该考虑*p(*A*)* = ⅓和*p(*B*)*=⅔，因此通过换门，他们是在翻倍赢得的机会！

让我们推广到*N*（为了使视觉更简单，我们再次使用*N*=100 作为类比）。

当我们开始时，所有门赢得奖品的概率都是*p*=1/*N*。在交易者选择一个门之后，我们将称之为 D₁，意味着*p*(*Y*=D₁)=*1/N*，我们现在应该注意剩余的门集合{D₂, …, Dₙ}将有一个机会*p*(*Y*∈{D₂, …, Dₙ})=*(N-1)/N*。

![](img/15ec92c088dea197b5c6b7422ba05c06.png)

当主持人揭示含有山羊的(*N*-2)个门{D₃, …, Dₙ}时：

+   *p*(D₁)仍然是*1/N*

+   p({D₂, …, Dₙ})=*p*(D₂)+*p*(D₃)+… + *p*(Dₙ)仍然是*(N-1)/N*

+   *p*(D₃)=*p*(D₄)= …=*p*(Dₙ₋₁) =*p*(Dₙ) = 0；我们刚刚了解到它们有山羊，而不是奖品。

+   *p*(D₂)=p({D₂, …, Dₙ}) — *p*(D₃) — … — *p*(Dₙ)=*(N-1)/N*

交易者现在应该考虑两个门值*p*(D₁)=*1/N*和*p*(D₂)=*(N-1)/N*。

因此，获胜的几率提高了 *N-1* 倍！在 *N*=100 的情况下，这意味着获胜比率为 99！（即，换门时赢得奖品的可能性为 99%，如果不换门则只有 1%。）

在 *N*=3 到 100 之间所有场景中，获胜概率比率的提高可以在以下图表中看到。细线是选择任何门之前的获胜概率 p(*Y*)=1/*N*。注意，它也代表了干预后如果他们决定坚持自己的选择而不换门的话获胜的概率 p(*Y=D*₁*|Z={D*₃*…D*ₙ*}*）。（这里我重新引入了之前提到的更严谨的条件形式。）粗线是换门后获胜奖品概率 p(*Y=D*₂*|Z={D*₃*…D*ₙ*}*)=(*N-1)*/*N*：

![](img/a1c2109d7ad2a0e8fd062a92ff82d0ca.png)

获胜概率作为 N 的函数。p(Y)=p(Y=no switch|Z)=1/N 是细线；p(Y=switch|Z)=N/(N-1) 是粗线。（按定义，对于每个 N，两条线的总和为 1。）

也许这个图表（尽管按定义也是）最有趣的一个方面是，*N*=3 的情况在主持人干预 **之前** 具有最高的概率，但在干预 **之后** 的概率最低，反之亦然，对于 *N*=100。

另一个有趣的特点是，换门者的获胜概率迅速上升：

+   *N*=3: p=67%

+   *N*=4: p=75%

+   *N*=5=80%

换门者的曲线逐渐达到一个接近 100% 的渐近线，而 *N*=99 时为 98.99%，*N*=100 时等于 99%。

这开始回答了一个有趣的问题：

> 为什么对于大的 N 来说换门是明显的，但对于 N=3 来说却不是？

答案是这个谜题有点含糊不清。只有高度警觉的人才会意识到，通过揭示山羊（而不是奖品！）主持人实际上传达了 **大量信息**，这些信息应该纳入到自己的计算中。稍后我们将讨论基于直觉在心中进行这种计算与放慢速度，通过把笔放在纸上或编写代码来解决问题的差异。

## 从蒙提霍尔问题中学到的经验

在他的著作《思考，快与慢》中，已故的丹尼尔·卡尼曼，行为经济学的共同创造者，提出了我们有两种思维过程：

+   系统 1 — 快速思考 🐇：基于直觉。这有助于我们快速自信地应对熟悉的情况。

+   系统 2 – 慢速思考 🐢：基于深入思考。这有助于我们弄清楚生活中向我们抛来的新复杂情况。

假设这个前提，你可能已经注意到在上文中你同时应用了两种（思维过程）。

通过观察 *N*=100 门的视觉图，你的系统 1 🐇 立即启动，并立即知道了答案。我猜在 *N*=3 的情况下，你处于系统 1 和 2 之间。考虑到你在进行概率练习时不得不停下来思考，这肯定是系统 2 🐢。

### (1) 评估概率可能具有反直觉性……

或

舒适地转向深入思考 🐢

我们已经清楚地展示了在 *N*=3 的情况下。正如之前提到的，这使许多人感到困惑，包括一些杰出的统计学家。

另一个经典的例子是 *生日悖论* 🥳🎂，它展示了我们如何低估巧合的可能性。在这个问题中，大多数人会认为需要一大群人才能找到一对共享同一天生日的人。结果是，你只需要 23 个人就有 50%的机会，而 70 个人就有 99.9%的机会。

数据分析领域中最令人困惑的悖论之一是辛普森悖论，我在一篇 [之前的文章](https://medium.com/towards-data-science/mastering-simpsons-paradox-my-gateway-drug-to-causality-87e10b613a80) 中详细介绍了它。这是一种趋势在总体中可能在其子群体中逆转的情况。

所有这些悖论的共同之处在于它们要求我们习惯于从系统 1 快速思考 🐇 转向系统 2 慢 🐢。这也是下面概述的教训的共同主题。

一些更经典的例子包括：赌徒谬误 🎲、基础率谬误 🩺 和琳达 [银行出纳员] 问题 🏦。这些超出了本文的范围，但我强烈建议查找它们以进一步磨练关于数据思考的方式。

### (2) ……特别是在处理模糊性时

或

在模糊中寻找清晰 🔎

让我们再次阅读这个问题，这次是按照“问玛丽莲”的表述

*假设你在一个游戏节目中，你有三个门可以选择：其中一个门后面是一辆车；其他门后面是山羊。你选择了一个门，比如说 №1，主持人* ***他知道门后面是什么，打开另一个门****，比如说 №3，门后面有山羊。然后他对你说，“你想要选择门 №2 吗？”改变你的选择对你有利吗？*

我们讨论了最重要的信息没有被明确指出。它说主持人“知道门后面是什么”，但没有说他们随机打开一个门，尽管隐含地理解主持人永远不会打开有车的门。

数据科学中的许多实际问题涉及处理模糊的需求以及利益相关者提供的数据。

对于研究人员来说，追踪任何可能产生影响的相关信息，并将其更新到解决方案中至关重要。统计学家将此称为“信念更新”。

### (3) 在获得新信息时，我们应该更新我们的信念 🔁

这是区分贝叶斯思维流派和频率主义的主要方面。频率主义方法直接接受数据（称为 *平坦先验*）。贝叶斯方法结合先验信念，并在引入新发现时更新它。这在处理模糊情况时特别有用。

为了使这一点更加明确，让我们重新审视这个图，比较干预后的 *N*=3 设置（顶部面板）和 *N*=100 设置（底部面板）。

![图片](img/9c256192473a7a0be67cdace9b715a7f.png)

从上面复制。干预后的 N=3 设置（顶部）和 N=100（底部）。

在这两种情况下，我们事先都认为所有门赢得奖品的概率是相等的，即 *p*=1/N。

当主持人打开了一扇门 (*N*=3；或者当 *N*=100 时的 98 道门) 时，揭示了大量的有价值信息，而在 *N*=100 的情况下，这些信息比 *N*=3 时更为明显。

然而，在频率主义方法中，大部分这些信息都会被忽略，因为它只关注两扇关闭的门。因此，频率主义者的结论是，无论其他关于情况的信息如何，赢得奖品的概率都是 50%。因此，频率主义者采取了保罗·埃尔德什的“没有区别”的观点，我们现在知道这是不正确的。

如果展示的只是两扇门，而不是干预和山羊，这是合理的。然而，如果提供了这些信息，就应该切换到系统 2 思维，并更新系统中的信念。这就是我们通过不仅关注关闭的门，而是考虑关于整个系统的学习所做的事情。

对于勇敢的人 ⚔️，在下面的补充部分称为**贝叶斯观点**中，我使用贝叶斯形式解决了蒙提霍尔问题。

### (4) 与主观性合而为一 🧘

经验主义者在“转向贝叶斯”方面的主要保留意见是——“统计学应该是客观的”。

贝叶斯派的回应是——频率主义者也在不知不觉中应用了一个先验，那就是一个平坦的先验。

不论是贝叶斯派还是频率主义派的争论，作为研究人员，我们尽力在分析的每一步都尽可能客观。

话虽如此，主观决策是不可避免的。

例如，在一个偏斜分布中，应该引用平均值还是中位数？这高度取决于上下文，因此需要做出主观的决定。

分析师的责任是首先为自己的选择提供合理的解释，以说服自己和利益相关者。

### (5) 当感到困惑时——寻找有用的类比

…但需谨慎行事 ⚠️

我们看到，从 *N*=3 设置到 *N*=100 设置，解决方案是显而易见的。这是科学家经常使用的一个技巧——如果问题一开始看起来有点令人困惑/压倒性，就将其分解，并尝试找到一个有用的类比。

这可能不是一个完美的比较，但从 *N*=3 的设置到 *N*=100 的设置，就像是从近距离检查一张图片，然后放大以看到整体画面。想象一下，你只有一片拼图 🧩，然后瞥了一眼盒子上的拼图照片。

### (6) 模拟很强大，但并非总是必要的 🤖

有趣的是，保罗·埃尔德什和其他数学家只有在看到问题的模拟后才会确信。

在解决问题时，我对使用模拟的态度是矛盾的。

一方面，模拟是分析复杂和难以处理问题的强大工具。特别是在现实生活中，人们不仅想要掌握基本公式，还想要了解随机性。

但是，这里有一个大问题——如果一个问题可以像蒙提霍尔问题那样通过分析解决，那么即使模拟很有趣（比如 MythBusters 所做的⁶），可能也不是必要的。

根据奥卡姆剃刀原理，解释现象只需要简短的直觉。这正是我在这里通过运用常识和一些基本的概率推理所尝试做的。对于那些喜欢深入研究的人来说，我下面提供了两个分析方法，一个使用贝叶斯统计，另一个使用因果关系。

## 补充 2：因果观点 ➡️

本节假设对有向无环图（DAGs）和结构因果模型（SCMs）有基本的了解是有用的，但不是必需的。简而言之：

+   有向无环图（DAGs）定性可视化参数节点之间的因果关系。

+   结构因果模型（SCMs）定量地表达了参数之间的公式关系。

给定有向无环图（DAG）

![](img/c5ce4672a86329ea3ef049ef4762c07c.png)

我们将定义与经典 *N*=3 摩纳哥大厅问题相对应的 SCM，并使用它来描述所有变量的联合分布。我们稍后将普遍扩展到 *N*。 (受《入门》教材中问题 **1.5.4** 以及对其 *N* 门问题的简要提及的启发。)

+   *X* — 被选中的门 ☝️

+   *Y* — 奖品 🚗 所在的门

+   *Z* — 主持人 🎩 打开的大门

+   *U = {X,Y}*, *V={Z}*, *F= {f(Z)}*

其中 *X*，*Y* 和 *Z* 有门值：

+   *D = {A, B, C}*

主持人选择 🎩 定义为 *f(Z)*：

df = pd.DataFrame({"X": (["A"] * 9) + (["B"] * 9) + (["C"] * 9), "Y": ((["A"] * 3) + (["B"] * 3) + (["C"] * 3) )* 3, "Z": ["A", "B", "C"] * 9})

df["P(Z|X,Y)"] = None

p_x = 1./3

p_y = 1./3

df.loc[df.query("X == Y == Z").index, "P(Z|X,Y)"] = 0

df.loc[df.query("X == Y != Z").index, "P(Z|X,Y)"] = 0.5

df.loc[df.query("X != Y == Z").index, "P(Z|X,Y)"] = 0

df.loc[df.query("Z == X != Y").index, "P(Z|X,Y)"] = 0

df.loc[df.query("X != Y").query("Z != Y").query("Z != X").index, "P(Z|X,Y)"] = 1

df["P(X, Y, Z)"] = df["P(Z|X,Y)"] * p_x * p_y

print(f"Testing normalisation of P(X,Y,Z) {df['P(X, Y, Z)'].sum()}")
