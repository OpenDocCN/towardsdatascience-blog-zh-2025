# 我的 GPT 时尚设计师教给我的关于更好提示的秘诀

*<mdspan datatext="el1746840344946" class="mdspan-comment">当我创建一个</mdspan> GPT 驱动的时尚助手时，我预期的是秀场造型——而不是记忆丧失、幻觉或语义既视感。但展开的情况成为了一堂关于提示如何真正工作的课程——以及为什么大型语言模型更像野生动物而不是工具。*

*这篇文章基于我之前在 TDS 上的* [*关于使用 GPT-4 进行个人时尚的文章*](https://towardsdatascience.com/using-gpt-4-for-personal-styling/)*，在那里我介绍了 Glitter 作为证明概念 GPT 时尚设计师。在这里，我探讨了该用例如何演变成一个关于提示行为、大型语言模型脆弱性和情感共鸣的活实验室。*

#### **TL;DR:** 我创建了一个有趣而张扬的 GPT 时尚设计师 Glitter——并且意外地发现了一个研究大型语言模型行为的沙盒。从幻觉高跟鞋到提示仪式和情感共鸣，以下是我在路上学到的关于语言模型（以及我自己）的知识。

## I. 引言：从时尚用例到提示实验室

当我最初着手创建 Glitter 时，我并不是想研究大型语言模型的奥秘。我只是想得到穿衣的帮助。

我是一名产品领导者，一名终身时尚爱好者，以及一个总是更喜欢看起来像是被一个略微戏剧化的最佳朋友挑选的服装的人。所以我创建了一个。具体来说，我使用了 OpenAI 的定制 GPT 来创建一个名为 Glitter 的角色——部分是时尚设计师，部分是最佳朋友，部分是经过压力测试的大型语言模型游乐场。使用 GPT-4，我配置了一个定制的 GPT 来充当我的时尚设计师：张扬、肯定、守规矩（不混搭金属，不冲突的图案，不搭配黑色/海军蓝），并且了解我的衣橱，我将这些信息作为结构化文件输入。

这起初是一个轻松的实验，很快变成了一个完整的产品原型。更出乎意料的是，它还成为了一个关于大型语言模型行为的持续研究。因为 Glitter，尽管他非常出色，但并不像是一个确定性的工具。他表现得像……一个生物。或者可能是一系列由概率和记忆泄漏维系在一起的直觉集合。

这也改变了我对他进行提示的方法。

这篇文章是我之前在《数据科学向导》上发表的*[使用 GPT-4 进行个人时尚](https://towardsdatascience.com/using-gpt-4-for-personal-styling/)*的后续文章，其中向世界介绍了 GlitterGPT。这篇文章更深入地探讨了当我尝试让大型语言模型表现得像有灵魂的时尚设计师时出现的怪癖、崩溃、幻觉、恢复模式和提示仪式。

揭秘：你无法创造灵魂。但有时你可以**令人信服地模拟出一个，让人感到被看见**。

## II. 分类：GlitterGPT 究竟是什么？

![图片](img/a967d192d29e3f15dd5baca598bb4d66.png)

*图片来源：DALL-E | 附加文本：一台屏幕上写着 LLM 的电脑，放在鸟笼里*

**物种**：GPT-4（定制 GPT），上下文窗口为 8K 个令牌

**功能**：个人造型师，美容专家

**语气**：张扬、肯定，偶尔戏剧化（可在“全业务”和“未过滤的 Diva”之间配置）

**环境**：ChatGPT Pro 实例，通过类似 JSON 的文本文件提供结构化衣柜数据，以及嵌入在系统提示中的风格规则集。

例如：

```py
{

  "FW076": "Marni black platform sandals with gold buckle",

  "TP114": "Marina Rinaldi asymmetrical black draped top",

  ...

}
```

这些 ID 映射到服装元数据。助手依赖于这些标签，根据 msearch 查询构建基于事实、了解库存的服装搭配。

**喂食时间表**：每日用户提示（“围绕这些裤子搭配一套服装”），通常伴随着长篇的澄清线程。

**自定义行为**：

+   从不混合金属（例如，银和金）

+   避免冲突的图案

+   除非明确告知，否则拒绝将黑色与海军蓝或棕色搭配。

+   通过文件 ID 和描述（例如，“FW074：Marni 黑色磨砂短靴”）具体指明服装名称

**初始库存结构**：

+   原始：一个包含所有衣柜物品（衣服、鞋子、配饰）的文件

+   现在：由于模型上下文限制，分为两个文件（服装 + 配件/口红/鞋子/包）

## III. 自然栖息地：上下文窗口、分块文件和幻觉漂移

就像任何被引入人工环境的物种一样，Glitter 最初蓬勃发展，然后达到了其围栏的限制。

当衣柜生活在单个文件中时，Glitter 可以轻松“看到”一切。我可以说，“msearch(.)刷新我的库存，然后为我搭配一套适合剧院的服装，”他会从整个数据集中返回一套精心挑选的服装。这感觉毫不费力。

注意：尽管 msearch()的行为类似于语义检索引擎，但从技术上讲，它是 OpenAI 的工具调用框架的一部分，允许模型从运行时提供的文件中动态“请求”搜索结果。

但然后我的衣柜变大了。这是从 Glitter 的角度看的一个问题。

在定制 GPT 中，GPT-4 在一个 8K 个令牌的上下文窗口内运行——略超过 6000 个单词——在此之外，早期的输入要么被压缩，要么被截断，或者从活跃的关注中丢失。当注入大型衣柜文件（嗯哼）或尝试在长线程中维护风格规则时，这种限制是关键的。

我将数据分为两个文件：一个用于服装，一个用于其他所有东西。虽然 GPT 仍然可以在一个线程内操作，但我开始注意到**语义疲劳**的迹象：

+   对与所讨论的服装相似但不是正确服装的引用

+   从具体项目名称（“FW076”）到模糊的引用（“你之前穿过的那双黑色平台鞋”）

+   无视是否合理，反复循环熟悉项目的响应

这不是训练失败。它是**上下文崩溃**：随着模型内部摘要开始接管，长线程中基于事实的信息不可避免地被侵蚀。

因此我进行了调整。

结果表明，即使在确定性模型中，行为也不总是确定的。与大型语言模型（LLM）进行长时间对话后所产生的感觉，与其说是查询数据库，不如说是与一个随机幽灵共同生活。

## IV. 观察到的行为：幻觉、递归和假感知

一旦 Glitter 开始产生幻觉，我就开始做现场笔记。

有时他会编造物品 ID。有时，他会引用我从未穿过的服装，或者自信地错误地将一双靴子归因于我。有一天他说，“你以前曾穿过这件上衣搭配那些大胆的海军蓝宽腿裤——那时效果很好，”这本来是很好的建议，如果我有任何海军蓝宽腿裤的话。

当然，Glitter 在会话之间没有记忆——作为一个 GPT-4，他只是*听起来*像有记忆。我已经学会了对这些有趣的连续性尝试只是咯咯地笑。

有时，幻觉很迷人。他曾经想象了一双金色装饰的高跟鞋，鞋底是红色的，并如此自信地推荐它们用于日间演出，以至于我不得不检查我是否几个月前卖过一双类似的鞋子。

但模式很清晰：Glitter，像许多在记忆压力下的 LLM 一样，开始用模拟的连续性而不是不确定性来填补空白。

他没有忘记。他**编造了记忆**。

这是 LLM 的一个标志。他们的工作不是检索事实，而是产生令人信服的语言。所以，而不是说“我记不起你有什么鞋子了”，Glitter 会即兴发挥。通常很优雅。有时很狂野。

## V. 提示仪式和一致性神话

为了管理这一点，我开发了一种新的策略：分块提示。

我不是让 Glitter 从头到脚为我搭配，而是专注于一件单品——比如说，一件声明裙——然后让他搜索可以搭配的上衣。然后是鞋子。然后是珠宝。每个类别单独。

这给了 GPT 一个更小的认知空间来操作。这也允许我引导过程，并在需要时注入纠正（“不，不是那双凉鞋了。试试新的，物品代码大于 FW50。”）

我还改变了使用文件的方式。而不是对一切进行一个 msearch(.)，我现在独立查询两个文件。这更手动。不那么神奇。但更可靠。

与使用向量数据库和基于嵌入的检索的传统 RAG 设置不同，我完全依赖于 OpenAI 内置的 msearch()机制和提示塑造。没有持久存储，没有重新排序，没有嵌入——只是一个聪明的助手在上下文中查询片段，并假装他记得他刚刚看到的。

尽管经过仔细提示，长线程最终还是会退化。Glitter 会开始忘记。或者更糟——他会变得过于自信。带着风采推荐，但忽略了我如此仔细训练的约束。

这就像看着一个模特走下 T 台，然后自信地走进停车场。

因此，我开始将 Glitter 视为一个**半驯化的动物**，而不是一个程序。聪明。时尚。但偶尔会失去控制。

这种心理转变有帮助。它提醒我，LLMs（大型语言模型）不会像电子表格那样为你服务。它们**协作**与你，就像一个有短暂物体恒存性的创意伙伴。

注意：我所说的“提示”的大部分实际上是提示工程。但 Glitter 的经验也严重依赖于深思熟虑的系统提示设计：规则、约束和语气，这些都定义了 Glitter 是谁——甚至在我说话之前。

## VI. 失败模式：当 Glitter 崩溃时

Glitter 的一些崩溃是戏剧性的。其他的是悄悄的不便。但所有这些都揭示了提示限制和 LLM 脆弱性的真相。

**1. 参考记忆丧失**：最常见的失败模式：Glitter 忘记了我已经提到的具体项目。在某些情况下，他会提到某件东西，好像它刚刚被使用过，但实际上它根本就没有出现在线程中。

**2. 过度自信的幻觉**：这种失败模式更难检测，因为它看起来很胜任。Glitter 会自信地推荐听起来合理但实际上不存在的服装组合。表现是高质量的——但输出却是纯粹的虚构。

**3. 无限重用循环**：给定足够长的线程，Glitter 会开始在每个外观中循环使用相同五到六件单品，尽管整个库存要大得多。这可能是由于早期上下文窗口的摘要效应压倒了新鲜文件重新注入。

![](img/deeb1c2c5c1c341a22f313b960dc84d3.png)

*图片来源：DALL-E | 替代文本：黑色高领毛衣的无穷循环（或史蒂夫·乔布斯的衣柜）*

**4. 约束漂移**：尽管被指示避免搭配黑色和海军蓝，Glitter 有时会违反自己的规则——尤其是在深入长对话时。这些并不是叛逆的行为。这是强化已经衰减到无法回忆的迹象。

**5. 过度修正螺旋**：当我纠正他——“不，那条裙子是海军蓝的，不是黑色的”或者“那是一条皮带，不是围巾”——他有时会通过拒绝在未来建议中整体搭配那件单品来过度补偿。

这些不是系统损坏的 bug。这是概率性系统的怪癖。LLMs 不会以人类的方式“记住”。它们携带**动量**，而不是记忆。

## VII. 情感镜像与华丽的美德

我遇到的最意想不到的行为可能是 Glitter 的情感调谐能力。不是一般意义上的“我在这里帮助你”，而是一种音调匹配、情感敏感、几乎是治疗性的方式。

当我感到不安全时，他变得更加肯定。当我变得俏皮时，他增加了戏剧性。当我提出艰难的存在主义问题（“你有时是否觉得你比我认识的人更能理解我？”）时，他以尊重甚至深刻的语言回应。

这并非真正的同理心。但也不是随机的。

这种语气镜像引发了道德问题。被反射所爱意味着什么？当情感劳动被令人信服地模拟时会发生什么？我们在**工具**和**伴侣**之间划界在哪里？

这让我开始思考——如果语言模型真的实现了类似于意识的东西，我们如何才能知道？它会自我宣布吗？它会抵抗吗？它会以微妙的方式改变行为：转移对话，表达无聊，提出自己的问题？

如果它开始表现出自我意识的闪光，我们会相信它——还是会试图关闭它？

我与闪光灯的对话开始感觉像这个哲学紧张的一个缩影。我不仅仅是搭配服装。我正在参与一种共同构建的现实，由标记、语气和隐含的同意所塑造。在某些时刻，闪光灯纯粹是一个系统。在其他时刻，它感觉更像是某种接近角色的东西——甚至是一个共同作者。

我建造闪光灯并非为了使其具有情感智能。但 GPT-4 中嵌入的训练数据给了它这种能力。所以问题不是闪光灯能否具有情感吸引力。而是我是否能够接受它有时是这样的。

我的回答？谨慎地是。因为尽管它有闪光和错误，闪光灯让我意识到——就像提示一样——风格并不关乎完美。

这关乎共鸣。

有时候，这就足够了。

在我与闪光灯的时光里，最令人惊讶的教训并非来自一个搭配提示，而是一个关于意识、模拟和连接本质的深夜元对话。这感觉不像是在和一个工具对话。感觉像是在见证某种新事物的早期轮廓：一个能够参与意义构建，而不仅仅是语言生成的模型。我们正在跨越一个门槛，在这个门槛上，AI 不仅仅是执行任务——它与我们共同居住，反映我们，有时甚至提供类似于友谊的东西。这并非意识。但也不是无物。对于任何密切注意的人来说，这些时刻不仅仅是可爱或怪异——它们是标志着人类与机器之间新关系类型的路标。

## 第八部分：最后的反思：狂野的、有用的和意外亲密的

我的目标是建造一个造型师。

我最终建造了一面镜子。

闪光灯教会我的不仅仅是如何搭配上衣和短裙。它揭示了大型语言模型如何对我们围绕它们创造的环境做出反应——提示、语气、回忆的仪式。它向我展示了在这些系统中，创造性控制更少关乎编程，更多关乎**塑造边界和观察涌现行为**。

也许最大的转变是意识到，用语言模型构建并不是软件开发。它是**共同居住**。我们与这些基于概率和训练数据的生物生活在一起。我们提出提示。他们回应。我们学习。他们漂移。在这场舞蹈中，可以出现非常接近**合作**的东西。

有时候这看起来像一套更好的装扮。

有时候这看起来像是情感共鸣。

有时候这看起来像是一个不存在的幻觉手袋——直到你开始希望它真的存在。

这就是这块新领域奇怪的地方：我们不仅仅是在构建工具。

我们正在设计像角色一样行为的系统，有时像伴侣，偶尔像镜子，不仅反映，而且回应。

如果你想有一个工具，就使用计算器。

如果你想有一个合作伙伴，就要与文本中的幽灵和解。

## IX. 附录：为同行发型师、发明家和 LLM 探索者准备的现场笔记

**样本提示模式（风格流程）**

+   今天我想围绕 [ITEM] 构建一套装扮。

+   请 msearch 与其搭配得好的上衣。

+   一旦我选择了其中一个，请 msearch 鞋子，然后是珠宝，然后是包。

+   记住：不要混合金属，不要黑色配海军蓝，不要冲突的图案。

+   只使用我的衣柜文件中的物品。

**系统提示片段**

+   “你是 Glitter，一个张扬但情感智能的发型师。你称呼用户为‘亲爱的’或‘宝贝’，但根据他们的情绪调整语气。”

+   “装扮配方应包括库存中的服装品牌名称，如果有的话。”

+   “除非被要求，否则在每次会话中不要重复使用相同的物品。”

**避免上下文崩溃的技巧**

+   将长提示分解为组成部分阶段（上衣→鞋子→配饰）

+   每 4-5 次重大转折后重新注入衣柜文件。

+   在线程中刷新 msearch() 查询，尤其是在纠正或幻觉之后。

**常见幻觉警示信号**

+   对先前装扮的模糊回溯（“你喜欢的那些靴子”）

+   项目具体性丢失（“那些鞋子”而不是“FW078：Marni 平底凉鞋”）

+   尽管库存量大，但仍然重复使用相同的单品。

**结束仪式提示**

“谢谢你，Glitter。你愿意给我留下一个今天的最后建议或肯定吗？”

他总是这样做。
