## 理解数据稀缺与不平衡

机器学习，尤其是深度学习，依赖于拥有大量数据集来工作。没有足够的数据，这些模型难以有效地学习模式——尤其是那些罕见的关键模式（我稍后会详细说明）。

但现实是：收集我们需要的庞大数据集通常**极其**具有挑战性，尤其是在处理缺乏历史数据的较新系统或组件时。

现在，有些人可能想知道…

> 为什么数据稀缺和不平衡很重要？

这很简单——这些挑战对模型预测和预防关键事件的能力有重大影响。在深入探讨解决这些问题的技术（本文的重点）之前，我会用几个例子来说明解决这些问题为什么如此重要！

### **医疗示例：淋巴癌**

在医疗保健领域，收集足够的数据本身就具有挑战性，而且可用的数据通常是不平衡的。

> 你能猜到为什么吗？

**例如，考虑淋巴癌。**由于以下几个原因，对这种罕见疾病的数据收集受到限制：

+   **隐私问题：**对敏感患者信息的访问受到限制，限制了数据集的范围。

+   **疾病罕见性：**罕见疾病与更常见条件相比，自然会产生更少的数据点。

因此，数据集通常由健康病例主导，导致数据不平衡。当数据集不平衡——一个类别主导另一个类别时——模型往往会偏向多数类别。这意味着在这样数据上训练的诊断模型可能会不成比例地预测“健康”，导致漏诊。你可以想象，在医疗环境中，这可能会产生严重的后果。

### **工业示例：核电站**

工业部门面临类似的问题。以核电站为例，设备故障数据对于预测性维护至关重要。这些故障很少见（谢天谢地！）但正是这种罕见性使得积累用于训练模型的稳健数据集变得困难。

理解可能表明设备故障的模式对于避免昂贵的停机时间至关重要。然而，故障数据的稀缺意味着收集足够的信息来训练有效的模型需要大量的时间，如果可能的话。

例如，Ali Hakami 发表在《自然》杂志上的一项研究，使用 Kaggle 数据集分析了生产厂数据，**显示健康到故障的观察比率为 28,552:1**。这种惊人的不平衡实际上显示了在现实世界的应用中，由于故障数据如此有限，开发可靠的预测模型是多么困难。

### **我们如何应对这些挑战？**

等待更多数据并不总是可行的，这既昂贵又耗时。有道理吧？当我们需要立即解决方案时，我们不想等待几年去收集数据。然而，简单地复制现有数据也不会有帮助，因为它不会引入新的信息，可能会导致过拟合。

> 相反，如果我们生成*合成数据，或称为“数据增强”*呢。

你可能听说过“数据增强”，对吧？数据增强简单来说就是利用现有数据创建新的、人工的数据点。通过根据现有数据中的模式创建人工数据点，我们可以解决由稀缺和不平衡造成的差距。

### 进入 GANs 和 SMOTE

生成对抗网络（GANs）和合成少数过采样技术（SMOTE）是两种数据增强技术，有助于克服这些挑战。它们能够创建新的、真实的数据点，减少对昂贵的数据收集的依赖，并提高模型性能。

+   **GANs**：通过学习原始数据集的分布并创建新的、真实的数据示例来生成合成数据。

+   **SMOTE**：通过生成连接少数类样本的线段上的合成示例来专注于过采样。

## 介绍生成对抗网络（GANs）

“生成合成数据”正是**生成对抗网络（GANs）**发挥作用的地方。GANs 因其能够通过生成**与真实数据集非常相似的合成数据**来解决数据稀缺问题而在各个行业中受到广泛关注。

从高层次来看，**生成对抗网络（GANs）**是一个机器学习框架，旨在创建与给定数据集非常相似的新合成数据。这种能力在现实世界数据有限、不平衡或难以获取的情况下非常有价值。

GANs 由两个核心组件组成：

1.  **生成器**

1.  **判别器**

每个这些组件都有一个非常具体的作用。

### 1. 生成器

生成器的角色是产生合成数据。它以随机噪声（一组随机值）作为输入，并生成类似于真实数据的数据样本。

初始时，生成器产生无意义的随机输出。然而，通过迭代训练，它改进了创建合成数据的能力，使其越来越难以与真实世界数据区分。

### 2. 判别器

判别器充当一个简单的二进制（0，1）分类器。它被训练来区分训练集中的真实数据和生成器产生的虚假数据。

它的任务是将输入数据正确分类为“真实”或“虚假”。随着训练的进行，判别器在分类任务上变得越来越擅长，提高了它区分这两种数据类型的能力。

## 以酒吧为主题的 GANs 解释

我不知道你们，但我喜欢时不时地和朋友们一起去酒吧喝酒。你喜欢在酒吧喝酒吗？如果你不喜欢……那可真不幸，因为我将使用这个场景作为理解 GANs 的类比。

**想象一位刚开始学习技巧的新调酒师**。起初，他们的饮料混合得不好，味道远非你所期望（随机输出）。然而，在经验丰富的调酒师的指导和反馈下，他们逐渐提高了自己的技能。随着时间的推移，他们掌握了技艺，开始制作出与经验丰富的调酒师制作的饮料难以区分的饮料。

> **在这个场景中，新调酒师就像 GAN 中的生成器**。

他们的任务是创造出可以以假乱真的东西。

但作为一个顾客，你不会想为新手调酒师的劣质饮料付钱，对吧？你更希望你的钱能用来购买高质量的鸡尾酒。那么，你如何判断调酒师是新手还是专家呢？

> 这就是鉴别器发挥作用的地方。

将鉴别器想象成酒吧的忠实顾客——就是你！你的任务是品尝饮料并决定它们是否制作得很好（真实）或者是一次糟糕的尝试（伪造）。

生成器（新调酒师）和鉴别器（你，这位忠实顾客）不断相互挑战。随着生成器在制作逼真饮料方面变得更好，鉴别器必须更加努力地分辨差异。这种轻松的来回互动是 GANs 如此有效的原因。

### 生成器和鉴别器之间的爱恨情仇

正如示例中提到的，GANs 的 brilliance 在于它们的对抗性训练过程，这在一对生成器和鉴别器之间建立了一个动态的“游戏”。

**把它想象成一场轻松的竞争**：生成器就像一个新手调酒师，试图将他的作品伪装成精心制作的饮料，而鉴别器则是一位经验丰富的鸡尾酒鉴赏家，决心找出任何瑕疵并指出冒牌货。

+   **生成器的目标是产生足够令人信服的合成数据，以欺骗鉴别器**。

+   **鉴别器的任务是精确地区分真实数据和伪造数据**。

**随着这个游戏的展开**，生成器不断改进其创建真实数据的能力，而鉴别器则越来越擅长识别什么是真实的，什么是伪造的。这个迭代过程一直持续到生成的合成数据几乎无法与真实数据集区分开来，在这两个对手之间达到了微妙的平衡！

## GANs 的实际应用

在预测性维护（如发电厂）中，GANs 可以生成设备的合成故障数据，使模型能够在真实故障数据稀缺的情况下学习故障模式。同样，在医疗保健中，GANs 可以创建模拟罕见病例的合成患者数据，有助于解决数据集中的不平衡问题。

## 介绍合成少数过采样技术（SMOTE）

与生成对抗网络（GANs）一样，**SMOTE（合成少数过采样技术）**也是一种流行的方法，用于通过生成新的合成样本来解决不平衡数据集的问题。

SMOTE 通过一种称为**插值**的过程为少数类创建新的数据点（尽管这个术语可能对你们中的许多人来说很熟悉，但不要让它吓到你）。插值简单地说就是比较彼此接近的数据点，并基于它们的特征生成新的数据。

好的，这句话实际上可能定义了 SMOTE 的功能。但我知道这个定义并不令人满意，所以我会给出一个具体的例子，然后是技术解释。

## SMOTE 如何工作（以调酒师为例）

**想象你是一名调酒师。**你已经注意到，顾客几乎都点了一种饮料，比如“健康朗姆酒配一些羽衣甘蓝”（多么不寻常的选择），而只有少数人点了一种稀有的特色饮料（我们可以称之为“淋巴癌鸡尾酒”）。你想要创造更多稀有关键饮料的变体来推广它，并平衡菜单。

> 这是你要做的

1.  **选择少数样本**：从一个稀有饮料的食谱开始。

1.  **寻找最近邻**：查看类似特色饮料的成分。

1.  **生成合成变体**：混合搭配食谱及其邻居的成分比例，以创建稀有饮料的新、独特的变体。

1.  **重复**：继续创建变体，直到菜单在常见和稀有选项之间平衡。

**稀有选项现在不再那么稀有了！**这正是 SMOTE 想要实现的效果。

## SMOTE 如何工作（技术层面）

正如你在调酒师示例中看到的，SMOTE 的工作方式很容易理解。它被分为四个快速直观的步骤。

### **1. 选择少数类样本**

这相当简单。首先从少数类中随机选择一个数据点。

### 2. **寻找最近邻**

实质上，你想要查看整个数据集并识别所选样本的 k 个最近邻（我们称之为特征空间）。通常，k 设置为 5，但可以根据你的数据集和模型需求进行调整！

### **3. 生成合成样本**

随机选择 k 个最近邻中的一个。使用插值，创建一个位于所选样本及其邻居之间的新合成样本。

![新样本的方程式](img/6256586b2531f40eecd4a44424c71d06.png)

新样本的方程式

你可以将 xᵢ视为原始少数样本，λ为介于 0 和 1 之间的随机数。

### 4. 重复

继续这个过程，直到数据集在多数类和少数类之间达到所需的平衡。

## 为什么 SMOTE 有效？

如果你坚持到这里，你可能已经理解了为什么 SMOTE 如此有效！简单来说，它生成**新的、有意义的**数据点，而不是简单地复制现有的数据点。

由于复制数据可能导致模型记住模式而不是学习它们，SMOTE 降低了这种风险。此外，通过解决数据不平衡问题，SMOTE 使模型能够专注于少数类模式，提高其预测罕见情况的能力。

### 示例：医疗保健

考虑一个关于淋巴癌的数据集，其中健康患者的数据远多于患病患者的数据。使用 SMOTE，我们可以创建与淋巴癌真实世界案例非常相似的合成患者档案。这些合成案例填补了数据集中的空白，使模型能够学习与疾病相关的微妙模式，并提高其对罕见情况的预测准确性。

例如，SMOTE 可能在两个相似的癌症病例之间进行插值：

+   患者 A：年龄 60 岁，肿瘤大小 3.2 厘米

+   患者 B：年龄 62 岁，肿瘤大小 3.5 厘米

SMOTE 可以创建一个新的合成患者：

+   患者 C：年龄 61 岁，肿瘤大小 3.35 厘米

### 示例：核电站

在预测性维护中，核电站的故障幸运地很少发生。然而，这种罕见性使得在故障模式上训练模型变得困难。SMOTE 通过基于现有故障记录及其最近邻生成合成故障数据来解决这一问题。

例如，如果我们有两个故障事件：

+   事件 A：温度峰值 500°C，振动水平 8.0

+   事件 B：温度峰值 520°C，振动水平 8.5

SMOTE 可能会创建一个合成的故障：

+   事件 C：温度峰值 510°C，振动水平 8.25

### SMOTE 的局限性

然而，虽然 SMOTE 非常有用，但它并非没有挑战。我会简要说明这一点，让你知道所有框架和技术都有其独特的局限性。对于 SMOTE 来说，这一点非常明显。

1.  **计算成本高**：对于大型数据集，寻找 k 个最近邻可能变得耗时，从而减慢了处理过程。

1.  **类别边界问题**：SMOTE 在生成合成样本时没有考虑类别之间的边界。这有时会导致重叠或不相关的区域出现样本，从而降低模型的有效性。

对于那些对简单的 Python 实现感兴趣的人，看这里！

```py
from imblearn.over_sampling import SMOTE
from sklearn.model_selection import train_test_split
from sklearn.datasets import make_classification

# Create an imbalanced dataset
X, y = make_classification(n_classes=2, class_sep=2, weights=[0.9, 0.1], 
                           n_informative=3, n_redundant=1, flip_y=0, 
                           n_features=5, n_clusters_per_class=1, 
                           n_samples=1000, random_state=42)

# Apply SMOTE
smote = SMOTE(random_state=42)
X_resampled, y_resampled = smote.fit_resample(X, y)

# Print class distribution before and after SMOTE
from collections import Counter
print("Before SMOTE:", Counter(y))
print("After SMOTE:", Counter(y_resampled))
```
