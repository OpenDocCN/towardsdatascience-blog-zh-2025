# 从 FOMO 到机遇：LLM 代理时代的分析型 AI

> 原文：[`towardsdatascience.com/from-fomo-to-opportunity-analytical-ai-in-the-era-of-llm-agents/`](https://towardsdatascience.com/from-fomo-to-opportunity-analytical-ai-in-the-era-of-llm-agents/)

<mdspan datatext="el1745984322790" class="mdspan-comment">你对 LLM 代理感到</mdspan>“错失恐惧症”（FOMO）吗？嗯，这对我来说已经持续了一段时间。

在最近几个月里，感觉我的在线动态完全被“LLM 代理”轰炸了：每隔一篇技术博客都在尝试展示“如何在 5 分钟内构建一个代理”。每隔一条科技新闻都在突出另一个闪亮的初创公司正在构建基于 LLM 代理的产品，或者一个大科技公司发布了一些新的代理构建库或花哨命名的代理协议（是否已经看够了 MCP 或 Agent2Agent？）。

突然间，似乎 LLM 代理无处不在。那些炫目的演示展示了这些数字巨兽似乎完全能够编写代码、自动化工作流程、发现洞察，并且似乎威胁到取代……好吧，几乎一切。

不幸的是，这种观点也得到我们工作中许多客户的认同。他们正在积极要求将代理功能集成到他们的产品中。他们并不犹豫为新的代理开发项目提供资金，因为担心在利用这项新技术方面落后于竞争对手。

作为一名分析型 AI 从业者，看到我的同事构建的令人印象深刻的代理演示以及客户的热烈反馈，我必须承认，这让我患上了严重的 FOMO。

这真的让我感到困惑：我从事的工作是否变得无关紧要了？

在挣扎了这个问题之后，我得出以下结论：

**不，情况根本不是这样。**

在这篇博客文章中，我想分享我对为什么 LLM 代理的快速崛起并没有减少分析型 AI 重要性的看法。事实上，我相信它正在做相反的事情：它为分析型 AI 和代理型 AI 创造了前所未有的机遇。

让我们来探讨一下原因。

> 在深入探讨之前，让我们快速澄清一下术语：
> 
> +   **分析型 AI**：我主要指的是应用于定量、数值数据的统计建模和机器学习方法。想想工业应用，如异常检测、时间序列预测、产品设计优化、预测性维护、数字孪生等。
> +   
> +   **LLM 代理**：我指的是使用 LLM 作为核心的 AI 系统，可以通过结合自然语言理解、推理、规划、记忆和工具使用来自主地执行任务。

* * *

> **观点 1：分析型 AI 为 LLM 代理提供了至关重要的定量基础。**

尽管在自然语言理解和生成方面具有显著的能力，但 LLM 在许多工业应用中基本缺乏所需的定量精度。这就是分析型 AI 变得不可或缺的地方。

分析人工智能可以通过一些关键方式提升自己，用数学严谨性为 LLM 代理提供基础，并确保它们按照现实运行：

## 🛠️ 分析人工智能作为基本工具

将分析人工智能作为专门的、可调用的工具集成，可以说是为 LLM 代理提供定量基础的最常见模式。

在 LLM 的当前炒作之前，就已经有了一种传统，即开发各种行业的专业分析人工智能工具，以使用现实世界的运营数据来解决挑战。这些挑战，无论是预测设备维护还是预测能源消耗，都需要高数值精度和复杂的建模能力。坦白说，这些能力与今天 LLM 所具有的语言和推理优势在本质上不同。

这项长期以来的分析人工智能基础不仅相关，而且是至关重要的，因为它为将 LLM 代理扎根于现实世界的准确性和操作可靠性提供了基础。这里的核心动机是**关注点的分离**：让 LLM 代理处理理解、推理和规划，而分析人工智能工具执行它们被训练的专业定量分析。

在这个范例中，分析人工智能工具可以扮演多个关键角色。首先，它们可以用分析超级力量增强代理的能力，这些力量代理本身缺乏。此外，它们可以**验证代理的输出/假设**与真实数据和学到的模式。最后，它们可以**强制物理约束**，确保代理在现实可行的空间中运行。

以一个具体的例子来说明，想象一个被分配优化复杂半导体制造工艺以最大化产量并保持稳定的 LLM 代理。该代理不是仅仅依赖于文本日志/操作员笔记，而是持续与一系列专业的分析人工智能工具互动，以实时获得定量、丰富的过程理解。

例如，为了实现其高产量目标，代理查询一个预训练的**XGBoost 模型**，根据数百个传感器读数和过程参数预测可能的产量。这使代理能够预见质量结果。

同时，为了确保过程稳定性以保持一致的质量，代理调用一个**自动编码器模型**（在正常过程数据上预训练的）来识别偏差或潜在的设备故障，*在*它们干扰生产之前。

当异常检测模型指示可能出现问题时，代理必须以最佳方式执行航向校正。为此，它调用一个**基于约束的优化模型**，该模型采用*贝叶斯优化*算法来推荐对过程参数的最佳调整。

在这个场景中，LLM 代理本质上充当智能协调者的角色。它解释高级目标，规划对适当的分析人工智能工具的查询，对它们的定量输出进行推理，并将这些复杂分析转化为操作员或甚至触发自动化调整的可操作见解。这种合作确保了 LLM 代理在解决复杂、现实世界工业问题时保持接地和可靠。

## 🪣 分析人工智能作为数字沙盒

除了作为可调用的工具外，分析人工智能还提供了另一个关键能力：创建真实的**模拟环境**，在 LLM 代理与物理世界互动之前，它们可以在这些环境中接受训练和评估。这在工业环境中尤其有价值，因为失败可能导致严重后果，如设备损坏或安全事故。

分析人工智能技术通过从工业资产或过程的历史操作数据和控制物理方程中学习，能够高度有效地构建高保真度的表示（例如，考虑像物理信息神经网络这样的方法）。这些*数字孪生*捕捉了基本的物理原理、操作约束和固有的系统可变性。

在这个由分析人工智能驱动的虚拟世界中，一个 LLM 代理可以通过首先接收模拟的传感器数据，决定控制动作，然后观察由分析人工智能模拟计算的系统响应来训练。因此，代理可以在更短的时间内迭代许多试错学习周期，并安全地接触到各种真实的操作条件。

除了代理训练外，这些由分析人工智能驱动的模拟提供了一个受控的环境，用于在现实世界部署之前严格**评估和比较**不同代理设置版本或控制策略的性能和鲁棒性。

以一个具体的例子来说明，考虑电网管理案例。一个旨在优化可再生能源整合的 LLM 代理（或多个代理）可以在由多个分析人工智能模型驱动的模拟环境中进行测试：我们可以有一个**物理信息神经网络**（PINN）模型来描述复杂的动态电力流动。我们可能还会有概率预测模型来模拟真实的天气模式及其对可再生能源的影响。在这个丰富的环境中，LLM 代理可以学习在各种天气条件下制定复杂的决策策略，以平衡电网，而无需冒实际服务中断的风险。

事实上，没有分析人工智能，这一切都不可能实现。它构成了定量基础和物理约束，使得安全有效的代理开发成为现实。

## 📈 分析人工智能作为操作工具包

现在，如果我们退一步，换一个全新的视角，**LLM 智能体——甚至是一组 LLM 智能体——难道不是另一种运营系统，需要像任何其他工业资产/流程一样进行管理吗**？

这实际上意味着：系统设计、优化和监控的所有原则仍然适用。而且你知道吗？分析型人工智能正是为此而生的工具箱。

再次强调，分析型人工智能（Analytical AI）有潜力将我们带出经验试错（当前实践）的范畴，并朝着管理智能体系统时更加*客观*、*数据驱动*的方法迈进。那么，使用**贝叶斯优化算法**来设计智能体架构和配置如何？采用**运筹学技术**来优化计算资源的分配或高效管理请求队列如何？或者使用**时间序列异常检测**方法来警告智能体的实时行为？

将 LLM 智能体视为一个受定量分析约束的复杂系统，开辟了许多新的机会。正是分析型人工智能带来的这种运营严谨性，才能使这些 LLM 智能体从“仅仅是一个演示”提升到在现代工业运营中可靠、高效且“真正有用”的层面。

* * *

> **观点 2：分析型人工智能可以通过具有情境智能的 LLM 智能体得到增强**。

我们已经详细讨论了分析型人工智能对于 LLM 智能体生态系统的重要性。但这个强大的协同效应是双向的。分析型人工智能也可以利用 LLM 智能体的独特优势来增强其可用性、有效性和最终的实际影响。这些都是分析型人工智能从业者可能不想错过的 LLM 智能体的要点。

## 🧩 从模糊的目标到可解决的问题

通常，分析的需求始于一个高级别的、模糊陈述的业务目标，比如“我们需要提高产品质量。”为了使这一目标可操作，分析型人工智能从业者必须反复提出澄清问题，以揭示真正的目标函数、具体约束和可用的输入数据，这不可避免地导致一个非常耗时的工作流程。

好消息是，LLM 智能体在这方面表现出色。它们可以解释这些模糊的自然语言请求，提出澄清问题，并将它们制定成分析型人工智能工具可以直接处理的良好结构化、定量问题。

## 📚 通过上下文和知识丰富分析型人工智能模型

传统的分析型人工智能模型主要在数值数据上运行。对于大量未被挖掘的非结构化数据，LLM 智能体可以非常有助于那里提取有用的信息，以供定量分析使用。

例如，LLM 代理可以分析文本文档/报告/日志以识别有意义的模式，并将这些定性观察转化为分析 AI 模型可以处理的定量特征。这一**特征工程**步骤通常通过让它们访问嵌入在非结构化数据中的见解（否则它们可能会错过）来显著提高分析 AI 模型的性能。

另一个重要的用例是**数据标注**。在这里，LLM 代理可以自动生成准确的类别标签和注释。通过提供高质量的训练数据，它们可以极大地加速高性能监督学习模型的发展。

最后，通过利用 LLM 代理的**知识**，无论是预先在 LLM 中**预训练**的还是在外部数据库中**主动搜索**的，LLM 代理可以自动化复杂分析流程的设置。LLM 代理可以根据问题特征[1]推荐合适的算法和参数设置，生成实现定制问题解决策略的代码，甚至自动运行实验以进行超参数调整[2]。

## 💡从技术输出到可操作见解

分析 AI 模型往往会产生密集的输出，正确地解释它们需要专业知识和时间。另一方面，LLM 代理可以通过将这些密集的定量结果转换为清晰、易于理解的自然语言解释来充当“翻译”。

这种可解释性功能在**解释**分析 AI 模型做出的决策方面发挥着至关重要的作用，使得人类操作员可以快速理解并采取行动。此外，这些信息对于模型开发者来说可能非常有价值，以验证模型输出的正确性，识别潜在问题，并提高模型性能。

除了技术解释之外，LLM 代理还可以为不同类型的受众生成定制化的响应：技术团队将收到详细的方法论解释，运营人员可能会得到实际的影响，而高管可能会获得强调业务影响指标的摘要。

通过作为分析系统和人类用户之间的**解释者**，LLM 代理可以显著提高分析 AI 的实际价值。

* * *

> **观点 3：未来可能在于分析 AI 和代理 AI 之间真正的点对点协作。**

无论 LLM 代理调用分析 AI 工具，还是分析系统使用 LLM 代理进行解释，我们迄今为止讨论的方法始终是关于一种 AI 负责另一种 AI。事实上，这引入了几个值得关注的限制。

首先，在当前范式下，分析 AI 组件仅作为被动工具使用，并且只有在 LLM 决定时才会被调用。这阻止了它们主动提供见解或质疑假设。

此外，典型的代理循环“计划-调用-响应-行动”本质上是顺序的。对于可能从并行处理或两个 AI 之间更异步的交互中受益的任务来说，这可能是不高效的。

另一个限制因素是有限的通信带宽。API 调用可能无法提供进行真正对话或交换中间推理所需丰富上下文。

最后，LLM 代理对分析人工智能工具的理解通常基于简短的文档字符串和参数模式。LLM 代理在工具选择上可能会出错，而分析人工智能组件缺乏上下文来识别它们被错误使用的情况。

仅因为今天工具调用模式的普及并不意味着未来应该看起来一样。可能，未来在于一个真正的点对点协作范式，其中没有一种 AI 类型是主导者。

这在实践中可能是什么样子呢？我发现的一个有趣例子是由西门子[3]提供的一个解决方案。

在他们的智能工厂系统中，有一个数字孪生模型持续监控设备的健康状况。当变速箱的状况恶化时，分析人工智能系统不会等待被查询，而是主动发出警报。一个 Copilot LLM 代理监视相同的事件总线。在警报发生时，它（1）交叉引用维护日志，（2）“询问”孪生体使用即将到来的班次模式重新运行模拟，然后（3）建议调整日程以防止昂贵的停机时间。这个例子独特之处在于，分析人工智能系统不仅仅是被动工具。相反，当需要时，它会主动发起对话。

当然，这只是可能的一种系统架构。其他方向，如具有专门认知功能的**多代理系统**，或者甚至将这些系统**交叉训练**以开发内化两种 AI 系统方面特点的混合模型（就像人类发展综合数学和语言思维一样），或者简单地从已建立的**集成学习技术**中汲取灵感，将 LLM 代理和分析人工智能视为可以以系统方式结合的不同模型类型。未来的机会是无限的。

但这些也提出了令人着迷的研究挑战。我们如何设计**共享表示**？哪种架构最能支持**异步信息交换**？在分析人工智能和代理之间，哪些**通信协议**是最优的？

这些问题代表了新的前沿，肯定需要分析人工智能实践者的专业知识。再次强调，用定量严谨性构建分析模型的知识深度并没有过时，但对于构建未来的这些混合系统是必不可少的。

> **观点 4：让我们拥抱互补的未来**。

正如我们在这篇帖子中看到的，未来不是“分析人工智能 vs. LLM 代理”。而是**“分析人工智能 + LLM 代理”**。

因此，与其对 LLM 代理感到 FOMO（恐惧错过），我现在对分析型 AI 的演变角色感到新的兴奋。我们建立的分析基础并没有变得过时，它们是更强大 AI 生态系统的基本组成部分。

让我们开始构建吧。
