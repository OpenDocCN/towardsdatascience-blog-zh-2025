在我最初在 Medium 上写的一篇文章中，我谈到了在 Pandas 数据框上使用 apply() 方法，并说如果可能的话，应该避免在较大的数据框上使用它。如果你想查看这篇文章，我将在本文末尾提供链接。

虽然我那时谈到了一些可能的替代方案，即使用向量化，但我没有给出很多使用向化的例子，所以我打算在这里纠正这一点。具体来说，我想谈谈如何使用 NumPy 和它的一些不太为人所知的函数 **( `where`** 和 **`select`**) 来加速涉及复杂 if/then/else 条件的 Pandas 操作。

在 Pandas 的上下文中，向量化指的是一次对整个数据块应用操作，而不是逐行或逐元素迭代。这种方法之所以可行，是因为 Pandas 依赖于 NumPy，NumPy 支持高度优化的向量化操作，这些操作是用 C 编写的，从而实现了更快的处理速度。当你使用 Pandas 中的向量化操作时，例如对 DataFrame 或 Series 对象应用算术运算或函数，操作会同时发送到多个数据元素。

这不仅使代码更加简洁易读，而且可以通过减少 Python 循环的开销并利用现代 CPU 在并行处理多个数据点的能力来显著提高性能。向量化是 Pandas 在 Python 中进行数据操作和分析的关键特性。

问题是一些向量化操作是显而易见的，你甚至没有意识到底层正在进行这样的操作。然而，很多时候，当你想要执行的操作类型变得更加复杂时，很难看到在这些情况下如何应用向量化。

在这篇文章中，我将讨论一些常见场景，在这些场景中这是适用的，并展示如何应用向量化。

### 获取一些可操作的数据

在我许多关于检查 Python 库和数据库系统性能的文章中，我使用一组合成销售数据来测试。这个数据集的模式看起来像这样，

+   `order_id` (整数)

+   `order_date` (日期)

+   `customer_id` (整数)

+   `customer_name` (字符串)

+   `product_id` (整数)

+   `product_name` (字符串)

+   `category` (字符串)

+   `quantity` (整数)

+   `price` (浮点数)

+   `total` (浮点数)

这里有一个你可以用来生成此类数据集的 Python 程序。它生成一个 CSV 文件。要创建的记录数量和输出文件的位置是可以配置的。它使用了 NumPy 和 polars 库，所以在运行之前你必须安装这些库。

对于这个测试，由于我有一台相当高性能的 PC，我将创建并使用一个包含 100 万条记录的 CSV 文件。

```py
# generate the 1m record CSV file
#
import polars as pl
import numpy as np
from datetime import datetime, timedelta

def generate(nrows: int, filename: str):
    names = np.asarray(
        [
            "Laptop",
            "Smartphone",
            "Desk",
            "Chair",
            "Monitor",
            "Printer",
            "Paper",
            "Pen",
            "Notebook",
            "Coffee Maker",
            "Cabinet",
            "Plastic Cups",
        ]
    )

    categories = np.asarray(
        [
            "Electronics",
            "Electronics",
            "Office",
            "Office",
            "Electronics",
            "Electronics",
            "Stationery",
            "Stationery",
            "Stationery",
            "Electronics",
            "Office",
            "Sundry",
        ]
    )

    product_id = np.random.randint(len(names), size=nrows)
    quantity = np.random.randint(1, 11, size=nrows)
    price = np.random.randint(199, 10000, size=nrows) / 100

    # Generate random dates between 2010-01-01 and 2023-12-31
    start_date = datetime(2010, 1, 1)
    end_date = datetime(2023, 12, 31)
    date_range = (end_date - start_date).days

    # Create random dates as np.array and convert to string format
    order_dates = np.array([(start_date + timedelta(days=np.random.randint(0, date_range))).strftime('%Y-%m-%d') for _ in range(nrows)])

    # Define columns
    columns = {
        "order_id": np.arange(nrows),
        "order_date": order_dates,
        "customer_id": np.random.randint(100, 1000, size=nrows),
        "customer_name": [f"Customer_{i}" for i in np.random.randint(2**15, size=nrows)],
        "product_id": product_id + 200,
        "product_names": names[product_id],
        "categories": categories[product_id],
        "quantity": quantity,
        "price": price,
        "total": price * quantity,
    }

    # Create Polars DataFrame and write to CSV with explicit delimiter
    df = pl.DataFrame(columns)
    df.write_csv(filename, separator=',',include_header=True)  # Ensure comma is used as the delimiter

# Generate data with random order_date and save to CSV
generate(1_000_000, "/mnt/d/sales_data/sales_data_1m.csv")
```

### 示例 3 – 向量化多个 if/then/else 条件。

嵌套的 if/then/else 条件可以使用嵌套的 Numpy.where 条件来处理。但是，当有多个此类情况时，代码的可读性和可维护性会变得困难。如果你曾经在使用 Excel 中的嵌套 IF 语句，你会知道我的意思。

在这个例子中，我们可以使用另一个相当酷的 NumPy 方法，很多人可能都没听说过，它被称为**select()**。

就像 where()一样，select()也接受 3 个参数。第一个是一个 Python 列表，包含要测试的**`条件`**。第二个参数是一个 Python 列表，告诉 NumPy 在满足等效条件时返回什么。第三个参数是**`默认`**语句，用于在没有满足任何条件时返回什么。

这些顺序很重要。choices 列表中的第一个项目与 conditions 列表中的第一个项目相对应，依此类推。

在某种程度上，这就像在 Python、C、Java 等编程语言中使用的 case 语句。

将我们的前一个例子推向极端。比如说，我们想要根据每种不同类型的产品乘以不同的数量。

```py
import pandas as pd
import numpy as np

# Start the timer
start_time = time.time()

# Define conditions
conditions = [
    df['product_names'] == 'Smartphone',
    df['product_names'] == 'Printer',
    df['product_names'] == 'Notebook',
    df['product_names'] == 'Laptop',
    df['product_names'] == 'Cabinet'
]

# Define choices
choices = [
    df['quantity'] * 1.5,
    df['quantity'] * 2.5,
    df['quantity'] * 3.5,
    df['quantity'] * 4.5,
    df['quantity'] * 5.5
]

# Default value if none of the conditions are met
default_choice = 0

# Update 'quantity' using numpy.select
df['quantity'] = np.select(conditions, choices, default=default_choice)

# Recalculate 'total' based on the new 'quantity'
df['total'] = df['quantity'] * df['price']

# Display the updated DataFrame
print(df.head(10))

# End the timer and calculate the elapsed time
elapsed_time = time.time() - start_time
print(f"Time taken : {elapsed_time} seconds")
```

### 示例 4 – 向量化嵌套多个 if/then/else 条件

我的最后一个例子与之前的例子类似，但现在只有当智能手机的现有数量大于 5 时，智能手机的数量才会乘以 1.5。否则，数量会乘以 2。

其他条件保持不变。换句话说，我们有一个嵌套的 if/then/else 场景。如果这段逻辑在一个像 Python 这样的编程语言中的函数里，它看起来会像这样，

```py
def adjust_quantity(row):
    if row['product_names'] == 'Smartphone':
        if row['quantity'] > 5:
            row['quantity'] *= 1.5
        else:
            row['quantity'] *= 2
    elif row['product_names'] == 'Printer':
        row['quantity'] *= 2.5
    elif row['product_names'] == 'Notebook':
        row['quantity'] *= 3.5
    elif row['product_names'] == 'Laptop':
        row['quantity'] *= 4.5
    elif row['product_names'] == 'Cabinet':
        row['quantity'] *= 5.5
    else:
        row['quantity'] = 0  # Default case if none of the conditions are met
    return row
```

我们能否将其向量化？是的，我们可以通过修改之前的方法来实现。这次，我们在`choices`列表中添加了一些额外的布尔逻辑测试和额外的动作。

我不会重复整个代码，只重复变化的部分。

```py
# Define conditions
conditions = [
  (df['product_names'] == 'Smartphone') &amp; (df['quantity'] > 5),
  (df['product_names'] == 'Smartphone') &amp; (df['quantity'] <=5),    
  df['product_names'] == 'Printer',
  df['product_names'] == 'Notebook',
  df['product_names'] == 'Laptop',
  df['product_names'] == 'Cabinet'
]

# Define choices
choices = [
  df['quantity'] * 1.5,
  df['quantity'] * 2,    
  df['quantity'] * 2.5,
  df['quantity'] * 3.5,
  df['quantity'] * 4.5,
  df['quantity'] * 5.5
```
