## 为什么你应该阅读这篇文章

数据科学吸引了各种不同的背景。根据我的专业经验，我曾与曾是：

+   核物理学家

+   研究引力波的后博士

+   计算生物学博士

+   语言学家

只举几个例子。

能够遇到这样多样化的背景真是令人愉快，我看到了各种不同的思维如何导致一个富有创造力和有效性的数据科学功能的成长。

然而，我也看到了这种多样性的一个重大缺点：

> 每个人对关键软件工程概念的了解程度不同，导致编程技能参差不齐。

因此，我看到了一些数据科学家完成的工作非常出色，但却是：

+   不可读的——你不知道他们试图做什么。

+   不稳定的——一旦有人尝试运行它就会崩溃。

+   不可维护的——代码很快就会过时或容易损坏。

+   不可扩展的——代码是一次性使用，其行为无法扩展。

…这最终削弱了他们工作的影响力，并导致了一系列问题。

+   不可读的代码意味着错误被忽视，

+   未注意到的错误会破坏模型和/或数据管道，

+   同时，即使找到错误也需要花费几天时间，因为代码不可读

+   尽管代码的原始作者已经离开公司，因此无法获得帮助，

+   导致模型需要从头开始重新训练，数据管道需要重新编写，整个项目需要重新做。

## 今日概念：封装

![照片由 Laura Gariglio 在 Unsplash 上拍摄。](img/ff5b2e3130a6e139b78215168209cc7a.png)

图片由[Laura Gariglio](https://unsplash.com/@laugariglio?utm_source=medium&utm_medium=referral)在[Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral)上提供。

互联网上有许多令人困惑和抽象的定义，它们使一个简单的概念变得过于复杂。

> 本质上，封装就是隐藏对将要阅读或使用你代码的人来说不重要的复杂细节。

你只想揭示它做什么以及某人如何使用它。我在学习这个时记得的一个类比是：

> 所有驾驶员都知道如何使用方向盘。但大多数人不知道它是如何或为什么工作，而且他们不需要知道。

封装也等同于信息/数据隐藏。

有效地使用封装应该使你的代码：

+   更容易阅读。

+   更容易维护（即诊断和修复错误）。

+   更容易扩展（即添加或更改功能）。

### 你已经见过封装了

`scikit-learn`的估算器套件是封装的理想例子。

> [**KMeans**](https://scikit-learn.org/1.5/modules/generated/sklearn.cluster.KMeans.html#sklearn.cluster.KMeans)

我们都知道 KMeans 是如何工作的以及它应该做什么。我们也知道如何使用`scikit-learn`实现的`KMeans` – 它的所有文档都作为 docstrings。

```py
from sklearn.cluster import KMeans
```

```py
model = KMeans() # initialise a model
model.fit(X_train) # fit the model
model.predict(X_test) # predict on new data
```

但我们是否需要知道`fit`和`predict`方法是如何实现的？我是否需要阅读`fit`中的所有 126 行代码来了解如何使用它？

**不**。**我们只需要知道它为我们给出的数据输出了一些聚类。**

即使是新手数据科学家也能使用这个接口并在一定程度上理解它。这就是封装的点。

![图片由 Dan Crile 在 Unsplash 上提供](img/4f348ae5a3124fe23e1af629b8960098.png)

图片由[Dan Crile](https://unsplash.com/@dancrile?utm_source=medium&utm_medium=referral)在[Unsplash](https://unsplash.com?utm_source=medium&utm_medium=referral)上提供。

作为用户，我们不需要知道底层发生的复杂细节。所有这些细节都被整理得很好，放在一个有意义的函数`fit`下面。

我们只需要知道`fit`的预期输入和输出是什么，并适当地使用它。

**现在，让我们通过一个 Jupyter Notebook 中的实际例子来看看这如何应用于典型数据科学家的日常工作中。**

### 文章生命周期统计

此数据集通过一个唯一的 ID 列“文章”识别文章，以及它们的各种统计数据和截至 2024 年 12 月的收入。

1.  `earnings` 列有一个无效的值为 `9999999999` 的单元格。

## 封装在工作中

要在数据科学中有效地使用封装，你需要考虑以下因素：

+   **你的受众是谁？**

+   **未来谁可能需要阅读这段代码？**

+   **他们为什么需要阅读这段代码？**

你的代码可能需要被其他人阅读，原因有很多：

+   **为了审计目的复制结果。**

+   **为了使用不同的参数重新运行代码。**

+   **为了调试影响下游流程的输出或结论中的问题。**

根据你的判断，你需要做出以下决定：

+   **哪些部分的代码是重要的，并且是打算直接供读者使用的？（参数、文件路径、常量）**

+   **哪些部分的代码可以被隐藏起来？**

而隐藏这些代码应该通过将它们打包成可管理和有意义的块来使你的代码更容易阅读和理解。

现在，考虑以下使用封装原则重写的代码。

### 在你的 Jupyter Notebook 的第 1 个单元格中：

```py
def clean_published_date_column(col:pd.Series) -> pd.Series:
    """Cleaning steps for `published date` column:

    1\. One entry of '2024-07-32' which is a date that is out of range
    for July. We replace this with 1st Aug.
```

```py
 2\. Dates are of type `str`, and in different formats. This is 
    handled using `pandas.to_datetime` with args `format='mixed'`.
```

```py
 Parameters
    ----------
    col: pandas.Series
        A Series containing article published date entries.
```

```py
 Returns
    -------
    col: pandas.Series
        The cleaned `published date` column.
    """
    jul_date_out_of_range_mask = (col == '2024-07-32')
    jul_date_out_of_range_replacement = '2024-08-01'
    col.loc[jul_date_out_of_range_mask] = jul_date_out_of_range_replacement
    return pd.to_datetime(col, format='mixed') 
```

```py
def clean_earnings_column(col:pd.Series):
    """Cleaning steps for `earnings` column:

    1\. One entry where the value is 999999999999\. We replace
    any values exceeding 10 figures with zero.
```

```py
 Parameters
    ----------
    col: pandas.Series
        A Series containing article lifetime earnings data.
```

```py
 Returns
    -------
    col: pandas.Series
        The cleaned `earnings` column.
    """
    earnings_too_big_mask = (np.log10(col) > 10)
    earnings_too_big_replacement = 0
    col.loc[earnings_too_big_mask] = earnings_too_big_replacement
    return col 
```

```py
def clean_claps_column(col:pd.Series):
    """Cleaning steps for `claps` column:

    1\. Fill null values with zero.
```

```py
 Parameters
    ----------
    col: pandas.Series
        A Series containing number of claps per article.
```

```py
 Returns
    -------
    pandas.Series
        The cleaned `claps` column.
    """
    return col.fillna(0)
```

### 在第 2 个单元格中：

```py
df = pd.read_csv('~/Downloads/test.csv')
```

```py
# I bet you didn't read the functions above. 
# Well, that's the point of encapsualtion. You shouldn't need to to understand
# what's going on here.
df['published date'] = clean_published_date_column(df['published date'])
df['claps'] = clean_claps_column(df['claps'])
df['earnings'] = clean_earnings_column(df['earnings'])
```

这很简单，我们只是把现有的代码放入它们各自的功能中，给它们起有意义的名字，并添加文档字符串来指导读者如何使用函数以及它的预期行为。

你可能会想

> “真浪费时间，这篇文章只是告诉我把东西放入函数里，这有什么大不了的。”

嗯，是的，部分如此。

将代码放入任何函数中总会实现一定程度的封装。但有效的封装不仅仅是把所有东西都塞进一个函数里。

这种差异可以类比为

+   清理你的房间，

与之相对的是

+   把所有的衣服和垃圾扫到床下。

```py
def clean(df):
  df.loc[df['published date'] == '2024-07-32', 'published date'] = '2024-08-01'
  df['published date'] = pd.to_datetime(df['published date'], format='mixed')
  df['claps'] = df['claps'].fillna(0)
  df.loc[np.log10(df['earnings']) > 10, 'earnings'] = 0
  return df
```

```py
df = pd.read_csv('~/Downloads/test.csv')
df = clean(df) # the 'sweeping everything under the bed' equivalent.
```

让我们先检查有效的封装实现了什么，以及这是如何实现的。

## 封装实现了什么？

为了简洁起见，我们只介绍两个与数据科学最相关的优势。

此外，请注意，关于如何做这件事并没有正确答案。根据你的意图和假设，可能有多种方式应用封装。以下是我的尝试，我将尝试解释为什么我这样设计，以及为什么这样做是有意义的。

最重要的是封装产生的影响。

### 代码的可读性更好

首先，我们在信息层次上创建了一个关于其细节和与读者的相关性的信息体系。

```py
df = pd.read_csv('~/Downloads/test.csv')
```

```py
# data cleansing
df['published date'] = clean_published_date_column(df['published date'])
df['claps'] = clean_claps_column(df['claps'])
df['earnings'] = clean_earnings_column(df['earnings'])
```

我故意将*“清理”*这个数据集的细节封装到三个精心设计的函数中，这反过来又服务于以下目的：

1.  它们向读者表明，对输入数据已执行了一些复杂操作，

1.  而读者是否需要深入了解取决于他们的目标。

1.  如果读者需要查看内部结构，他们可以自由地查看每个函数及其相应的文档字符串，并立即以可管理、有意义的方式理解每个函数的工作原理。

1.  如果他们不需要细节，他们可以快速跳转到与他们相关的部分，至少在较高层次上了解上述代码的目的是什么。

这种方法假设笔记本中还有其他更重要的部分，例如模型构建或特征探索。因此，我觉得数据清理部分不是这段代码的主要焦点。

因此，我将每个函数命名为`clean_xxx_column`格式，假设读者只需要知道*“这里进行了一些清理以创建一个可工作的数据集”，*并且如果他们的兴趣在于模型构建/特征探索，他们可以跳过细节。

> 最终，封装应该使你的代码更容易阅读，向读者展示恰好足够的信息，以便他们能够了解正在发生的事情，同时避免信息轰炸。

更好的可读性使得其他人更容易使用你的代码，理解它并维护它。

### 更好的可扩展性和关注点分离

假设现在你从相同的数据源接收到了一个更大的数据集。你使用与之前相同的代码将其加载进来：

```py
df = pd.read_csv(f'~/Downloads/bigger_test.csv')
```

```py
# data cleansing
df['published date'] = clean_published_date_column(df['published date'])
df['claps'] = clean_claps_column(df['claps'])
df['earnings'] = clean_earnings_column(df['earnings'])
```

然而，这次你发现需要额外的清理步骤；`claps`列与`earnings`列有相同的问题，其中一些值非常大：`9999999999`

在这个例子中，封装显示了两个好处：

1.  ***可扩展性***：扩展我们现有的代码以处理新的数据问题既简单又平凡。即使是代码的新手也能很容易地知道他们需要修改哪个函数。

```py
def clean_claps_column(col:pd.Series):
    """Cleaning steps for `claps` column:

    1\. Fill null values with zero.
    2\. Replace any values exceeding 10 figures with zero.
```

```py
 Parameters
    ----------
    col: pandas.Series
        A Series containing number of claps per article.
```

```py
 Returns
    -------
    pandas.Series
        The cleaned `claps` column.
    """
    # additional functionality added here.
    # Note: since this code is repeated in `clean_earnings_column`, 
    # it could be put into its own function. For brevity we leave it
    # as is for now. 
    too_big_mask = (np.log10(col) > 10)
    too_big_value_replacement = 0
    col.loc[too_big_mask] = too_big_value_replacement
```

```py
 # existing functionality
    return col.fillna(0)
```

而不是不得不搜索每一行代码以知道在哪里添加更改：

```py
df = pd.read_csv('~/Downloads/bigger_test.csv')
```

```py
# I bet you won't even bother reading the below because it is so untidy.
df.loc[df['published date'] == '2024-07-32', 'published date'] = '2024-08-01'
df['published date'] = pd.to_datetime(df['published date'], format='mixed')
df['claps'] = df['claps'].fillna(0)
df.loc[np.log10(df['claps']) > 10, 'claps'] = 0
df.loc[np.log10(df['earnings']) > 10, 'earnings'] = 0
```

1.  ***关注点分离***：代码更改被整洁地包含在`clean_claps_column`函数中，而所有其他代码保持不变。

```py
df = pd.read_csv(f'~/Downloads/bigger_test.csv')
```

```py
# Nothing changes here. Same as usual. 
df['published date'] = clean_published_date_column(df['published date'])
df['claps'] = clean_claps_column(df['claps'])
df['earnings'] = clean_earnings_column(df['earnings'])
```

负责运行清理的代码部分不需要担心任何清理函数底层的操作，只要函数行为没有改变。

这使得代码更改更加整洁且易于管理。

## 这实现了什么？

封装的概念是从软件工程世界中数十年的试错经验中发展起来的。

最终目标是使代码更容易维护。

不幸的是，我仍然看到数据科学家完全依赖于笔记本和他们的无组织代码来维护整个生产流程。当我看到这种情况时，我担心这个行业的未来。

## 结论

总结来说，封装是软件工程的一个关键原则，可以显著提高代码的可读性、可维护性和可扩展性。

通过隐藏复杂细节并仅暴露必要的部分，你使你的代码更加用户友好，并最小化返工。

对于数据科学家来说，掌握封装不仅提高了你工作的质量，而且使你成为一个能够构建健壮、生产就绪系统的专业人士。
