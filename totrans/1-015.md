# 机器学习“圣诞日历”第五天：Excel 中的高斯混合模型

> 原文：[https://towardsdatascience.com/the-machine-learning-advent-calendar-day-5-gmm-in-excel/](https://towardsdatascience.com/the-machine-learning-advent-calendar-day-5-gmm-in-excel/)

在 [上一篇文章](https://towardsdatascience.com/the-machine-learning-advent-calendar-day-4-k-means-in-excel/) 中，我们探讨了基于距离的 K-Means 聚类。

今天，我们更进一步：为了改进距离的测量方法，我们添加了方差，以便得到马氏距离。

所以，如果 k-Means 是 **最近邻质心分类器** 的无监督版本，那么自然的问题就是：

**QDA 的无监督版本是什么？**

这意味着，就像 QDA 一样，每个簇现在不仅需要描述其 **均值**，还需要描述其 **方差**（如果特征数量超过 2，我们还需要添加协方差）。但在这里一切都是无标签学习的。

所以你明白了这个想法，对吧？

好吧，这个模型的名称是 **高斯混合模型 (GMM)**…

## GMM 以及这些模型的名称……

如常，模型的名称往往源于历史原因。它们并不总是旨在突出模型之间的联系，除非它们被一起发现。

不同的研究者、不同的时期、不同的应用场景……最终我们得到了一些名称，有时这些名称隐藏了背后的真实结构。

在这里，“高斯混合模型”这个名称简单意味着数据被表示为 **几个高斯分布的混合**。

如果我们遵循与 *k-Means* 相同的命名逻辑，那么将其称为类似 **k-Gaussian Mixture** 会更清晰。

因为在实践中，我们不仅使用均值，还添加了方差。我们可以直接使用马氏距离，或者使用均值和方差加权的其他距离。但高斯分布给出了更容易解释的概率。

因此，我们选择一个数字 *k* 来表示高斯成分的数量。

顺便说一下，GMM 并非唯一。

实际上，整个 **机器学习框架** 比它包含的许多模型都要新得多。这些技术中的大多数最初都是在统计学、信号处理、计量经济学或模式识别中开发的。

然后，在很久以后，我们现在称之为“机器学习”的领域出现了，并将所有这些模型统一归类。但名称并没有改变。

因此，今天我们使用来自不同时代、不同社区和不同意图的词汇混合体。

这就是为什么当你只看名称时，模型之间的关系并不总是显而易见的。

如果我们必须用现代、统一的 **机器学习风格** 来重新命名所有事物，那么整个领域实际上会更加清晰：

+   **GMM** 将变为 **k-Gaussian Clustering**

+   **QDA** 将变为 **最近高斯分类器**

+   LDA，嗯，**具有相同类间方差的最近高斯分类器**。

突然，所有的链接都出现了：

+   k-Means ↔ 最近质心

+   GMM ↔ 最近高斯（QDA）

这就是为什么GMM在K-Means之后如此自然。如果K-Means通过最近的质心将点分组，那么GMM通过最近的**高斯形状**将它们分组。

为什么整个章节都在讨论名字？

好吧，事实是，因为我们已经介绍了k-means算法，我们已经从最近质心分类器过渡到了QDA，我们已经了解了这个算法的所有内容，训练算法不会改变…

那这个训练算法的名字是什么呢？

哦，Lloyd算法。

实际上，在k-means被这样称呼之前，它只是简单地被称为Lloyd算法，由**Stuart Lloyd**在**1957年**发表。只是后来，机器学习社区将其改为“k-means”。

而且这个算法只操作了均值，所以我们需要另一个名字，对吧？

你可以看到这是走向：期望最大化算法！

EM算法仅仅是Lloyd思想的通用形式。Lloyd更新均值，EM更新一切：均值、方差、权重和概率。

所以，你已经了解了GMM的所有内容！

但由于我的文章标题是“Excel中的GMM”，我不能在这里结束我的文章…

## 二维GMM

逻辑与1D中的完全相同。在概念上没有新内容。我们只是扩展了公式…

我们现在不再有一个特征对应一个点，而是**两个**。

每个高斯现在必须学习：

+   x1的均值

+   x2的均值

+   x1的方差

+   x2的方差

+   以及两个特征之间的协方差项。

一旦你在Excel中写下这些公式，你就会发现整个过程完全相同：

嗯，事实上，如果你看截图，你可能会想：“哇，公式这么长！”而且这还不是全部。

![图片](../Images/2e256cb4303534d84067a8e570bff15f.png)

Excel中的2D GMM – 图像由作者提供

但不要被误导。公式之所以长，仅仅是因为我们明确地写出了**二维高斯密度**：

+   一部分是x1的距离

+   一部分是x2的距离

+   协方差项

+   归一化常数

没有更多。

这只是按单元格逐个展开的密度公式。

虽然输入很长，但一旦你看到结构：一个加权距离，在指数内部，除以行列式。

所以是的，公式看起来很大…但背后的想法极其简单。

## 与k-Means的比较

现在，GMM与[K-Means](https://towardsdatascience.com/the-machine-learning-advent-calendar-day-4-k-means-in-excel/)相比如何？

更重要的是，在哪些情况下它们的结果实际上会有所不同？

答案很简单：**一旦方差变得重要**。

GMM比K-Means有更多的参数。即使在单维度中，每个簇也不再仅由均值定义，还由方差定义。这个单一的增加足以改变模型的行为。

让我们用一个有两个簇的简单例子来说明。一组点非常紧密，方差很小。另一组点分布得更广，方差更大。

![图片](../Images/dbf7f9b48c902eb7f8ed39a51553f887.png)

K-Means看不到这个差异。它只看距离质心的距离，并且隐含地假设两个簇的分布相同。因此，簇之间的边界固定在两个中心之间的中点。

K-Means看不到这个差异。它只看距离质心的距离，并且隐含地假设两个簇的分布相同。因此，簇之间的边界固定在两个中心之间的中点。

![图片](../Images/a22ce59cb9baa552698b545d0f744bae.png)

相反，GMM会适应数据的结构。紧密簇群保持狭窄，宽散簇群保持宽散。它们之间的分离不再是对称的，因为距离现在是按方差缩放的。

![图片](../Images/895954e24b110b956d884506c71f67d4.png)

这就是结果开始出现差异的地方。

离紧密簇群“较远”的点对于宽散簇群来说可能仍然是完全正常的。GMM能够自然地捕捉到这一点，而K-Means则不能，因为它没有扩散的概念。

这正是GMM比K-Means更具表现力的情况。不是因为它更复杂，而是因为它允许模型描述数据实际的样子。

## 结论

K-Means给出的是硬边界。

GMM给出概率。

一旦将EM公式写入Excel，模型就变得简单易懂：均值移动，方差调整，高斯自然地围绕数据分布。

GMM是k-Means之后的下一个逻辑步骤，提供了一种更灵活的方式来表示簇及其形状。
