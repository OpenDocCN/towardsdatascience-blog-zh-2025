## 角色比较

在我们开始之前，让我们确保我们都在同一页上，关于这些角色之间的区别是什么，从数据分析开始。

数据分析师专注于使用结构化数据来推动业务决策。他们的工具箱通常包括 SQL、Excel、Tableau 或 PowerBI，以及用于数据处理、可视化和可能简单的统计分析的基本 Python。这个角色集中在理解*历史*数据，以回答关于发生了什么以及为什么的问题。

数据科学家在这些基础上进行扩展，进入预测建模和自动化决策。虽然他们也使用 SQL 和 Python，但他们更多地使用统计建模、机器学习框架和云平台。他们的重点转向预测*未来*结果和推荐行动。

一个常见的误解是，数据分析师必须成为数据科学家才能提升他们的职业生涯。这绝对不是真的！

高级分析师可以赚取高薪，并具有非常强的商业影响力，而无需深入的了解机器学习或统计学。

说实话，并不是每个人都喜欢数据科学的工作，很多人可能会更愿意留在分析的道路上。

因此，在我们继续之前，请自问以下问题：

1.  你对机器学习和它是如何工作的好奇吗？

1.  你是否对（或者至少对）高级数学和统计学感到舒适（或至少感兴趣）？

1.  你是否对技术挑战和软件工程概念感到舒适？

1.  你是否可以接受一个在日常工作以及职业发展方面都有很多不确定性的角色？

如果你还在这里，并且想着“是的，我确实想追求数据科学”，那我们就来谈谈如何让它真正实现。

## 转型所需的技能

好的，既然你已经决定开始转型，那么让我们来分析一下你需要培养的关键技能。我们将重点关注四个核心领域，这些领域构成了数据科学工作的基础。

### 数学与统计学

如果你来自分析背景，你可能已经接触过一些统计学知识，但数据科学可能需要在数学方面有更深入的理解。你需要熟悉：

+   多变量微积分和线性代数，特别是矩阵运算和梯度，以理解机器学习算法。但别担心——你不需要成为数学专家，你只需要足够的数学知识来理解基础，帮助你掌握算法的工作原理。

+   你还需要概率理论和假设检验来设计实验。

+   以及统计概念，如不同类型的分布和回归技术

+   理想情况下，一些因果推断的经验

### 编程

如果你已经在你的职位上使用 SQL 和基本的 Python，那么你在这里已经领先一步。现在只是关于提升水平。关注：

+   更高级的 Python，比如面向对象编程（OOP）基础、编写模块化可维护的代码、单元测试、性能优化等等。

+   使用 ML 框架，如 scikit-learn、Tensorflow 和 PyTorch。

+   并且熟悉基本的数据结构和算法，以应对编码面试。通常这将是关于数组和字符串的问题，所以你不需要在这方面做得太过分，但了解这些对于面试来说很重要。

### 机器学习与人工智能基础

这也是数据科学的核心支柱之一，所以你需要熟悉机器学习（ML）的基础知识，比如：

+   监督学习（即回归和分类）。

+   无监督学习（如聚类和降维）。

+   模型评估和验证。

+   深度学习基础。

+   现在，熟悉生成式人工智能（GenAI）是一个加分项（但我的意思是学习如何使用 API，而不是从头开始训练模型）。

### 大数据与数据工程概念

最后，许多数据科学职位涉及处理大规模数据集和构建自动化管道。为此，你需要关注：

+   与云计算平台合作，特别是 AWS 服务如 S3 和 SageMaker。

+   使用 Airflow 等工具进行数据管道开发。

+   可能还需要一些基本系统设计原则，以扩展你的解决方案（随着你变得更加资深或更专注于机器学习，这一点变得越来越重要）。

### 坦克营

现在，也许您在想您可能更喜欢在学习中有一点更多的结构性和外部问责制。如果您不想承诺一个完整的学位，坦克营可以是一个选择。

坦克营的一些优点是：

+   快速学习——您通常可以在几个月内完成它们。

+   结构化课程，因为所有内容都为您准备好了，所以您不必自己拼凑学习计划。

+   以及社区支持——您可以与同行一起学习，并从可能已经在该领域工作的导师那里获得指导。

需要记住的一件事是，速成班的质量各不相同，并不是所有都受到雇主的极高评价。在注册之前，做好研究——所以，查看评论，与校友交谈，并确保他们提供职业支持。

### 硕士学位

对于那些想要深入数据科学并拥有强大人脉机会的人来说，硕士学位可以是一个稳固的投资。这尤其适用于你从非技术背景过渡，或者如果你担心你的背景无法通过简历筛选工具。

立刻的缺点很明显，那就是硕士学位课程可能很昂贵且耗时。但好消息是，现在有一些负担得起、可以兼职在线学习的课程，允许你在工作的同时学习。例如，乔治亚理工学院的课程真的很实惠，而且质量相当不错。

### 导师

无论你选择哪条路，导师都可以提供极大的帮助。有一个人来指导你，提供反馈，并帮助你进行职业导航，这可以产生巨大的影响。

寻找导师的一些方法：

+   在你的公司——如果你的公司有数据科学家，问问你是否可以与他们合作或跟随他们。

+   领英——加入数据科学小组或联系专业人士（如果你需要这方面的帮助，我有一个关于[导师策略](https://www.youtube.com/watch?v=T0LP6Pv3_zs&t=35s)的完整视频！）。

+   在线社区如 Reddit、Discord 服务器和 Slack 群组可以成为与其他学习者和专业人士建立联系的另一种途径。

+   或者，雇佣一个导师——如果你急于快速提升，投资一个导师*可能*是值得的。

## 展示经验

好的，所以你已经学到了所有需要的技能。那很好，但你怎么向潜在雇主证明你实际上能胜任数据科学家的职位？

我有一个关于如何构建作品集和在你全职工作之外获得经验的[完整视频](https://www.youtube.com/watch?v=oG-IIUoPzBA&t=3s)。那里的总结是，你应该尽力去做一些自我激励的项目，尽可能模拟实际工作中的工作条件。

但如果你正在阅读这篇文章，那么你很可能已经是一名数据分析师，这为你提供了在当前角色中可以利用的另一个机会集。

例如，假设你经常在 Excel 或 Tableau 中创建报告。你可以用 Python 脚本来自动化这个过程，甚至可能添加一些预测元素。或者，如果你的公司进行 A/B 测试，你可以自愿帮助进行统计分析。

如果你有一个数据科学团队，尝试与他们合作一个项目。如果没有数据科学团队，向你的雇主提出一些有影响力的项目，这些项目也能帮助你学习。

最佳情况，这可能导致内部转变。最坏的情况，你现在有了具体的影响力和真实的数据科学项目，可以包括在你的简历中。

## 找工作

如果你能够内部转换角色，那就太好了，你已经完成了！如果不能，以下是一些帮助你获得第一个数据科学角色的策略：

首先，让我们谈谈如何在线定位自己。你的简历、LinkedIn 和 GitHub 需要讲述一个一致的故事，表明你已经是一名合格的数据科学家（因为如果你有技能并且完成了扎实的项目，你就是！）。所以，与其写“数据分析师寻求数据科学家职位”，你可以说“专注于预测分析和机器学习的数据专业人士。”

当涉及到你的 GitHub 时，确保将最好的内容放在最上面。这对于分析师来说尤为重要，因为你的编码技能将受到更多的审查。所以，

+   将你最好的机器学习项目置于顶部

+   编写清晰的 README 文件，解释你的方法

+   确保你的代码结构良好且经过良好文档化，这表明你理解软件工程原则

+   并添加可视化结果来展示影响，这对于你来说应该很容易，因为你具备这样的背景！

到时候申请时，优先考虑混合角色。这些角色位于传统分析和数据科学之间，通常是一个极好的跳板。

例如，许多公司（包括像 Meta 和 Amazon 这样的科技巨头）都有被称为“数据科学家”的角色，但实际上更像是高级分析职位。而且说实话，在许多公司，这些界限本身就是模糊的。利用这种模糊性来为你自己谋利！

当你在建立人脉和准备面试时，利用你的分析背景。利用你对商业背景的深入理解、清晰的沟通技巧以及你如何影响业务以产生可衡量影响的例子。其他可能比你更技术性的候选人可能会在商业和沟通方面遇到困难。所以，不要害怕发挥你的优势。
