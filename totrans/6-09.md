让我们以形状为 [2,4,2048] 的虚拟输入为例 → [batch_size, seq_len, embedding dimension]。Llama 使用具有 32 个头的多头注意力。

### **块 1:**

经过 proj → query_states 后是一个 [2,4,2048] 的张量，key_states 是一个 [2,4,512] 的张量，value_states 是一个 [2,4,512] 的张量。

观察并转置后为：query_states → [2,32,4,64] key_states → [2,8,4,64] value_states → [2,8,4,64]

这里 64 是嵌入维度，key 和 value 的头数为 8，因为 llama 使用键值组，基本上在 32 个总头数中，4 个头的组共享相同的 key_states 和 value_states。

### **块 2:**

在这个块中，我们只是应用了位置编码，特别是 Llama 使用的旋转位置编码（RoPE）。我不会详细介绍为什么需要这个，但你可以阅读以下文章以获得更好的理解：

> [**掌握位置编码：第一部分**](https://towardsdatascience.com/master-positional-encoding-part-i-63c05d90a0c3)

### **块 3：**

在这里，我们只是应用了 repeat_kv 函数，它只是重复了 kv 值在 4 个组的 4 个值，我们还使用了 past_key_value，这样我们就可以使用一些预先计算的 kv 值，从而提高计算效率，不必再次计算。

### 块 4：

块 4 处理了注意力准备的两个主要步骤：设置因果掩码以确保标记只关注先前位置，以及通过连续张量优化内存布局以提高 GPU 操作的效率。

### 块 5：

这是我们应用 softmax 注意力的地方——我们将在实现中替换的组件。

### 块 6：

注意力输出将是一个形状为[2, 32, 4, 64]的张量。我们将它转换回[2, 4, 2048]，并应用最终的输出投影。

这就是输入通过 Llama 自注意力过程的旅程！

## 混合注意力块

现在让我们看看我们的 HybridAttention 块：

```py
class HybridAttention(LlamaSdpaAttention):
    def __init__(self, config, layer_idx=None):
        super().__init__(config, layer_idx=layer_idx)
        self.window_size = 64
        #self.layer_idx = layer_idx

        # Initialize learnable factors
        # Create one factor pair per attention head
        num_heads = config.num_attention_heads
        self.window_factors = torch.nn.Parameter(torch.ones(1, num_heads, 1, 1) * 0.5)
        self.linear_factors = torch.nn.Parameter(torch.ones(1, num_heads, 1, 1) * 0.5)

        self.factor_activation = torch.nn.Sigmoid()

    def sliding_window_attention(self, query_states, key_states, value_states, window_size, window_factor):
        """Compute sliding window attention"""
        batch_size, num_heads, seq_len, head_dim = query_states.shape

        key_windows = F.pad(key_states, (0, 0, window_size - 1, 0), value=0)
        key_windows = key_windows.unfold(2, window_size, 1)

        value_windows = F.pad(value_states, (0, 0, window_size - 1, 0), value=0)
        value_windows = value_windows.unfold(2, window_size, 1)

        attn_weights = torch.einsum('bhld,bhldw->bhlw', query_states, key_windows) * (head_dim ** -0.5)
        attn_weights = torch.where(attn_weights == 0,
                                 torch.tensor(-float('inf'), device=attn_weights.device),
                                 attn_weights)

        # Apply learnable window factor (with sigmoid to ensure positivity)
        attn_weights = self.factor_activation(window_factor) * F.softmax(attn_weights, dim=-1)

        attn_output = torch.einsum('bhlw,bhldw->bhld', attn_weights, value_windows)
        sum_weights = attn_weights.sum(dim=-1, keepdim=True)

        return attn_output, sum_weights

    def linear_attention(self, query_states, key_states, value_states, window_size, linear_factor):
        """Compute linear attention with cumsum"""
        def feature_map(x):
            return F.elu(x) + 1

        query_prime = feature_map(query_states)
        key_prime = feature_map(key_states)

        key_prime = F.pad(key_prime, (0, 0, window_size, 0), value=0)[:, :, :-window_size, :]
        value_padded = F.pad(value_states, (0, 0, window_size, 0), value=0)[:, :, :-window_size, :]

        # Compute KV
        kv = torch.einsum('bhlf,bhld->bhlfd', key_prime, value_padded)
        # Apply learnable linear factor (with sigmoid to ensure positivity)
        qkv = self.factor_activation(linear_factor) * torch.einsum('bhlf,bhlfd->bhld',
                                                                  query_prime,
                                                                  kv.cumsum(dim=2))

        sum_k = key_prime.cumsum(dim=2)
        sum_qk = self.factor_activation(linear_factor) * torch.einsum('bhld,bhld->bhl',
                                                                     query_prime,
                                                                     sum_k)[..., None]
        sum_qk = torch.where(sum_qk == 0, torch.tensor(1e-12, device=sum_qk.device), sum_qk)

        return qkv, sum_qk

    def hybrid_attention(self, query_states, key_states, value_states):
        """Combine sliding window and linear attention with learnable factors"""
        qkv_window, sum_window = self.sliding_window_attention(
            query_states, key_states, value_states,
            self.window_size, self.window_factors
        )

        qkv_linear, sum_linear = self.linear_attention(
            query_states, key_states, value_states,
            self.window_size, self.linear_factors
        )

        output = (qkv_window + qkv_linear) / (sum_window + sum_linear)
        return output

    def forward(
        self,
        hidden_states: torch.Tensor,
        attention_mask: Optional[torch.Tensor] = None,
        position_ids: Optional[torch.LongTensor] = None,
        past_key_value: Optional[Cache] = None,
        output_attentions: bool = False,
        use_cache: bool = False,
        cache_position: Optional[torch.LongTensor] = None,
        position_embeddings: Optional[Tuple[torch.Tensor, torch.Tensor]] = None,
        **kwargs,
    ):
        bsz, q_len, _ = hidden_states.size()

        query_states = self.q_proj(hidden_states)
        key_states = self.k_proj(hidden_states)
        value_states = self.v_proj(hidden_states)

        query_states = query_states.view(bsz, q_len, -1, self.head_dim).transpose(1, 2)
        key_states = key_states.view(bsz, q_len, -1, self.head_dim).transpose(1, 2)
        value_states = value_states.view(bsz, q_len, -1, self.head_dim).transpose(1, 2)

        if position_embeddings is None:
            cos, sin = self.rotary_emb(value_states, position_ids)
        else:
            cos, sin = position_embeddings
        query_states, key_states = apply_rotary_pos_emb(query_states, key_states, cos, sin)

        if past_key_value is not None:
            cache_kwargs = {"sin": sin, "cos": cos, "cache_position": cache_position}
            key_states, value_states = past_key_value.update(key_states, value_states, self.layer_idx, cache_kwargs)

        key_states = repeat_kv(key_states, self.num_key_value_groups)
        value_states = repeat_kv(value_states, self.num_key_value_groups)

        attn_output = self.hybrid_attention(
            query_states,
            key_states,
            value_states
        )

        attn_output = attn_output.transpose(1, 2).contiguous()
        attn_output = attn_output.view(bsz, q_len, -1)
        attn_output = self.o_proj(attn_output)

        return attn_output, None, past_key_value
```

            query_states,
            key_states,
            value_states
        )

我们基本上将注意力机制分成了**滑动窗口**和**线性注意力**块。

### 滑动窗口注意力：

```py
def sliding_window_attention(self, query_states, key_states, value_states, window_size, window_factor):
        """Compute sliding window attention"""
        batch_size, num_heads, seq_len, head_dim = query_states.shape

        key_windows = F.pad(key_states, (0, 0, window_size - 1, 0), value=0)
        key_windows = key_windows.unfold(2, window_size, 1)

        value_windows = F.pad(value_states, (0, 0, window_size - 1, 0), value=0)
        value_windows = value_windows.unfold(2, window_size, 1)

        attn_weights = torch.einsum('bhld,bhldw->bhlw', query_states, key_windows) * (head_dim ** -0.5)
        attn_weights = torch.where(attn_weights == 0,
                                 torch.tensor(-float('inf'), device=attn_weights.device),
                                 attn_weights)

        # Apply learnable window factor (with sigmoid to ensure positivity)
        attn_weights = self.factor_activation(window_factor) * F.softmax(attn_weights, dim=-1)

        attn_output = torch.einsum('bhlw,bhldw->bhld', attn_weights, value_windows)
        sum_weights = attn_weights.sum(dim=-1, keepdim=True)

        return attn_output, sum_weights
```

对于对窗口注意力概念的深入理解，我建议参考以下论文：

> [**高效的流式语言模型与注意力汇聚**](https://arxiv.org/abs/2309.17453)

我在这里实现的想法是，我们不是一起计算所有键值对的注意力（其中每个标记都会关注其他所有标记），而是将其分成大小为‘w’的窗口，然后计算每个窗口的注意力。使用上述代码，时间复杂度从 O(n²)降低到 O(n*w)，因为每个标记只需要关注 w 个标记而不是所有 n 个标记。通过使用诸如汇聚等概念，并且只对最后 w 个标记进行窗口操作，这可能会在未来的更新中实现，可以使其变得更好。

### 线性注意力：

```py
def linear_attention(self, query_states, key_states, value_states, window_size, linear_factor):
        """Compute linear attention with cumsum"""
        def feature_map(x):
            return F.elu(x) + 1

        query_prime = feature_map(query_states)
        key_prime = feature_map(key_states)

        key_prime = F.pad(key_prime, (0, 0, window_size, 0), value=0)[:, :, :-window_size, :]
        value_padded = F.pad(value_states, (0, 0, window_size, 0), value=0)[:, :, :-window_size, :]

        # Compute KV
        kv = torch.einsum('bhlf,bhld->bhlfd', key_prime, value_padded)
        # Apply learnable linear factor (with sigmoid to ensure positivity)
        qkv = self.factor_activation(linear_factor) * torch.einsum('bhlf,bhlfd->bhld',
                                                                  query_prime,
                                                                  kv.cumsum(dim=2))

        sum_k = key_prime.cumsum(dim=2)
        sum_qk = self.factor_activation(linear_factor) * torch.einsum('bhld,bhld->bhl',
                                                                     query_prime,
                                                                     sum_k)[..., None]
        sum_qk = torch.where(sum_qk == 0, torch.tensor(1e-12, device=sum_qk.device), sum_qk)

        return qkv, sum_qk
```

对于线性注意力，我使用了一个非常简单的特征映射 elu(x) + 1，但要注意的主要部分是初始填充。这里的想法是，我们只能使用线性注意力来处理[序列长度 – 窗口大小]，因为我们已经有了滑动窗口来跟踪最近的环境。

这两种注意力类型的组合形成了我们新的混合注意力，我们使用 _window*factor*和 _linear*factor*作为可学习的参数，以控制每种类型的注意力对最终输出的贡献程度。

现在我们有了混合块，受到 "**[基于 Mamba 的语言模型的实证研究](https://arxiv.org/abs/2406.07887)**" 论文的启发，我们将只替换一半的 softmax 注意力层，而且是在交替的顺序中。Llama-3.2-1B 有 16 个 softmax 注意力层，我们将按照以下顺序替换其中的 8 个：[0,2,4,6,8,10,12,14]。

## 注意力迁移

实现方法遵循了在 "**[LoLCATs: On Low-Rank Linearizing of Large Language Models](https://arxiv.org/abs/2410.10254)**" 中描述的方法。注意力迁移步骤涉及使用原始块的权重初始化 8 个混合块，并且在训练中我使用了来自[fineweb-edu](https://huggingface.co/datasets/HuggingFaceFW/fineweb-edu)[1] 10B 版本中的 1M 个标记。

这里的基本目标是，我们将冻结 llama-3.2–1B 中的所有参数，然后使用一个训练输入进行前向传递。通过这种方式，我们可以得到我们每个自注意力块输入和输出。然后，我们可以将相同的输入传递给相应的混合块，并计算两个之间的均方误差损失，以训练混合块。这有助于我们明确地告诉混合块模仿 softmax 注意力输出，这将有助于保持准确性。我们为所有块分别这样做，一旦训练完成，我们就可以用我们的混合块替换 llama-3.2–1B 中的自注意力了。从这个新模型中取一个样本输出看起来像这样，

测试结果揭示了模型评估的有趣见解。虽然混合模型略优于 Llama-3.2-1B，但这种差异（大约 2%）应被视为微不足道，尤其是考虑到混合模型还接受了额外的训练，特别是指令微调数据集。

最令人着迷的观察是，在运行相同代码的不同 GPU 上运行时，性能差异很大。当 Llama-3.2-1B 在 L40S GPU 上运行与在 RTX A6000 上运行时，准确率从 25.38%跃升至 32.13%——考虑到所有其他变量保持不变，这是一个显著的区别。这种差异归因于不同的 GPU 处理浮点操作的方式，这表明硬件选择可以意外地影响你的模型性能。

另一个引人注目的是，在这些结果中，5 次提示和 0 次提示性能之间的差异很小，尤其是在 RTX A6000 上。这是出乎意料的，因为 5 次提示通常可以提高性能，尤其是对于像 Llama-3.2-1B 这样的基础模型。实际上，当我在 L40S GPU 上运行 Llama-3.2-1B 时，我观察到 5 次提示和 0 次提示分数之间存在显著的差距——再次突显了 GPU 差异如何影响基准分数。

将同一模型与所有相同的变量进行基准测试，但使用不同的 GPU，这将是一个有趣的未来练习。

## 结论

希望这篇文章已经展示了 softmax 注意力替代品的潜力以及传统 softmax 注意力的固有优势。使用相对有限的计算资源和少量数据集，我们能够实现更快的推理速度，同时保持与混合方法相当的准确率水平。

另一点需要理解的是，基于 softmax 的注意力转换器已经经历了大量的硬件优化，这使得它们在计算复杂度方面与线性替代品具有竞争力。如果对像 mamba 这样的架构投入同样的努力，也许它们可以更具竞争力。

一个有前景的方法是使用 softmax 注意力和线性注意力替代品的混合，以尝试获得两者的最佳效果。Nvidia 在“**[基于 Mamba 的语言模型的实证研究](https://arxiv.org/abs/2406.07887)**”中做到了这一点，并展示了混合方法是如何成为一种有效的替代方案。
