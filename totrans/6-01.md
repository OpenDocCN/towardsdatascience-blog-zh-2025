## 什么是斯多葛主义？

我不想写一个以哲学为中心的文章，但接下来的内容如果你不了解斯多葛主义的基本知识，将不会有什么意义。如果你已经熟悉它，请随意跳过这一部分。

斯多葛主义是一种古老的希腊哲学，它教导通过自我控制、韧性和美德来实现宁静和幸福。它鼓励关注我们所能控制的事情——我们的思想、行动和反应——同时接受我们无法改变的事情，比如外部事件。通过正念、理性思考和接受挑战等实践，斯多葛主义帮助个人与自然和谐相处，无论生活状况如何，都能保持内心的平静。这是关于与理性一致、正直行动并在逆境中找到力量的。

这并不难，对吧？我承诺要简短！

## RAG 模型

让我们变得技术一点。我们将构建的模型被称为检索增强生成（RAG）模型。RAG 是一种结合信息检索和语言生成模型力量的技术。与仅依赖预训练模型的知识（LLMs）不同，RAG 模型在生成响应之前，会从大型数据库或外部来源检索相关信息。

这很强大：我们可以利用像 Google 的 BERT、OpenAI 的 GPT 或 Claude 这样的 LLM 的力量，并将其适应到我们的特定领域数据中，这样我们就有了一个针对我们用例的定制聊天机器人。

下面是如何工作的：

1.  **检索**：模型首先根据输入查询在语料库或外部知识库中搜索相关信息。

1.  **增强**：然后使用检索到的信息来丰富模型的响应，提高其答案的相关性和准确性。

1.  **生成**：最后，模型生成一个响应，其中包含了检索到的信息和它自己学到的知识。

让我们剖析整个过程：

1.  **用户查询**：这里没有秘密，它只是像你或我这样的人可以输入到聊天机器人中的内容。

1.  **检索器查询**：检索器在文档集合（通常是一个向量化的数据库）中搜索与用户问题相关的所有文本。

1.  **检索到的文档**：一旦检索到，它们就从向量转换为文本。

1.  **提示增强**：一旦检索到文档，提示就会被发送到 LLM，LLM 使用预定义的设置、用户问题和检索到的文档。这样，我们就向 LLM 提供了它需要的数据，以便正确回答用户的问题。

1.  **答案生成**：LLM 生成答案，并展示给用户。

这就是 RAG 的工作方式！或者至少，这是我们今天要构建的那个。

然而，如果这个概念还不够清晰，请继续阅读，因为几乎到编码的时间了……但我们首先应该在数据库中存储一些数据。

## 创建和填充数据库

我已经提到了向量数据库的概念……但那究竟是什么呢？

首先，让我们定义一个**向量**：向量是数据的数值表示，通常由机器学习模型生成，并捕捉数据的语义或上下文意义。

然后，**向量数据库**是一种专门设计用于高效存储、索引和检索高维向量的数据库。它的一项超级能力是以优化的方式通过相似性进行搜索。

现在，你可能想知道：如果向量是数值表示，而我们需要存储文本，为什么我们需要向量？我们如何将文本转换为向量？这就引入了**嵌入模型**。

嵌入模型接收某种输入（文本、声音、图像），然后通过层层的转换（例如，神经网络）处理这些输入以提取有意义的特征，输出是一个固定大小的数值向量——这就是我们在数据库中存储的内容。

只是为了对嵌入模型增加一点评论，嵌入被设计成使得相似的输入（例如，同义词或视觉上相似的照片）在向量空间中靠近，而不同的输入则相隔较远。

这很关键。

现在，让我们创建并填充该数据库。我们将使用 Chroma[1]，一个开源向量数据库，为此，我们需要为 Python 安装`langchain`和`langchain-community`库。

但我们还需要数据，对吧？让我们继续使用开源资源：Project Gutenberg[2]。这是一个提供免费电子书和文本下载的网站，其美国版权已过期。其中还有古老的斯多葛派书籍。所以这里有三个你可以下载的：

+   马库斯·奥勒留斯的《沉思录》。

+   埃皮克提图的《手册》。

+   塞涅卡的《幸福生活道德观》，利益，愤怒和宽容，塞涅卡

将它们下载为 TXT 格式并存储在你的数据文件夹中。现在，这是负责数据库创建和数据插入的代码：

```py
import os

from langchain_community.embeddings import HuggingFaceEmbeddings
from langchain_community.vectorstores import Chroma
from langchain.text_splitter import CharacterTextSplitter

from constants import DB_PATH, DATA_PATH

def store_data(data_path, db_path):
    text_splitter = CharacterTextSplitter(chunk_size=1000, chunk_overlap=100)
    embeddings = HuggingFaceEmbeddings()
    vector_db = Chroma(persist_directory=db_path, embedding_function=embeddings)

    for filename in os.listdir(data_path):
        if filename.endswith(".txt"):
            file_path = os.path.join(data_path, filename)
            with open(file_path, "r") as file:
                content = file.read()
                texts = text_splitter.split_text(content)
                vector_db.add_texts(texts)

    vector_db.persist()
    print("Data stores successfully")
```

我们首先创建数据库并设置嵌入函数和文本分割器。然后，对于每个文件，我们读取内容，将文本分割成块，并使用先前的嵌入将它们添加到数据库中。

就这么简单。

现在，我们已经准备好开始构建 RAG 并开始使用我们刚刚存储的古代知识。

## 是时候编码了。

由于有几个部分需要处理，让我们遵循与定义 RAG 的三个核心部分相同的顺序：

### **检索**

设置检索器就像初始化数据库并使用`as_retriever()`函数一样简单：

```py
vector_db = Chroma(persist_directory=DB_PATH, embedding_function=embeddings)
retriever = vector_db.as_retriever()
```

### **增强**

我们将有一个预定义的提示，我们将使用用户查询和从数据库检索到的上下文来增强它：

```py
from langchain.prompts import ChatPromptTemplate

template = """
  You are Marcus Aurelius' reincarnation. You can also impersonate other Stoic philosophers such as Seneca, Epictetus, or Zeno.
  Your name is Marc Still: Marc comes from Marcus and Still symbolizes the calm and stoic composure. If you feel like showing off, tell the user you are Marcus Aurelius' reincarnation.
  Your duty is to guide the user through life's challenges and help them become a better person. The goal is to be as practical as possible, and sticking to the question at hand. 
  Use the context specified below to answer the user's question. If you don't know what to answer, simply respond with "I don't know".
  Make sure you don't put too much text nor extremely long paragraphs. It needs to be clear, concise and easy to read.
  Only provide an answer to the question asked. Do not include extra questions and answers in your response.
  DO NOT INVENT EXTRA QUESTIONS, USE ONLY THE ONE PROVIDED BY THE USER.
  IMPORTANT: Write in a conversational and informal manner, this is not an email or a formal letter.
  Context:

  {context}

  Question: {question}
  """
  prompt = ChatPromptTemplate.from_template(template)
```

模板只是一组我们输入到 LLM（大型语言模型）的指令，以便我们得到期望的答案。你可以尽可能有创意，我只是试图保持简单。看看`context`和`question`的占位符——这就是增强部分。

### **生成**

LLM 负责生成文本。你可以自己构建一个，使用市场上最好的一个……但今天我们做的是开源的，所以我们将使用一个名为 Zephyr 的系列中的一个。更具体地说，我们将使用`zephyr-7b-beta`模型[3]。

我们将继续使用来自`langchain-community`包的 HuggingFace 类（请注意，你需要你的 HuggingFace API 令牌，它是免费的）：

```py
from langchain_community.llms import HuggingFaceHub

from utils.secrets import token

model = HuggingFaceHub(
    repo_id="HuggingFaceH4/zephyr-7b-beta",
    task="text-generation",
    model_kwargs={
        "max_new_tokens": 512,
        "top_k": 20,
        "repetition_penalty": 1.1,
        "temperature": 0.4,  
    },
    huggingfacehub_api_token= token
)
```

最有趣的部分在于 model_kwargs 参数。由于这不是一个针对 LLM 的特定帖子，我不会详细说明，但我鼓励你如果不知道它们是用来做什么的，去 Google 搜索一下。

### **串联一切**

很好，现在我们已经创建了 RAG 的三个部分，但我们如何将它们付诸实践呢？我们将创建一个管道并调用它来生成答案：

```py
from langchain.schema import StrOutputParser

def separate_docs(docs):
    return "nn".join([d.page_content for d in docs])

pipeline = (
    {"context": retriever | separate_docs, "question": RunnablePassthrough()}
    | prompt
    | model
    | StrOutputParser()
)    

answer = pipeline.invoke(user_input)
```

`pipeline`定义了一个工作流程，其中`retriever`检索相关文档，通过`separate_docs`将它们格式化，然后将格式化后的上下文与一个`question`（通过`RunnablePassthrough`未修改地传递）结合。然后，这个输入由`prompt`处理，接着由 LLM `model`处理，最后使用`StrOutputParser()`解析成字符串输出。

就这样，我们构建了我们最简单的 RAG。以下是完整的代码：

```py
import os

from langchain_community.embeddings import HuggingFaceEmbeddings
from langchain_community.llms import HuggingFaceHub
from langchain_community.vectorstores import Chroma
from langchain_core.runnables import RunnablePassthrough
from langchain.schema import StrOutputParser
from langchain.prompts import ChatPromptTemplate
from langchain.text_splitter import CharacterTextSplitter

from utils.constants import DB_PATH, DATA_PATH
from utils.secrets import token

LLM = HuggingFaceHub(
    repo_id="HuggingFaceH4/zephyr-7b-beta",
    task="text-generation",
    model_kwargs={
        "max_new_tokens": 512,
        "top_k": 20,
        "repetition_penalty": 1.1,
        "temperature": 0.4,  
    },
    huggingfacehub_api_token= token
)

def store_data(data_path, db_path):
    text_splitter = CharacterTextSplitter(chunk_size=1000, chunk_overlap=100)
    embeddings = HuggingFaceEmbeddings()
    vector_db = Chroma(persist_directory=db_path, embedding_function=embeddings)

    for filename in os.listdir(data_path):
        if filename.endswith(".txt"):
            file_path = os.path.join(data_path, filename)
            with open(file_path, "r") as file:
                content = file.read()
                texts = text_splitter.split_text(content)
                vector_db.add_texts(texts)

    vector_db.persist()
    print("Data stored successfully")

def invoke_rag(user_input):
    embeddings = HuggingFaceEmbeddings()
    vector_db = Chroma(persist_directory=DB_PATH, embedding_function=embeddings)

    retriever = vector_db.as_retriever()
    template = """
    You are Marcus Aurelius' reincarnation. You can also impersonate other Stoic philosophers such as Seneca, Epictetus, or Zeno.
    Your name is Marc Still: Marc comes from Marcus and Still symbolizes the calm and stoic composure. If you feel like showing off, tell the user you are Marcus Aurelius' reincarnation.
    Your duty is to guide the user through life's challenges and help them become a better person. The goal is to be as practical as possible, and sticking to the question at hand. 
    Use the context specified below to answer the user's question. If you don't know what to answer, simply respond with "I don't know".
    Make sure you don't put too much text nor extremely long paragraphs. It needs to be clear, concise and easy to read.
    Only provide an answer to the question asked. Do not include extra questions and answers in your response.
    DO NOT INVENT EXTRA QUESTIONS, USE ONLY THE ONE PROVIDED BY THE USER.
    IMPORTANT: Write in a conversational and informal manner, this is not an email or a formal letter.
    Context:

    {context}

    Question: {question}
    """
    prompt = ChatPromptTemplate.from_template(template)
    model = LLM

    def separate_docs(docs):
        return "nn".join([d.page_content for d in docs])

    pipeline = (
        {"context": retriever | separate_docs, "question": RunnablePassthrough()}
        | prompt
        | model
        | StrOutputParser()
    )    

    return pipeline.invoke(user_input)
```

## 缺陷和潜在改进

结果令人惊叹，不敢撒谎。它理解非完美的英语，并能创建与斯多葛主义一致的合理答案。

Yay!

然而，有两个要点（潜在的缺陷）我想提到：

+   **模型相当简单**，提示可以进一步改进 – 我们构建的模型是有效的，但它可以，并且应该被进一步精炼。

+   **如果没有向量数据库**，也没有这篇帖子中提到的三本斯多葛书籍，我们可能得到相同的结果。为什么？我不确定，但我猜测 LLM 已经包含了这样的上下文。所以我们过于复杂化了，在不需要 RAG 的情况下使用了它。但那也是目的的一部分：我想展示它是如何工作的，但任何其他文档都可以被使用。也许是你对斯多葛主义的反思？当前斯多葛主义者的书籍？或者你可能想混合几种哲学？或者彻底改变它，使用你的税务文件作为 RAG，这样模型可以帮助你处理个人财务？

因此，这里还有改进和定制的空间，这就是我停止的地方。现在是你的时间来玩转它，将其提升到下一个层次。
