# 用户推荐树的递归遍历

> 原文：[`towardsdatascience.com/recursive-walks-down-user-referral-trees-0864e14042ec/`](https://towardsdatascience.com/recursive-walks-down-user-referral-trees-0864e14042ec/)

在许多现代软件产品中，用户有机会向其他用户推荐/推广应用程序。对于这些推荐计划，一个自然的问题是要问：“谁是我们的最有影响力的产品推广者？”一个简单的方法来回答这个问题是简单地计算每个用户所做的所有推荐，并宣布拥有最多推荐的用户是最有影响力的。

这种方法忽略了关键点。特别是：

+   用户推荐一个有影响力的用户不会得到奖励。也就是说，他们不会因为推荐了一个反过来又推荐了更多用户的用户而获得任何信用。

+   没有奖励是给那些推荐付费更多的用户的。在这种方法中，一个用户可以推荐一个付费很少的人，并且被认为和推荐了一个高消费用户的人一样有影响力。

因此，我们将寻找一种替代方法来观察这个问题。我们将采取递归遍历用户的推荐图，以计算用户推荐的加权和无加权总效果。

### 定义用户推荐树

**用户推荐树**是用户如何向他人推广产品或服务的表示，通常以有向图的形式可视化。在这个树中：

+   **节点**代表单个用户。

+   **边**代表推荐关系，其中从用户 A 到用户 B 的有向边表示用户 A 推荐了用户 B。

### 分析

使用我们的示例数据，如果我们把直接推荐的次数作为衡量用户影响力的标准，那么我们会得出用户 D 是最有影响力的结论（3 次直接推荐）。然而，从图中我们可以看到，这可能是误导性的。让我们探索一些其他的方法来解决这个问题，即通过递归遍历每个给定用户的“推荐树”。

**基本算法**

![用户推荐遍历的基本算法（作者图片）](img/62f7e2da4a84654131178e581026ae62.png)

用户推荐遍历的基本算法（作者图片）

在这里，我们只是计算用户直接推荐的直接推荐数量，并递归地添加每个直接推荐所提供的推荐数量。**这奖励了那些推荐了更多用户的用户。**

使用这种方法，我们最有影响力的用户（拥有最多相关下游推荐的用户）是用户 I，其完整的推荐树包括 5 个用户。

+   用户 A = 2，用户 D = 4，用户 I = 5，用户 F = 1，用户 J = 1，用户 K = 2

**加权算法**

![收入加权的用户推荐遍历（作者图片）](img/3c578b97a9ff284b7bb1e54bb55e862e.png)

收入加权的用户推荐遍历（作者图片）

使用先前的方法，我们更好地理解了给定用户的总推荐影响。但这种方法未能考虑到每个推荐的质量。**为了代理推荐质量，我们将使用给定用户生成的收入量**。这个收入量将作为任何给定推荐的“权重”。

如我们所见，使用这种方法，我们最有影响力的用户（在考虑用户直接或间接推荐的收入总额时）仍然是用户 I，其完整的推荐树包括 5 个用户。

+   用户 A = $70，用户 D = $60，用户 I = $140，用户 F = $20，用户 J = $10，用户 K = $100

**衰减算法**

![衰减用户推荐遍历（作者图片）](img/964658aaaeb3b55294115e64ed2ddc53.png)

衰减用户推荐遍历（作者图片）

两种先前的方法帮助我们更好地理解用户推荐对总影响。为了更进一步，我们可能还需要考虑这样一个事实：间接推荐（即推荐人的推荐）受原始用户的影响较小。为了纠正这一事实，**我们可以“衰减”或减少用户随着推荐层越来越远而获得的归因量**。这种衰减函数可能有多种形式，但在这里为了示例的简便，我们将简单地使用 1 除以当前距离层的分数。

这次，我们最终在用户 D 和用户 I 之间打成了平手。

+   用户 A = 2，用户 D = 3.5，用户 I = 3.5，用户 F = 1，用户 J = 1，用户 K = 2

**综合起来**

![将所有组件组合到一个遍历公式中（作者图片）](img/2a98623606677bc1b4873110fc669330.png)

现在三种技术（递归计数、收入加权和衰减影响）**可以协同使用，以获得完整的图景**，了解在我们的推荐树中哪个用户影响最大。

令人惊讶的是，我们发现用户 K 是我们最具影响力的推荐者。尽管他们只推荐了几个用户，但这些用户的总直接收入影响超过了任何其他推荐用户。这种加权影响的衰减也是用户 I 不再是最具影响力的原因（大部分加权影响是间接发生的）

+   用户 A = $70，用户 D = $50，用户 I = $85，用户 F = $20，用户 J = $10，用户 K = $100

### 结论

准确衡量用户在推荐计划中的影响力，不仅仅是简单地计算直接推荐的数量。通过利用用户推荐树下的递归遍历、纳入收入权重以及应用衰减因子，我们能够更深入地理解特定用户的影响。

这种方法不仅奖励用户推荐的数量，也奖励其质量，考虑到他们产生的收入和树状结构的级联影响。它强调了在评估推荐影响时全面视角的重要性，确保为对树状结构的长期增长和盈利做出贡献的用户得到适当的认可。请注意，上述方法仅作为起点，并且有无数种方式可以调整上述方法，以达到最能反映您产品特性的归因系统。

通过这些方法，推荐计划可以针对有影响力的用户模式进行优化，并设计与之相符的增长激励结构。最终，这种精细化的测量确保了公平的归因，并有助于揭示由推荐驱动的增长更清晰的视角。
